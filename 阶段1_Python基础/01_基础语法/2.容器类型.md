# 容器类型：list / dict / tuple / set

## 一、为什么需要容器？

一个变量只能存一个值。当你需要存储**一组数据**时，就需要容器。

```python
# 不用容器：需要 100 个变量存 100 个学生
student1 = "张三"
student2 = "李四"
# ... 太蠢了

# 用容器：一个变量搞定
students = ["张三", "李四", "王五", ...]
```

### 四种容器速览

| 容器 | 符号 | 有序 | 可变 | 重复 | 典型场景 |
|------|------|------|------|------|----------|
| list | `[]` | ✅ | ✅ | ✅ | 存储一组同类数据 |
| dict | `{}` | ✅(3.7+) | ✅ | key不可 | 键值对映射 |
| tuple | `()` | ✅ | ❌ | ✅ | 不应被修改的数据 |
| set | `{}` | ❌ | ✅ | ❌ | 去重、集合运算 |

## 二、列表 list —— 最常用的容器

### 创建

```python
fruits = ["苹果", "香蕉", "橘子"]
numbers = [1, 2, 3, 4, 5]
mixed = [1, "hello", True, 3.14]     # 可以混合类型（但不推荐）
empty = []                            # 空列表
```

### 访问元素（索引从 0 开始）

```python
fruits = ["苹果", "香蕉", "橘子", "葡萄", "西瓜"]

print(fruits[0])     # 苹果（第一个）
print(fruits[-1])    # 西瓜（最后一个）
print(fruits[-2])    # 葡萄（倒数第二个）
```

### 切片（截取子列表）

```python
fruits = ["苹果", "香蕉", "橘子", "葡萄", "西瓜"]

print(fruits[1:3])     # ['香蕉', '橘子']    索引 1 到 2（不包含 3）
print(fruits[:3])      # ['苹果', '香蕉', '橘子']  从头到索引 2
print(fruits[2:])      # ['橘子', '葡萄', '西瓜']  从索引 2 到末尾
print(fruits[::2])     # ['苹果', '橘子', '西瓜']  步长为 2
print(fruits[::-1])    # ['西瓜', '葡萄', '橘子', '香蕉', '苹果']  反转
```

**记忆口诀**：`[start:stop:step]`，包含 start，不包含 stop。

### 常用操作

```python
fruits = ["苹果", "香蕉"]

# === 增 ===
fruits.append("橘子")           # 末尾添加: ['苹果', '香蕉', '橘子']
fruits.insert(1, "草莓")        # 指定位置插入: ['苹果', '草莓', '香蕉', '橘子']
fruits.extend(["葡萄", "西瓜"]) # 合并列表: [..., '葡萄', '西瓜']

# === 删 ===
fruits.remove("草莓")           # 按值删除（只删第一个匹配的）
fruits.pop()                     # 删除并返回最后一个
fruits.pop(0)                    # 删除并返回指定索引
del fruits[0]                    # 删除指定索引
fruits.clear()                   # 清空列表

# === 改 ===
fruits = ["苹果", "香蕉"]
fruits[0] = "芒果"               # 直接修改: ['芒果', '香蕉']

# === 查 ===
print("苹果" in fruits)          # True   是否存在
print(fruits.index("香蕉"))      # 1      查找索引
print(fruits.count("苹果"))      # 1      计数
print(len(fruits))               # 2      长度

# === 排序 ===
nums = [3, 1, 4, 1, 5]
nums.sort()                      # 原地排序: [1, 1, 3, 4, 5]
nums.sort(reverse=True)          # 降序: [5, 4, 3, 1, 1]
sorted_nums = sorted(nums)       # 不修改原列表，返回新列表
```

## 三、字典 dict —— 键值对映射

### 创建

```python
user = {
    "name": "张三",
    "age": 20,
    "city": "北京"
}
empty = {}
```

### 常用操作

```python
user = {"name": "张三", "age": 20}

# === 访问 ===
print(user["name"])              # 张三（key 不存在会报 KeyError）
print(user.get("name"))          # 张三
print(user.get("email", "无"))   # 无（key 不存在返回默认值，不报错）

# === 增/改 ===
user["email"] = "zhangsan@qq.com"   # 新增
user["age"] = 21                     # 修改（key 已存在则为修改）

# === 删 ===
del user["email"]                    # 删除指定 key
user.pop("age")                      # 删除并返回值

# === 遍历 ===
for key in user:                     # 遍历 key
    print(key, user[key])

for key, value in user.items():      # 遍历 key-value 对（最常用）
    print(f"{key}: {value}")

for value in user.values():          # 只遍历 value
    print(value)

# === 其他 ===
print("name" in user)               # True   判断 key 是否存在
print(len(user))                     # 键值对数量
user.update({"age": 22, "job": "dev"})  # 批量更新/合并
```

### 字典嵌套

```python
# 实际项目中常见的嵌套结构（类似 JSON）
api_response = {
    "code": 200,
    "data": {
        "users": [
            {"id": 1, "name": "张三"},
            {"id": 2, "name": "李四"}
        ],
        "total": 2
    }
}

# 访问嵌套数据
first_user = api_response["data"]["users"][0]["name"]   # "张三"
```

## 四、元组 tuple —— 不可变的列表

### 创建

```python
point = (3, 4)
rgb = (255, 128, 0)
single = (42,)       # 单元素元组，注意逗号！
not_tuple = (42)     # 这只是整数 42，不是元组

# 拆包
x, y = point         # x=3, y=4
```

### 特点

```python
t = (1, 2, 3)
print(t[0])      # 1     可以访问
t[0] = 10        # ❌ TypeError: 不能修改

# 元组的意义：
# 1. 保护数据不被意外修改
# 2. 可以作为字典的 key（list 不行）
# 3. 比 list 更节省内存
```

### 实际使用场景

```python
# 1. 函数返回多个值（本质上返回元组）
def get_position():
    return 3, 4       # 等价于 return (3, 4)

x, y = get_position()

# 2. 交换变量
a, b = 1, 2
a, b = b, a           # a=2, b=1

# 3. 不可变的配置
ALLOWED_HOSTS = ("localhost", "127.0.0.1")
```

## 五、集合 set —— 去重 & 集合运算

### 创建

```python
fruits = {"苹果", "香蕉", "橘子"}
empty = set()          # 空集合不能用 {}，{} 是空字典！

# 从列表去重
nums = [1, 2, 2, 3, 3, 3]
unique = set(nums)     # {1, 2, 3}
unique_list = list(set(nums))   # [1, 2, 3]（转回列表）
```

### 集合运算

```python
a = {1, 2, 3, 4}
b = {3, 4, 5, 6}

print(a | b)     # {1, 2, 3, 4, 5, 6}  并集
print(a & b)     # {3, 4}               交集
print(a - b)     # {1, 2}               差集（a 有 b 没有）
print(a ^ b)     # {1, 2, 5, 6}         对称差集（互相没有的）
```

### 常用操作

```python
s = {1, 2, 3}

s.add(4)              # 添加元素
s.remove(2)           # 删除（不存在报错）
s.discard(10)         # 删除（不存在不报错）
print(3 in s)         # True   判断存在
```

## 六、容器选择指南

```
需要存一组有序数据？
├── 需要修改？ → list
└── 不需要修改？ → tuple

需要键值对映射？ → dict

需要去重或集合运算？ → set
```

### 性能对比

| 操作 | list | dict | set |
|------|------|------|-----|
| 查找 `in` | O(n) 慢 | O(1) 快 | O(1) 快 |
| 索引访问 | O(1) | - | - |
| 添加 | O(1) | O(1) | O(1) |
| 删除 | O(n) | O(1) | O(1) |

**实际建议**：数据量大且需要频繁查找时，用 dict 或 set，不要用 list。
