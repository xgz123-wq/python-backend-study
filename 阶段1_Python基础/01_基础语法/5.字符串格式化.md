# 字符串格式化

## 一、为什么需要格式化？

将变量插入到字符串中，是编程中最频繁的操作之一。

```python
name = "张三"
age = 20

# 字符串拼接（丑且容易出错）
print("我叫" + name + "，今年" + str(age) + "岁")

# 格式化（优雅且清晰）
print(f"我叫{name}，今年{age}岁")
```

---

## 二、三种格式化方式

### 1. f-string（推荐 ⭐⭐⭐）

Python 3.6+ 引入，**最简洁、最快、最推荐**。

```python
name = "张三"
age = 20
score = 95.678

# 基本用法
print(f"姓名: {name}")
print(f"明年 {age + 1} 岁")          # 可以写表达式
print(f"{'通过' if score >= 60 else '未通过'}")  # 可以写三元表达式

# 格式控制
print(f"分数: {score:.2f}")           # 95.68（保留 2 位小数）
print(f"百分比: {0.856:.1%}")         # 85.6%
print(f"编号: {42:05d}")              # 00042（补零到 5 位）
print(f"金额: {1234567:,}")           # 1,234,567（千分位逗号）

# 对齐
print(f"{'左对齐':<20}|")            # 左对齐               |
print(f"{'右对齐':>20}|")            #                右对齐|
print(f"{'居中':^20}|")              #         居中         |

# 调试技巧（3.8+）
x = 42
print(f"{x = }")                      # x = 42（自动显示变量名和值）
```

### 2. format() 方法

```python
# 位置参数
print("我叫{}，今年{}岁".format("张三", 20))

# 索引参数
print("{0}喜欢{1}，{1}也喜欢{0}".format("张三", "李四"))

# 命名参数
print("姓名: {name}, 年龄: {age}".format(name="张三", age=20))

# 格式控制（和 f-string 一样）
print("{:.2f}".format(3.14159))    # 3.14
```

### 3. % 格式化（旧式，了解即可）

```python
name = "张三"
age = 20
score = 95.5

print("我叫%s，今年%d岁，分数%.1f" % (name, age, score))
# %s = 字符串, %d = 整数, %f = 浮点数
```

---

## 三、格式化语法速查表

以 f-string 为例：`f"{value:格式}"`

| 格式 | 示例 | 结果 | 说明 |
|------|------|------|------|
| `.2f` | `f"{3.14159:.2f}"` | `3.14` | 保留 2 位小数 |
| `.1%` | `f"{0.856:.1%}"` | `85.6%` | 百分比 |
| `,` | `f"{1000000:,}"` | `1,000,000` | 千分位 |
| `05d` | `f"{42:05d}"` | `00042` | 补零 |
| `>10` | `f"{'hi':>10}"` | `        hi` | 右对齐 |
| `<10` | `f"{'hi':<10}"` | `hi        ` | 左对齐 |
| `^10` | `f"{'hi':^10}"` | `    hi    ` | 居中 |

## 四、实际应用场景

### 日志输出

```python
import datetime

level = "INFO"
message = "用户登录成功"
timestamp = datetime.datetime.now()

print(f"[{timestamp:%Y-%m-%d %H:%M:%S}] [{level:>5}] {message}")
# [2024-01-15 14:30:00] [ INFO] 用户登录成功
```

### API 响应构造

```python
def format_response(code, message, data=None):
    return {
        "code": code,
        "message": message,
        "data": data
    }

# 在错误信息中使用格式化
user_id = 42
error_msg = f"用户 {user_id} 不存在"
response = format_response(404, error_msg)
```

### SQL 查询（注意：不要用 f-string 拼 SQL！）

```python
user_id = 42

# ❌ 安全隐患！SQL 注入风险！
query = f"SELECT * FROM users WHERE id = {user_id}"

# ✅ 使用参数化查询
query = "SELECT * FROM users WHERE id = %s"
cursor.execute(query, (user_id,))
```

## 五、方式对比与选择

| 方式 | 可读性 | 性能 | 功能 | 推荐度 |
|------|--------|------|------|--------|
| f-string | ⭐⭐⭐ | ⭐⭐⭐ | ⭐⭐⭐ | 首选 |
| format() | ⭐⭐ | ⭐⭐ | ⭐⭐⭐ | 动态模板时使用 |
| % | ⭐ | ⭐⭐ | ⭐⭐ | 仅旧代码维护 |

**结论**：日常开发统一用 f-string，只有模板字符串需要延迟渲染时用 format()。
