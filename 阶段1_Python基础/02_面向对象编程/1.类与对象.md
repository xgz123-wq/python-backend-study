# 类与对象

## 一、为什么需要 OOP？

面向过程：一堆函数，数据和操作分散。  
面向对象：把**数据**和**操作这些数据的方法**打包在一起，形成"对象"。

```python
# 面向过程：数据和操作分散
name = "张三"
age = 20
balance = 1000.0

def deposit(balance, amount):
    return balance + amount

def withdraw(balance, amount):
    if balance >= amount:
        return balance - amount
    raise ValueError("余额不足")

# 面向对象：数据 + 操作打包成对象
class BankAccount:
    def __init__(self, name, balance=0):
        self.name = name
        self.balance = balance

    def deposit(self, amount):
        self.balance += amount

    def withdraw(self, amount):
        if self.balance >= amount:
            self.balance -= amount
        else:
            raise ValueError("余额不足")

account = BankAccount("张三", 1000)
account.deposit(500)   # 更清晰，account 自己的操作
```

**OOP 的三大好处**：
1. **封装**：数据和方法在一起，外部不需要关心内部细节
2. **复用**：通过继承复用代码
3. **扩展**：通过多态灵活扩展行为

## 二、定义类和创建对象

### 最简单的类

```python
class Dog:
    pass   # 空类，什么都没有

# 创建对象（实例化）
dog = Dog()
print(type(dog))    # <class '__main__.Dog'>
```

### `__init__` 构造方法

`__init__` 在创建对象时自动调用，用于**初始化属性**。

```python
class Dog:
    def __init__(self, name, age):
        # self 代表当前对象本身
        self.name = name    # 实例属性
        self.age = age      # 实例属性

    def bark(self):
        print(f"{self.name} 说：汪汪！")

    def info(self):
        return f"{self.name}，{self.age}岁"

# 创建对象
dog1 = Dog("小白", 3)
dog2 = Dog("旺财", 5)

dog1.bark()           # 小白 说：汪汪！
dog2.bark()           # 旺财 说：汪汪！
print(dog1.info())    # 小白，3岁
```

### self 是什么？

`self` 就是调用方法的对象本身，Python 自动传入，你不需要手动传。

```python
dog1.bark()
# 等价于
Dog.bark(dog1)   # self = dog1
```

## 三、实例属性 vs 类属性

```python
class Student:
    # 类属性：所有对象共享
    school = "北京大学"
    count = 0    # 统计创建了多少个学生

    def __init__(self, name, grade):
        # 实例属性：每个对象独有
        self.name = name
        self.grade = grade
        Student.count += 1   # 每创建一个对象，计数加 1

s1 = Student("张三", 90)
s2 = Student("李四", 85)

print(s1.name)         # 张三（实例属性）
print(s1.school)       # 北京大学（类属性，也可以通过实例访问）
print(Student.school)  # 北京大学（类属性，通过类访问）
print(Student.count)   # 2（所有对象共享）
```

**⚠️ 陷阱：通过实例修改类属性**

```python
# 这不是修改类属性，而是给实例创建了一个同名的实例属性！
s1.school = "清华大学"
print(s1.school)       # 清华大学（实例属性，遮蔽了类属性）
print(s2.school)       # 北京大学（类属性，不受影响）
print(Student.school)  # 北京大学（类属性，没变）

# 正确修改类属性的方式
Student.school = "清华大学"
print(s2.school)       # 清华大学（所有对象都变了）
```

## 四、方法类型

### 实例方法（最常用）

```python
class Circle:
    def __init__(self, radius):
        self.radius = radius

    def area(self):          # 操作实例数据，第一个参数是 self
        return 3.14 * self.radius ** 2
```

### 类方法 `@classmethod`

操作**类属性**，第一个参数是 `cls`（代表类本身）。常用于工厂方法（提供多种创建对象的方式）。

```python
class Date:
    def __init__(self, year, month, day):
        self.year = year
        self.month = month
        self.day = day

    @classmethod
    def from_string(cls, date_str):
        """从字符串 '2024-01-15' 创建 Date 对象"""
        year, month, day = map(int, date_str.split("-"))
        return cls(year, month, day)    # 等价于 Date(year, month, day)

    def __str__(self):
        return f"{self.year}-{self.month:02d}-{self.day:02d}"

# 普通方式
d1 = Date(2024, 1, 15)

# 使用工厂类方法
d2 = Date.from_string("2024-03-20")
print(d2)   # 2024-03-20
```

### 静态方法 `@staticmethod`

不操作实例数据，也不操作类属性，只是一个**归属于类的工具函数**。

```python
class MathUtils:
    @staticmethod
    def is_prime(n):
        """判断是否为质数（不需要 self 或 cls）"""
        if n < 2:
            return False
        for i in range(2, int(n**0.5) + 1):
            if n % i == 0:
                return False
        return True

print(MathUtils.is_prime(17))   # True
print(MathUtils.is_prime(4))    # False
```

### 三种方法对比

| 方法类型 | 装饰器 | 第一个参数 | 可访问 |
|----------|--------|-----------|--------|
| 实例方法 | 无 | `self` | 实例属性 + 类属性 |
| 类方法 | `@classmethod` | `cls` | 类属性 |
| 静态方法 | `@staticmethod` | 无 | 无（纯工具函数） |

## 五、`__str__` 和 `__repr__`

控制对象的字符串表示。

```python
class User:
    def __init__(self, name, email):
        self.name = name
        self.email = email

    def __str__(self):
        """给用户看的字符串，print() 时调用"""
        return f"用户: {self.name} <{self.email}>"

    def __repr__(self):
        """给开发者看的字符串，调试时显示，要能重现对象"""
        return f"User(name='{self.name}', email='{self.email}')"

user = User("张三", "zs@qq.com")

print(user)       # 用户: 张三 <zs@qq.com>   ← __str__
print(repr(user)) # User(name='张三', email='zs@qq.com')  ← __repr__

# 在列表中显示使用 __repr__
users = [user]
print(users)      # [User(name='张三', email='zs@qq.com')]
```

## 六、后端开发中的类

在 FastAPI 后端项目中，类无处不在：

```python
# Pydantic 数据模型（验证请求数据）
from pydantic import BaseModel

class UserCreate(BaseModel):
    name: str
    email: str
    password: str

# SQLAlchemy 数据库模型
from sqlalchemy import Column, Integer, String
from sqlalchemy.orm import declarative_base

Base = declarative_base()

class UserModel(Base):
    __tablename__ = "users"
    id = Column(Integer, primary_key=True)
    name = Column(String(50))
    email = Column(String(100), unique=True)

# 服务层（业务逻辑）
class UserService:
    def __init__(self, db_session):
        self.db = db_session

    def create_user(self, data: UserCreate) -> UserModel:
        user = UserModel(name=data.name, email=data.email)
        self.db.add(user)
        self.db.commit()
        return user
```

> 📝 **讲解（2026-04-16 补充）**
>
> 这段代码是一个"预告片"，现阶段不需要会写，只需要看懂结构。
> 整段代码只做一件事：**接收前端数据，验证后写入数据库**。三个类分工明确：
>
> ---
>
> **第一个类 `UserCreate`：数据验证层（相当于"表单"）**
>
> ```python
> from pydantic import BaseModel   # Pydantic 是一个第三方数据验证库，后面会专门学
>
> class UserCreate(BaseModel):     # 继承 BaseModel，就自动获得验证能力
>     name: str       # name 字段，必须是字符串（这是"类型提示"，之前学过）
>     email: str      # email 字段，必须是字符串
>     password: str   # password 字段，必须是字符串
> ```
>
> 作用：前端提交的数据会自动套进这个类验证。
> 缺字段 → 自动报错；类型不对 → 自动报错。不需要自己写 if 判断。
>
> ---
>
> **第二个类 `UserModel`：数据库模型层（相当于"数据库表的 Python 映射"）**
>
> ```python
> from sqlalchemy import Column, Integer, String   # SQLAlchemy 是操作数据库的库，后面会学
> from sqlalchemy.orm import declarative_base
>
> Base = declarative_base()   # 固定写法，创建一个基类，所有"数据库表类"都要继承它
>
> class UserModel(Base):              # 继承 Base，SQLAlchemy 就认识这个类对应一张表
>     __tablename__ = "users"         # 数据库里这张表的名字叫 "users"
>
>     id    = Column(Integer, primary_key=True)   # id 列，整数，主键（每行唯一标识）
>     name  = Column(String(50))                  # name 列，字符串，最长 50 字符
>     email = Column(String(100), unique=True)    # email 列，最长 100 字符，且不能重复
> ```
>
> 理解方式：把这个类想象成就是数据库里的一张表，
> 每个 `Column(...)` 就是表的一列，创建一个 `UserModel` 对象就等于准备写入一行数据。
>
> ---
>
> **第三个类 `UserService`：服务层（相当于"业务逻辑处理员"）**
>
> ```python
> class UserService:
>     def __init__(self, db_session):  # 接收数据库连接，保存在 self.db
>         self.db = db_session
>
>     #              ↓ 类型提示：data 应该是 UserCreate 类型
>     #                                        ↓ 类型提示：返回值是 UserModel 类型
>     def create_user(self, data: UserCreate) -> UserModel:
>         user = UserModel(name=data.name, email=data.email)
>         #      ↑ 用 UserCreate 里的数据，创建一个 UserModel 对象（准备写入数据库的一行）
>
>         self.db.add(user)    # 告诉数据库连接：我要插入这一行
>         self.db.commit()     # 提交事务，真正执行 INSERT INTO users ... 这条 SQL
>         return user          # 返回刚创建的用户对象
> ```
>
> `data: UserCreate` 和 `-> UserModel` 都是**类型提示**（你在基础语法中已经接触过），
> 只是说明参数/返回值的类型，不影响代码运行，主要用于代码提示和可读性。
>
> ---
>
> **三层关系（用快递来比喻）**：
>
> ```
> 前端发来请求数据
>   ↓
> UserCreate（验收员）：检查数据完不完整、格式对不对
>   ↓ 数据合格
> UserService（快递员）：拿到合格数据，执行业务
>   ↓ 调用
> UserModel（仓库货架标签）：描述数据库的表结构，把数据写进去
>   ↓
> 数据库真正存储
> ```
>
> **现阶段只需理解**：OOP 让后端代码有了清晰的分层，每个类只负责一件事。
> 具体的 Pydantic、SQLAlchemy、FastAPI 用法在第三、四阶段会详细学习。
