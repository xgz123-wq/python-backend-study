# @property 装饰器

## 一、问题：直接暴露属性的缺陷

```python
class User:
    def __init__(self, name, age):
        self.name = name
        self.age = age

user = User("张三", 20)
user.age = -5    # ❌ 年龄不可能为负，但没有任何阻止
user.age = "二十"  # ❌ 类型错误，也没有阻止
```

### 传统解决方案（getter/setter）：太啰嗦

```python
class User:
    def __init__(self, name, age):
        self.name = name
        self._age = age

    def get_age(self):
        return self._age

    def set_age(self, value):
        if not isinstance(value, int) or value < 0:
            raise ValueError("年龄必须是非负整数")
        self._age = value

user = User("张三", 20)
user.set_age(25)          # 啰嗦
print(user.get_age())     # 啰嗦
```

> **补充知识点：为什么要用下划线前缀 `_`**
>
> 在 Python 中，`_age` 是"伪私有属性"的约定写法。当你在属性名前加 `_` 时，是在告诉其他开发者："这个属性是内部实现细节，请不要直接访问，改用公开的方法/property"。
>
> 虽然 Python 没有真正的"私有"机制，`_age` 在技术上仍可被访问，但这是约定俗成的信号：
>
> - `self.age`：公开属性，外部可以直接读写
> - `self._age`：内部属性，标志"请走 property/方法"，不要直接碰
>
> 后面的代码里，你会看到所有用 `@property` 保护的字段，都会用下划线存储真实值。这样做的好处是：
>
> 1. 清晰地区分"对外接口"和"内部实现"
> 2. 避免外部代码绕过校验，直接修改内部状态
> 3. 如果将来改成非 property，也能快速识别哪些字段是特殊的

### Python 方案：`@property`

用起来像访问属性，底层走验证逻辑，两全其美：

```python
class User:
    def __init__(self, name, age):
        self.name = name
        self.age = age    # 触发 setter 验证

    @property
    def age(self):
        """getter：读取属性时调用"""
        return self._age

    @age.setter
    def age(self, value):
        """setter：赋值时调用"""
        if not isinstance(value, int) or value < 0:
            raise ValueError(f"年龄无效: {value}")
        self._age = value

user = User("张三", 20)
print(user.age)    # 20  ← 像普通属性一样读取
user.age = 25      # ← 像普通属性一样赋值，但底层走 setter 验证
# user.age = -5   # ← 会触发 ValueError
```

#### 补充讲解：这段 `@property` 代码到底是怎么工作的

按执行顺序看，会更容易理解：

1. 创建对象：`user = User("张三", 20)`
2. 进入 `__init__`，执行 `self.age = age`
3. 因为 `age` 是 property，这一步会调用 `@age.setter`
4. setter 校验通过后，真正把值存进 `self._age`
5. 读取 `user.age` 时，会调用 `@property def age(...)`（getter）

也就是说：

- 外部写法看起来是“访问属性”
- 内部实际走的是“方法逻辑”

这就是 `@property` 的核心价值：既保留属性语法的简洁，又能加上校验与控制。

#### 为什么内部要用 `_age`，而不是直接用 `age`

这是新手最容易踩坑的点。

如果你在 setter 里写：`self.age = value`，就会再次触发 setter，本质上无限递归。
如果你在 getter 里写：`return self.age`，也会再次触发 getter，同样无限递归。

所以需要一个“内部真实存储字段”，通常命名为 `_age`。

#### 你可以记住这个判断规则

- 只要一个字段“可能需要校验/转换/联动”，就考虑改成 property
- 只做普通存储、没有控制逻辑时，直接公开属性即可

#### 迷你对照（牢记）

- `obj.age`：触发 getter
- `obj.age = 18`：触发 setter
- `del obj.age`：若定义了 deleter，则触发 deleter

## 二、完整示例

```python
class Temperature:
    """温度类，内部用摄氏度存储，提供摄氏/华氏/开氏三种访问方式"""

    def __init__(self, celsius=0):
        self.celsius = celsius    # 触发 setter

    @property
    def celsius(self):
        return self._celsius

    @celsius.setter
    def celsius(self, value):
        if value < -273.15:
            raise ValueError(f"温度不能低于绝对零度: {value}°C")
        self._celsius = value

    @property
    def fahrenheit(self):
        """华氏度（只读属性，没有 setter）"""
        return self._celsius * 9/5 + 32

    @property
    def kelvin(self):
        """开氏度（只读属性）"""
        return self._celsius + 273.15

    def __repr__(self):
        return f"Temperature({self._celsius}°C)"


t = Temperature(100)
print(t.celsius)      # 100
print(t.fahrenheit)   # 212.0
print(t.kelvin)       # 373.15

t.celsius = 0
print(t.fahrenheit)   # 32.0

# t.fahrenheit = 50   # ← AttributeError: 只读属性，没有 setter
# t.celsius = -300    # ← ValueError: 低于绝对零度
```

## 三、`@property` 的三种形态

```python
class Circle:
    def __init__(self, radius):
        self.radius = radius

    # 1. getter（只读属性）
    @property
    def area(self):
        return 3.14159 * self.radius ** 2

    # 2. getter + setter（可读写，带验证）
    @property
    def radius(self):
        return self._radius

    @radius.setter
    def radius(self, value):
        if value < 0:
            raise ValueError("半径不能为负")
        self._radius = value

    # 3. getter + setter + deleter（可删除）
    @property
    def diameter(self):
        return self._radius * 2

    @diameter.setter
    def diameter(self, value):
        self._radius = value / 2

    @diameter.deleter
    def diameter(self):
        del self._radius


c = Circle(5)
print(c.area)       # 78.53975（只读）
print(c.diameter)   # 10
c.diameter = 20     # 通过 setter 同步修改 radius
print(c.radius)     # 10
```

## 四、`@property` 的实际应用场景

### 场景 1：计算属性（避免重复计算或数据不一致）

```python
class Rectangle:
    def __init__(self, width, height):
        self.width = width
        self.height = height

    @property
    def area(self):
        """面积：由 width 和 height 计算，始终与它们同步"""
        return self.width * self.height

    @property
    def perimeter(self):
        return 2 * (self.width + self.height)

r = Rectangle(4, 6)
print(r.area)       # 24
r.width = 10
print(r.area)       # 60（自动更新，不需要手动重新计算）
```

### 场景 2：数据验证

```python
class User:
    @property
    def email(self):
        return self._email

    @email.setter
    def email(self, value):
        if "@" not in value:
            raise ValueError(f"邮箱格式错误: {value}")
        self._email = value.lower().strip()    # 自动规范化

    @property
    def password(self):
        return "***"    # 密码永远不返回真实值！

    @password.setter
    def password(self, value):
        if len(value) < 8:
            raise ValueError("密码至少 8 位")
        import hashlib
        self._password_hash = hashlib.sha256(value.encode()).hexdigest()
```

### 场景 3：懒加载（第一次访问时才计算）

```python
class UserProfile:
    def __init__(self, user_id):
        self.user_id = user_id
        self._stats = None    # 先不加载

    @property
    def stats(self):
        """第一次访问时才从数据库加载，之后缓存"""
        if self._stats is None:
            print(f"正在加载用户 {self.user_id} 的统计数据...")
            # 模拟数据库查询
            self._stats = {"posts": 10, "followers": 100}
        return self._stats

profile = UserProfile(42)
# 创建对象时不查询数据库
print(profile.stats)    # 这时才查询：正在加载...
print(profile.stats)    # 直接返回缓存，不再查询
```

## 五、直接用属性 vs 用 `@property` 的选择

| 情况 | 选择 |
|------|------|
| 简单存储，无需验证 | 直接用属性 `self.name = name` |
| 需要验证输入值 | 用 `@property` + setter |
| 需要计算属性（由其他属性推导） | 用 `@property`（只读） |
| 需要懒加载 | 用 `@property` + 内部缓存 |
| 需要在属性变化时触发副作用 | 用 `@property` + setter |

**原则**：先用普通属性，需要控制时再改成 `@property`，外部调用代码不需要改变。
