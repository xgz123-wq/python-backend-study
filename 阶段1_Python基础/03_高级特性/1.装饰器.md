# 装饰器

## 一、为什么需要装饰器？

假设你有 10 个函数，想给每个函数都加上"执行耗时统计"功能：

```python
# ❌ 笨办法：每个函数里都手动加计时代码
import time

def create_user():
    start = time.time()
    # ... 业务逻辑 ...
    print(f"耗时: {time.time() - start:.3f}s")

def delete_user():
    start = time.time()
    # ... 业务逻辑 ...
    print(f"耗时: {time.time() - start:.3f}s")

# 10 个函数就要写 10 遍，重复代码爆炸
```

装饰器解决的问题：**不修改原函数代码，给函数添加新功能**。

```python
# ✅ 装饰器：写一次，到处用
@timer
def create_user():
    # ... 业务逻辑（完全不用改）...
    pass

@timer
def delete_user():
    # ... 业务逻辑 ...
    pass
```

## 二、前置知识：闭包

要理解装饰器，必须先理解闭包。

### 函数是"一等公民"

在 Python 中，函数可以像变量一样：赋值、传参、返回。

```python
def greet(name):
    return f"你好，{name}"

# 函数可以赋值给变量
say_hello = greet
print(say_hello("张三"))   # 你好，张三

# 函数可以作为参数传递
def call_func(func, arg):
    return func(arg)

print(call_func(greet, "李四"))   # 你好，李四
```

### 闭包 = 内部函数 + 记住外部变量

```python
def make_greeter(greeting):
    """外部函数，返回一个内部函数"""
    def greeter(name):
        # 内部函数引用了外部函数的变量 greeting
        return f"{greeting}，{name}！"
    return greeter    # 返回函数本身（不加括号）

# greeting="你好" 被记住了
hello = make_greeter("你好")
print(hello("张三"))   # 你好，张三！
print(hello("李四"))   # 你好，李四！

# greeting="再见" 被记住了
bye = make_greeter("再见")
print(bye("张三"))     # 再见，张三！
```

**闭包的关键**：`greeter` 函数"记住了"外部的 `greeting` 变量，即使 `make_greeter` 已经执行完毕。

## 三、装饰器的核心原理

### 最简单的装饰器

装饰器本质上就是：**接收一个函数，返回一个新函数**。

```python
import time

def timer(func):
    """计时装饰器"""
    def wrapper(*args, **kwargs):
        start = time.time()
        result = func(*args, **kwargs)     # 调用原函数
        elapsed = time.time() - start
        print(f"[{func.__name__}] 耗时: {elapsed:.3f}s")
        return result                      # 返回原函数的结果
    return wrapper
```

### 手动使用

```python
def slow_add(a, b):
    time.sleep(0.1)
    return a + b

# 手动装饰：把原函数传进去，得到新函数
slow_add = timer(slow_add)
result = slow_add(1, 2)    # [slow_add] 耗时: 0.100s
print(result)               # 3
```

### @ 语法糖

`@timer` 就是 `slow_add = timer(slow_add)` 的简写：

```python
@timer
def slow_add(a, b):
    time.sleep(0.1)
    return a + b

# 完全等价于：
# slow_add = timer(slow_add)
```

## 四、functools.wraps — 保留原函数信息

### 问题：装饰后函数信息丢失

```python
@timer
def slow_add(a, b):
    """两数相加"""
    time.sleep(0.1)
    return a + b

print(slow_add.__name__)    # wrapper  ← 不是 slow_add！
print(slow_add.__doc__)     # None     ← 文档字符串也丢了！
```

### 解决：使用 functools.wraps

```python
from functools import wraps

def timer(func):
    @wraps(func)    # ← 加这一行，把原函数信息复制到 wrapper 上
    def wrapper(*args, **kwargs):
        start = time.time()
        result = func(*args, **kwargs)
        print(f"[{func.__name__}] 耗时: {time.time() - start:.3f}s")
        return result
    return wrapper

@timer
def slow_add(a, b):
    """两数相加"""
    time.sleep(0.1)
    return a + b

print(slow_add.__name__)    # slow_add  ✅
print(slow_add.__doc__)     # 两数相加  ✅
```

> ⚠️ **规则**：写装饰器时，`wrapper` 上面必须加 `@wraps(func)`，没有例外。

## 五、带参数的装饰器

有时候装饰器本身也需要接受参数，比如指定重试次数。

### 结构：三层嵌套

```python
from functools import wraps

def retry(max_attempts=3):
    """带参数的装饰器：指定最大重试次数"""
    def decorator(func):              # 真正的装饰器
        @wraps(func)
        def wrapper(*args, **kwargs):
            for attempt in range(1, max_attempts + 1):
                try:
                    return func(*args, **kwargs)
                except Exception as e:
                    print(f"  第 {attempt} 次失败: {e}")
                    if attempt == max_attempts:
                        raise
        return wrapper
    return decorator                  # 返回真正的装饰器

@retry(max_attempts=3)   # 先调用 retry(3)，返回 decorator，再 @decorator
def unstable_api():
    import random
    if random.random() < 0.7:
        raise ConnectionError("网络超时")
    return "成功"
```

### 为什么要“多一层”？

先看无参装饰器：

```python
@timer
def foo():
    pass

# 等价于
foo = timer(foo)
```

这里 `timer` 直接拿到的是函数 `foo`，所以两层就够：

```
timer(func) -> wrapper
```

再看带参装饰器：

```python
@retry(max_attempts=3)
def foo():
    pass
```

这行代码会先执行 `retry(max_attempts=3)`。注意：

1. 这一步还没有 `foo`，只有装饰器参数 `max_attempts=3`。
2. 所以这一层不能直接包装函数，只能先把参数记住，返回一个“真正接收函数”的装饰器。

于是必须多一层：

```
第1层 factory:    retry(max_attempts)      # 接收装饰器参数，保存配置
第2层 decorator:  decorator(func)           # 接收被装饰函数
第3层 wrapper:    wrapper(*args, **kwargs)  # 接收函数调用参数，执行增强逻辑
```

可以把它理解成“先配置，再绑定，再执行”：

```
配置阶段：retry(3)                -> 得到 decorator（配置已记住）
绑定阶段：decorator(foo)          -> 得到 wrapper（foo 被包起来）
调用阶段：wrapper(...)            -> 真正执行重试逻辑 + 调用 foo
```

一句话总结：

`多出来的那一层` 不是为了“更复杂”，而是为了先接收并保存装饰器自己的参数；
等拿到真实函数后，再返回最终可调用的 `wrapper`。

**理解三层嵌套**：

```
retry(max_attempts=3)   → 返回 decorator
@decorator              → decorator(unstable_api) → 返回 wrapper
unstable_api()          → 实际调用 wrapper()
```

## 六、多个装饰器叠加

装饰器可以叠加使用，**执行顺序从下往上**（靠近函数的先执行）：

```python
@decorator_a
@decorator_b
def my_func():
    pass

# 等价于：
# my_func = decorator_a(decorator_b(my_func))
```

```python
from functools import wraps

def bold(func):
    @wraps(func)
    def wrapper(*args, **kwargs):
        return f"<b>{func(*args, **kwargs)}</b>"
    return wrapper

def italic(func):
    @wraps(func)
    def wrapper(*args, **kwargs):
        return f"<i>{func(*args, **kwargs)}</i>"
    return wrapper

@bold
@italic
def greet(name):
    return f"Hello, {name}"

print(greet("张三"))   # <b><i>Hello, 张三</i></b>
# 执行顺序：先 italic 包裹，再 bold 包裹
```

## 七、类装饰器

用类来实现装饰器，利用 `__call__` 魔术方法：

```python
class CountCalls:
    """统计函数被调用了多少次"""
    def __init__(self, func):
        self.func = func
        self.count = 0

    def __call__(self, *args, **kwargs):
        self.count += 1
        print(f"  [{self.func.__name__}] 第 {self.count} 次调用")
        return self.func(*args, **kwargs)

@CountCalls
def say_hello(name):
    print(f"  Hello, {name}!")

say_hello("张三")    # [say_hello] 第 1 次调用 → Hello, 张三!
say_hello("李四")    # [say_hello] 第 2 次调用 → Hello, 李四!
print(f"总共调用: {say_hello.count} 次")   # 2
```

## 八、后端实际场景

装饰器在后端开发中极其常见：

| 场景 | 说明 |
|------|------|
| `@app.get("/users")` | FastAPI 路由注册 |
| `@login_required` | 登录验证，未登录拒绝访问 |
| `@cache(ttl=300)` | 结果缓存 300 秒 |
| `@retry(max_attempts=3)` | 失败自动重试 |
| `@timer` | 接口耗时统计 |
| `@validate_params` | 参数校验 |
| `@permission("admin")` | 权限检查 |

### 应用时怎么落地（工程视角）

先记一条判断标准：

装饰器最适合处理“横切关注点”，也就是很多接口都要做、但不属于核心业务逻辑的事情。

例如：鉴权、权限、参数校验、日志、计时、重试、缓存。

#### 场景拆解

1. `@login_required`（登录验证）
    - 什么时候用：所有需要登录后访问的接口。
    - 核心动作：从请求中取 token/session，验证身份，不通过就直接返回 401。
    - 工程建议：只做“你是谁”的判断，不要把“你能做什么”混到这一层。

2. `@permission("admin")`（权限检查）
    - 什么时候用：同样是登录用户，但不同角色权限不同。
    - 核心动作：基于当前用户角色/权限集判断是否允许执行。
    - 工程建议：与登录验证解耦，便于复用和测试。

3. `@validate_params`（参数校验）
    - 什么时候用：需要补充框架默认校验之外的业务规则时。
    - 示例：时间范围是否合法、分页大小是否超限、字段组合是否冲突。
    - 工程建议：校验失败尽早返回，避免后续进入数据库或外部调用。

4. `@timer`（耗时统计）
    - 什么时候用：观察慢接口、定位性能瓶颈。
    - 核心动作：记录函数执行时长并输出到日志或监控系统。
    - 工程建议：不要只看平均值，更要关注 P95/P99 这样的高分位延迟。

5. `@retry(max_attempts=3)`（失败重试）
    - 什么时候用：调用外部依赖时出现“临时性失败”（网络抖动、短暂超时）。
    - 核心动作：失败后按策略重试，超过次数再抛错。
    - 工程建议：只对幂等操作使用重试；最好配合指数退避与随机抖动。

6. `@cache(ttl=300)`（结果缓存）
    - 什么时候用：读多写少、短时间内重复请求多、计算/查询昂贵的接口。
    - 核心动作：命中缓存直接返回，未命中才执行函数并回填缓存。
    - 工程建议：缓存 key 要包含关键参数（如用户、查询条件、版本号）；写操作后要考虑失效策略。

#### 装饰器叠加时的实战建议

- 思路：先拦截无效请求，再做昂贵操作。
- 经验顺序（按调用链目标理解）：登录验证 -> 权限检查 -> 参数校验 -> 缓存/重试 -> 业务函数。
- 注意：多个装饰器叠加时，要区分“装饰阶段”和“调用阶段”；框架装饰器（如 FastAPI 路由）通常按官方示例放在最上面。

#### 一句话落地口诀

把装饰器当作“接口中间件”：
能复用、能解耦、能统一治理，但不要让装饰器承载核心业务分支。

```python
# FastAPI 中的装饰器实际用法（预告）
from fastapi import FastAPI

app = FastAPI()

@app.get("/users/{user_id}")    # ← 这就是一个带参装饰器
def get_user(user_id: int):
    return {"id": user_id, "name": "张三"}
```

## 九、常见坑

### 坑1：忘记 @wraps(func)

```python
# ❌ 忘记 @wraps
def my_decorator(func):
    def wrapper(*args, **kwargs):
        return func(*args, **kwargs)
    return wrapper

@my_decorator
def hello():
    """打招呼"""
    pass

print(hello.__name__)   # wrapper（不是 hello）
print(hello.__doc__)    # None（文档丢了）
```

### 坑2：装饰器没有返回 wrapper

```python
# ❌ 忘记 return wrapper
def my_decorator(func):
    @wraps(func)
    def wrapper(*args, **kwargs):
        return func(*args, **kwargs)
    # 忘了 return wrapper！

@my_decorator
def hello():
    pass

hello()   # TypeError: 'NoneType' object is not callable
```

### 坑3：带参装饰器忘记调用

```python
# ❌ 忘记加括号
@retry           # 应该是 @retry() 或 @retry(max_attempts=5)
def my_func():
    pass
# TypeError: decorator() missing required argument 'func'
```

### 坑4：wrapper 没有传递 *args, **kwargs

```python
# ❌ wrapper 参数写死了
def my_decorator(func):
    @wraps(func)
    def wrapper():    # 没有 *args, **kwargs
        return func()
    return wrapper

@my_decorator
def add(a, b):
    return a + b

add(1, 2)   # TypeError: wrapper() takes 0 positional arguments
```

## 十、记忆口诀

```
装饰器 = 闭包 + 函数参数化
无参装饰器：两层嵌套（decorator → wrapper）
带参装饰器：三层嵌套（factory → decorator → wrapper）
必须：@wraps(func) + return wrapper + *args/**kwargs
```
