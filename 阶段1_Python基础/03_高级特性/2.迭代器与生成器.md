# 迭代器与生成器

## 一、为什么需要迭代器和生成器？

假设你要处理一个 1000 万行的日志文件：

```python
# ❌ 一次性全部读入内存 → 内存爆炸
lines = open("huge.log").readlines()   # 可能占用几 GB 内存
for line in lines:
    process(line)

# ✅ 逐行读取，用完即丢 → 内存始终只占一行的量
for line in open("huge.log"):          # 文件对象是迭代器，逐行产出
    process(line)
```

**核心思想**：不一次性生成所有数据，而是**按需一个一个产出**（惰性求值），节省内存。

---

## 二、可迭代对象 vs 迭代器

### 概念区分

| 概念 | 定义 | 判断方式 | 例子 |
|------|------|----------|------|
| **可迭代对象**（Iterable） | 能用 `for` 循环遍历的对象 | 有 `__iter__()` 方法 | list、dict、str、set、tuple、文件 |
| **迭代器**（Iterator） | 能用 `next()` 逐个取值的对象 | 有 `__iter__()` + `__next__()` | `iter([1,2,3])`、生成器、文件对象 |

```python
# list 是可迭代对象，但不是迭代器
nums = [1, 2, 3]
# next(nums)   # ❌ TypeError: 'list' object is not an iterator

# 用 iter() 把可迭代对象变成迭代器
it = iter(nums)
print(next(it))   # 1
print(next(it))   # 2
print(next(it))   # 3
# print(next(it))  # ❌ StopIteration（取完了）
```

### for 循环的本质

```python
for item in [1, 2, 3]:
    print(item)

# 等价于：
it = iter([1, 2, 3])       # 1. 调用 __iter__() 获取迭代器
while True:
    try:
        item = next(it)     # 2. 调用 __next__() 获取下一个值
        print(item)
    except StopIteration:   # 3. 没有更多值了，结束循环
        break
```

## 三、自定义迭代器

实现 `__iter__` 和 `__next__` 两个方法：

```python
class Countdown:
    """倒计时迭代器"""
    def __init__(self, start):
        self.current = start

    def __iter__(self):
        return self    # 迭代器返回自身

    def __next__(self):
        if self.current <= 0:
            raise StopIteration    # 没有更多值了
        value = self.current
        self.current -= 1
        return value

# 使用
for num in Countdown(5):
    print(num, end=" ")   # 5 4 3 2 1

# 也可以手动 next()
cd = Countdown(3)
print(next(cd))   # 3
print(next(cd))   # 2
print(next(cd))   # 1
```

## 四、生成器 — 用 yield 简化迭代器

### yield 关键字

`yield` 让函数变成**生成器函数**，调用时不执行函数体，而是返回一个生成器对象（迭代器）。

```python
def countdown(n):
    """生成器函数：用 yield 代替自定义 __iter__/__next__"""
    while n > 0:
        yield n       # 产出一个值，然后暂停
        n -= 1        # 下次 next() 时从这里继续

# 调用生成器函数，得到生成器对象
gen = countdown(5)
print(type(gen))      # <class 'generator'>

print(next(gen))      # 5（执行到 yield，暂停）
print(next(gen))      # 4（从上次暂停处继续）
print(next(gen))      # 3

# 用 for 循环消费
for num in countdown(5):
    print(num, end=" ")   # 5 4 3 2 1
```

### yield 的执行流程

```python
def simple_gen():
    print("开始")
    yield 1          # 暂停点 1
    print("继续1")
    yield 2          # 暂停点 2
    print("继续2")
    yield 3          # 暂停点 3
    print("结束")

gen = simple_gen()
print(next(gen))   # 输出"开始"，返回 1，暂停
print(next(gen))   # 输出"继续1"，返回 2，暂停
print(next(gen))   # 输出"继续2"，返回 3，暂停
# next(gen)        # 输出"结束"，抛出 StopIteration
```

## 五、生成器表达式

类似列表推导式，但用 `()` 包裹，返回生成器（惰性求值）：

```python
# 列表推导式：立即生成所有元素，占内存
squares_list = [x**2 for x in range(1000000)]    # 立即占用大量内存

# 生成器表达式：按需生成，几乎不占内存
squares_gen = (x**2 for x in range(1000000))     # 只占几十字节

# 对比内存
import sys
print(sys.getsizeof(squares_list))   # ≈ 8 MB
print(sys.getsizeof(squares_gen))    # ≈ 200 字节
```

**使用场景**：只需遍历一次、数据量大时，用生成器表达式代替列表推导式。

```python
# 求前 100 万个整数的平方和
total = sum(x**2 for x in range(1000000))   # 生成器表达式直接传给 sum
```

## 六、yield from — 委托生成器

`yield from` 简化嵌套生成器的写法：

```python
# ❌ 手动转发
def chain_manual(*iterables):
    for it in iterables:
        for item in it:
            yield item

# ✅ 用 yield from 简化
def chain(*iterables):
    for it in iterables:
        yield from it    # 等价于 for item in it: yield item

result = list(chain([1, 2], [3, 4], [5]))
print(result)   # [1, 2, 3, 4, 5]
```

## 七、send() 和 throw() — 高级用法

### send()：向生成器发送值

```python
def accumulator():
    """累加器：每次接收一个值，返回累计总和"""
    total = 0
    while True:
        value = yield total    # yield 既产出 total，又接收外部发来的 value
        if value is None:
            break
        total += value

gen = accumulator()
next(gen)           # 启动生成器（必须先调用一次 next）→ 0
print(gen.send(10)) # 发送 10 → 返回 10
print(gen.send(20)) # 发送 20 → 返回 30
print(gen.send(30)) # 发送 30 → 返回 60
```

> 📝 **注意**：`send()` 在后端日常开发中很少直接使用，但 Python 的 `asyncio` 协程底层就是基于 `send()` 实现的。了解即可。

### throw()：向生成器抛异常

```python
def careful_gen():
    try:
        yield 1
        yield 2
    except ValueError:
        print("捕获到 ValueError，优雅退出")

gen = careful_gen()
print(next(gen))         # 1
gen.throw(ValueError)    # 捕获到 ValueError，优雅退出
```

## 八、后端实际场景

| 场景 | 说明 |
|------|------|
| 大文件逐行处理 | 日志分析、CSV 导入，不需要全部加载到内存 |
| 数据库分批查询 | 每次查 1000 条 yield 出去，避免一次 SELECT 百万条 |
| 流式响应 | FastAPI 的 `StreamingResponse` 用生成器逐块发送数据 |
| 分页数据生成 | 按页 yield，客户端按需请求 |
| 管道处理 | 生成器串联：读取 → 过滤 → 转换 → 输出 |

```python
# 后端场景：数据库分批查询
def fetch_users_in_batches(db, batch_size=1000):
    """分批从数据库读取用户"""
    offset = 0
    while True:
        batch = db.query(f"SELECT * FROM users LIMIT {batch_size} OFFSET {offset}")
        if not batch:
            break
        yield from batch    # 逐个产出用户
        offset += batch_size

# 使用：无论数据库有多少用户，内存中始终只有 1000 条
for user in fetch_users_in_batches(db):
    process(user)
```

```python
# 后端场景：生成器管道（日志分析）
def read_log(filename):
    with open(filename) as f:
        yield from f

def filter_errors(lines):
    for line in lines:
        if "ERROR" in line:
            yield line

def extract_message(lines):
    for line in lines:
        yield line.split("ERROR")[-1].strip()

# 管道串联：读取 → 过滤错误 → 提取消息
logs = read_log("app.log")
errors = filter_errors(logs)
messages = extract_message(errors)

for msg in messages:
    print(msg)
```

## 九、常见坑

### 坑1：生成器只能遍历一次

```python
gen = (x for x in range(3))
print(list(gen))   # [0, 1, 2]
print(list(gen))   # []  ← 空了！生成器已耗尽

# 如果需要多次遍历，转成 list 或重新创建生成器
```

### 坑2：忘记 next() 启动就 send()

```python
def my_gen():
    value = yield
    print(value)

gen = my_gen()
# gen.send(42)   # ❌ TypeError: can't send non-None value to a just-started generator
next(gen)         # ✅ 先 next() 启动
gen.send(42)      # 然后才能 send
```

### 坑3：在生成器表达式中做有副作用的操作

```python
# ❌ 生成器是惰性的，不消费就不会执行
results = (print(x) for x in range(5))   # 什么都不会打印！
# 必须消费才会执行
list(results)   # 现在才打印 0 1 2 3 4
```

### 坑4：混淆 return 和 yield

```python
def my_gen():
    yield 1
    return "done"   # 生成器中 return 的值会成为 StopIteration 的参数
    yield 2         # 永远不会执行

gen = my_gen()
print(next(gen))   # 1
# next(gen)        # StopIteration: done
```

## 十、记忆口诀

```
可迭代 → 能 for 循环（有 __iter__）
迭代器 → 能 next()（有 __iter__ + __next__）
生成器 → 带 yield 的函数，自动就是迭代器
惰性求值 → 按需产出，省内存
一次性 → 生成器用完就空了
```
