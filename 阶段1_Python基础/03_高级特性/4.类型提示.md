# 类型提示

> 学习补充（AI讲解）  
> 这章核心一句话：类型提示是“静态约束 + 文档化接口”，主要作用在开发阶段，而不是运行阶段。

## 一、为什么需要类型提示？

Python 是动态类型语言，变量不需要声明类型。这很灵活，但也带来问题：

> 学习补充（AI讲解）  
> 动态类型降低了写代码门槛，但在项目变大后，理解成本会急剧上升。类型提示就是给函数签名“补全语义”。

```python
# ❌ 没有类型提示：看函数签名完全不知道传什么、返回什么
def process(data, config):
    pass
# data 是什么？字符串？列表？字典？
# config 是什么？文件路径？配置对象？
# 返回什么？None？布尔？字典？

# ✅ 有类型提示：一眼看懂
def process(data: list[dict], config: dict[str, str]) -> bool:
    pass
# data 是字典列表，config 是字符串字典，返回布尔值
```

**类型提示的好处**：
1. **代码可读性**：看签名就知道怎么调用
2. **IDE 智能提示**：自动补全、参数提示
3. **静态检查**：用 mypy 在运行前发现类型错误
4. **团队协作**：减少沟通成本

> ⚠️ **重要**：类型提示只是"标注"，Python **不会**在运行时强制检查。传错了类型不会自动报错。但 IDE 和 mypy 会帮你提前发现问题。

> 学习补充（AI讲解）  
> 你可以把它看成“编译前预警系统”：尽早发现问题，而不是等线上运行才暴露。

## 二、基本类型标注

> 学习补充（AI讲解）  
> 初学阶段优先把“函数参数 + 返回值”标全，收益最大；变量标注可按需补充。

### 变量标注

```python
name: str = "张三"
age: int = 20
price: float = 9.99
is_active: bool = True
```

### 函数参数和返回值

```python
def greet(name: str) -> str:
    return f"你好，{name}"

def add(a: int, b: int) -> int:
    return a + b

def no_return(msg: str) -> None:
    print(msg)
```

### 默认参数

```python
def create_user(name: str, age: int = 18, email: str = "") -> dict:
    return {"name": name, "age": age, "email": email}
```

> 学习补充（AI讲解）  
> 这里返回 `dict` 比较宽泛，实战建议标成更具体的类型（例如 `dict[str, str | int]`）。

## 三、容器类型标注

> 学习补充（AI讲解）  
> 记忆方式：容器外层写结构，方括号里写元素类型；越具体，IDE 和检查器越有帮助。

### Python 3.9+ 内置写法（推荐）

Python 3.9 开始可以直接用小写的内置类型：

```python
# 列表
names: list[str] = ["张三", "李四"]

# 字典
scores: dict[str, int] = {"张三": 90, "李四": 85}

# 元组（固定长度，每个位置类型可不同）
point: tuple[int, int] = (10, 20)

# 元组（不定长度，所有元素同类型）
ids: tuple[int, ...] = (1, 2, 3, 4, 5)

# 集合
tags: set[str] = {"python", "backend"}
```

### Python 3.8 及更早版本

需要从 `typing` 导入大写版本：

```python
from typing import List, Dict, Tuple, Set

names: List[str] = ["张三", "李四"]
scores: Dict[str, int] = {"张三": 90}
point: Tuple[int, int] = (10, 20)
tags: Set[str] = {"python"}
```

> 📝 现在推荐使用 3.9+ 的小写写法，大写版本在 3.9+ 中标记为已弃用。

> 学习补充（AI讲解）  
> 团队里最重要的是“统一风格”，避免一半 `List` 一半 `list` 的混写。

## 四、typing 模块常用类型

> 学习补充（AI讲解）  
> 这一节最常用的其实是 4 个：`Optional`、`Union`、`Callable`、`Literal`。

### Optional — 可能为 None

```python
from typing import Optional

def find_user(user_id: int) -> Optional[dict]:
    """返回用户字典，找不到返回 None"""
    users = {1: {"name": "张三"}}
    return users.get(user_id)    # 可能返回 dict，也可能返回 None

# Optional[X] 等价于 X | None（Python 3.10+）
```

### Union — 多种类型之一

```python
from typing import Union

def format_id(id_value: Union[int, str]) -> str:
    """接受 int 或 str 类型的 ID"""
    return str(id_value)

# Python 3.10+ 可以用 | 语法
def format_id(id_value: int | str) -> str:
    return str(id_value)
```

### Any — 任意类型

```python
from typing import Any

def log(message: str, data: Any = None) -> None:
    """data 可以是任何类型"""
    print(f"[LOG] {message}: {data}")
```

> 学习补充（AI讲解）  
> `Any` 是“关闭类型检查”，能不用尽量不用；优先用 `object` 或更具体的联合类型。

### Callable — 可调用对象（函数）

```python
from typing import Callable

def apply(func: Callable[[int, int], int], a: int, b: int) -> int:
    """接受一个函数作为参数
    Callable[[参数类型列表], 返回类型]
    """
    return func(a, b)

result = apply(lambda x, y: x + y, 3, 4)   # 7
```

### Literal — 限定具体值

```python
from typing import Literal

def set_status(status: Literal["active", "inactive", "banned"]) -> None:
    """status 只能是这三个值之一"""
    print(f"状态设为: {status}")

set_status("active")      # ✅
set_status("deleted")     # ❌ mypy 会报错（运行时不会报错）
```

> 学习补充（AI讲解）  
> `Literal` 很适合状态字段、策略名称、配置开关这类“枚举值有限”的参数。

## 五、TypeAlias — 类型别名

当类型标注很长时，用别名简化：

> 学习补充（AI讲解）  
> 别名本质是“给复杂类型起业务名”，可读性会大幅提升，尤其在 DTO/响应结构里。

```python
from typing import TypeAlias

# 定义类型别名
UserDict: TypeAlias = dict[str, str | int | bool]
UserList: TypeAlias = list[UserDict]
ResponseData: TypeAlias = dict[str, str | int | list | dict | None]

def get_users() -> UserList:
    return [
        {"name": "张三", "age": 20, "active": True},
        {"name": "李四", "age": 25, "active": False},
    ]

# Python 3.12+ 可以用 type 语句
# type UserDict = dict[str, str | int | bool]
```

## 六、TypeVar 与泛型

> 学习补充（AI讲解）  
> 泛型的价值是“保持输入输出类型关联”，而不是把所有东西都写成通用。

### TypeVar — 泛型类型变量

```python
from typing import TypeVar

T = TypeVar("T")    # 声明一个类型变量 T

def first(items: list[T]) -> T:
    """返回列表的第一个元素，返回类型和列表元素类型一致"""
    return items[0]

# 传入 list[str]，返回 str
name = first(["张三", "李四"])       # 类型推断为 str
# 传入 list[int]，返回 int
num = first([1, 2, 3])              # 类型推断为 int
```

### 限定 TypeVar 的范围

```python
from typing import TypeVar

# T 只能是 int 或 float
Number = TypeVar("Number", int, float)

def double(x: Number) -> Number:
    return x * 2

double(5)       # ✅ int
double(3.14)    # ✅ float
double("hi")    # ❌ mypy 报错
```

> 学习补充（AI讲解）  
> `TypeVar` 约束后，类型系统能保留更精确的信息，比直接写 `int | float` 更“可推断”。

## 七、Protocol — 结构化类型（鸭子类型的类型化）

Python 崇尚"鸭子类型"：不关心你是什么类，只关心你有没有所需的方法。`Protocol` 把这个理念正式化：

> 学习补充（AI讲解）  
> `Protocol` 的关键词是“结构兼容”。不需要继承同一个基类，只要方法签名匹配就能通过类型检查。

```python
from typing import Protocol

class Readable(Protocol):
    """任何有 read() 方法的对象都算 Readable"""
    def read(self) -> str:
        ...

def process_data(source: Readable) -> str:
    return source.read().upper()

# 不需要继承 Readable，只要有 read() 方法就行
class FileSource:
    def read(self) -> str:
        return "file data"

class ApiSource:
    def read(self) -> str:
        return "api data"

process_data(FileSource())   # ✅ 有 read() 方法
process_data(ApiSource())    # ✅ 有 read() 方法
```

## 八、后端实际场景

类型提示在后端开发中无处不在，尤其是 FastAPI 整个框架都基于类型提示构建：

> 学习补充（AI讲解）  
> 在 FastAPI 里，类型提示不仅是提示信息，还会直接驱动请求校验、文档生成和响应序列化。

```python
# FastAPI 路由（预告）
from fastapi import FastAPI
from pydantic import BaseModel

app = FastAPI()

class UserCreate(BaseModel):
    name: str
    email: str
    age: int | None = None    # 可选字段

class UserResponse(BaseModel):
    id: int
    name: str
    email: str

@app.post("/users", response_model=UserResponse)
def create_user(user: UserCreate) -> UserResponse:
    # FastAPI 自动根据类型提示：
    # 1. 验证请求体是否符合 UserCreate
    # 2. 生成 API 文档
    # 3. 序列化响应为 UserResponse 格式
    return UserResponse(id=1, name=user.name, email=user.email)
```

> 学习补充（AI讲解：逐行看这段代码在做什么）  
> 1. `app = FastAPI()`：创建 Web 应用实例，后面通过它注册路由。  
> 2. `UserCreate(BaseModel)`：定义“请求体模型”，前端传来的 JSON 会按这个结构校验与解析。  
> 3. `age: int | None = None`：`age` 可不传，或传 `null`，属于可选字段。  
> 4. `UserResponse(BaseModel)`：定义“响应体模型”，接口返回数据会按这个结构组织。  
> 5. `@app.post("/users", response_model=UserResponse)`：声明这是 `POST /users` 接口，并指定响应模型用于约束输出和生成文档。  
> 6. `def create_user(user: UserCreate) -> UserResponse`：  
>    `user: UserCreate` 表示请求体自动映射为 `UserCreate` 对象；  
>    `-> UserResponse` 表示返回结果语义应符合 `UserResponse`。  
> 7. `return UserResponse(...)`：返回标准化响应对象，字段和类型都更清晰可控。  
> 8. 关键理解：在 FastAPI 中，类型提示不仅给人看，还会参与运行流程（校验、文档、序列化）。

> 学习补充（AI讲解：这一知识点要掌握到什么程度）  
> 入门阶段不需要一次学很深，但建议至少达到“能独立写简单接口”的程度，而不是只停留在概念了解。  
> 最低够用标准：  
> 1. 能写清楚参数与返回值类型（如 `user: UserCreate`、`-> UserResponse`）。  
> 2. 能区分请求模型（`UserCreate`）和响应模型（`UserResponse`）。  
> 3. 知道 FastAPI 会基于类型提示做自动校验、自动文档、响应序列化。  
> 4. 知道“类型提示本身”不是 Python 运行时强校验，真正执行校验的是 FastAPI/Pydantic。  
> 达到这 4 点，就已经是后端学习阶段的可用水平。

| 场景 | 说明 |
|------|------|
| FastAPI 路由参数 | 自动验证 + 自动文档 |
| Pydantic 数据模型 | 完全基于类型提示进行数据校验 |
| SQLAlchemy 2.0 | 新版 ORM 大量使用类型标注 |
| 函数签名 | 团队协作中必须标注，减少沟通成本 |
| IDE 自动补全 | 提升开发效率，减少查文档次数 |

> 学习补充（AI讲解）  
> 所以后端项目里“类型提示写得好”通常等于“接口稳定、协作顺畅、重构风险低”。

## 九、mypy 静态检查

> 学习补充（AI讲解）  
> 学习阶段可先不强依赖 mypy，但建议至少在关键模块上跑检查，尽早习惯类型反馈。

```bash
# 安装
pip install mypy

# 检查单个文件
mypy my_script.py

# 检查整个项目
mypy src/
```

```python
# example.py
def add(a: int, b: int) -> int:
    return a + b

result = add("hello", "world")   # 类型错误！
```

```bash
$ mypy example.py
example.py:4: error: Argument 1 to "add" has incompatible type "str"; expected "int"
```

> 📝 初学阶段不强制使用 mypy，但养成写类型提示的习惯很重要。到 FastAPI 阶段，类型提示是必须的。

> 学习补充（AI讲解）  
> 一个实用策略：先把新代码标注好，再逐步补旧代码，避免一次性改动过大。

## 十、常见坑

> 学习补充（AI讲解）  
> 这部分建议你重点记住两件事：类型提示不等于运行时校验；类型提示不能修复 Python 语义坑（如可变默认值）。

### 坑1：类型提示不是运行时检查

```python
def greet(name: str) -> str:
    return f"你好，{name}"

greet(123)   # 运行时不会报错！只是 mypy 会提示类型不对
# 输出：你好，123
```

> 学习补充（AI讲解）  
> 想要运行时强校验，通常要配合 Pydantic / 手动校验逻辑，而不是只靠注解。

### 坑2：可变默认参数的类型标注

```python
# ❌ 还是那个经典坑，类型提示不能解决
def append_to(item: int, lst: list[int] = []) -> list[int]:
    lst.append(item)
    return lst

# ✅ 正确写法
def append_to(item: int, lst: list[int] | None = None) -> list[int]:
    if lst is None:
        lst = []
    lst.append(item)
    return lst
```

> 学习补充（AI讲解）  
> 这是非常高频面试点：`[]` 作为默认值只会创建一次，后续调用会复用同一对象。

### 坑3：混用 Python 3.9+ 写法和 typing 写法

```python
# ❌ 不要混用
from typing import List
def process(items: list[str], result: List[int]) -> None:   # 混用！
    pass

# ✅ 统一用一种风格（推荐 3.9+ 小写）
def process(items: list[str], result: list[int]) -> None:
    pass
```

### 坑4：忘记标注返回 None

```python
# ❌ 没标注返回值，别人不知道会不会返回 None
def save_user(user: dict):
    db.insert(user)

# ✅ 明确标注返回 None
def save_user(user: dict) -> None:
    db.insert(user)
```

## 十一、记忆口诀

```
类型提示 = 给代码加说明书，不影响运行
基本类型: str / int / float / bool / None
容器类型: list[X] / dict[K,V] / tuple[X,Y] / set[X]
可选: Optional[X] = X | None
多选: Union[X, Y] = X | Y
任意: Any
函数: Callable[[参数], 返回]
泛型: TypeVar("T")
鸭子类型: Protocol
后端必备: FastAPI/Pydantic 全靠类型提示
```

> 学习补充（AI讲解）  
> 自测一句话：类型提示的目标不是“让 Python 变静态语言”，而是“让协作和重构更可控”。
