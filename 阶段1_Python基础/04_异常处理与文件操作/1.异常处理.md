# 异常处理

## 一、为什么需要异常处理？

程序运行中总会遇到意外情况：文件不存在、网络超时、用户输入非法数据。如果不处理这些异常，程序会直接崩溃。

```python
# ❌ 不处理异常：程序崩溃
num = int("abc")    # ValueError: invalid literal for int()
# 后面的代码全部不会执行

# ✅ 处理异常：程序继续运行
try:
    num = int("abc")
except ValueError:
    num = 0
    print("输入不是数字，使用默认值 0")
# 程序正常继续
```

**后端开发中异常处理极其重要**：一个 API 接口如果因为未处理的异常崩溃，会导致整个服务不可用。

## 二、try/except 基础

### 基本语法

```python
try:
    # 可能出错的代码
    result = 10 / 0
except ZeroDivisionError:
    # 出错时执行
    print("除数不能为零")
```

### 捕获多种异常

```python
# 方式一：分别捕获（推荐，可以做不同处理）
try:
    num = int(input("请输入数字: "))
    result = 100 / num
except ValueError:
    print("输入的不是数字")
except ZeroDivisionError:
    print("不能除以零")

# 方式二：合并捕获（做相同处理时用）
try:
    num = int(input("请输入数字: "))
    result = 100 / num
except (ValueError, ZeroDivisionError) as e:
    print(f"输入有误: {e}")
```

### 获取异常信息

```python
try:
    open("不存在.txt")
except FileNotFoundError as e:
    print(f"异常类型: {type(e).__name__}")   # FileNotFoundError
    print(f"异常信息: {e}")                  # [Errno 2] No such file or directory: '不存在.txt'
```

### 捕获所有异常

```python
# Exception 是所有常规异常的基类
try:
    risky_operation()
except Exception as e:
    print(f"出错了: {e}")

# ⚠️ 不要用裸 except（不带异常类型），会捕获包括 KeyboardInterrupt 在内的所有异常
# except:    ← 不推荐
```

## 三、else 和 finally

### 完整语法

```python
try:
    result = 10 / 2
except ZeroDivisionError:
    print("除数不能为零")
else:
    # 没有异常时才执行
    print(f"计算成功: {result}")
finally:
    # 无论如何都执行（清理资源）
    print("计算完毕")
```

### 执行流程

```
正常情况：try → else → finally
异常情况：try → except → finally
```

### else 的价值

```python
# ❌ 不用 else：不知道 process(data) 的异常是 open 的还是 process 的
try:
    f = open("data.txt")
    data = f.read()
    process(data)       # 这里的异常也会被下面的 except 捕获！
except FileNotFoundError:
    print("文件不存在")

# ✅ 用 else：只捕获 try 块中的异常
try:
    f = open("data.txt")
    data = f.read()
except FileNotFoundError:
    print("文件不存在")
else:
    process(data)       # 只有文件打开成功才执行，这里的异常不会被上面的 except 捕获
finally:
    f.close()
```

## 四、常见内置异常

| 异常 | 触发场景 | 示例 |
|------|----------|------|
| `ValueError` | 值不合法 | `int("abc")` |
| `TypeError` | 类型不对 | `"1" + 1` |
| `KeyError` | 字典键不存在 | `d = {}; d["x"]` |
| `IndexError` | 列表索引越界 | `[1,2][5]` |
| `AttributeError` | 属性不存在 | `"hello".foo()` |
| `FileNotFoundError` | 文件不存在 | `open("xxx.txt")` |
| `ZeroDivisionError` | 除以零 | `1 / 0` |
| `ImportError` | 导入模块失败 | `import not_exist` |
| `StopIteration` | 迭代器耗尽 | `next(iter([]))` |
| `ConnectionError` | 网络连接失败 | 请求外部 API 超时 |
| `TimeoutError` | 操作超时 | 数据库查询超时 |
| `PermissionError` | 权限不足 | 写入只读文件 |

### 异常继承层级（了解）

```
BaseException
├── SystemExit
├── KeyboardInterrupt
├── GeneratorExit
└── Exception              ← 所有常规异常的基类
    ├── ValueError
    ├── TypeError
    ├── KeyError
    ├── IndexError
    ├── FileNotFoundError  ← 继承自 OSError
    ├── ConnectionError    ← 继承自 OSError
    └── ...
```

> 📝 `except Exception` 可以捕获所有常规异常，但不会捕获 `KeyboardInterrupt`（Ctrl+C）和 `SystemExit`。

## 五、自定义异常

后端项目中，自定义异常可以让错误处理更有语义：

```python
# 定义业务异常
class AppError(Exception):
    """应用基础异常"""
    def __init__(self, message, code=500):
        self.message = message
        self.code = code
        super().__init__(self.message)

class NotFoundError(AppError):
    """资源不存在"""
    def __init__(self, resource, resource_id):
        self.resource = resource
        self.resource_id = resource_id
        super().__init__(
            message=f"{resource} (id={resource_id}) 不存在",
            code=404
        )

class AuthError(AppError):
    """认证失败"""
    def __init__(self, message="认证失败，请重新登录"):
        super().__init__(message=message, code=401)

class PermissionError(AppError):
    """权限不足"""
    def __init__(self, action):
        super().__init__(
            message=f"无权执行操作: {action}",
            code=403
        )
```

使用：

```python
def get_user(user_id):
    user = db.find(user_id)
    if not user:
        raise NotFoundError("用户", user_id)
    return user

try:
    user = get_user(999)
except NotFoundError as e:
    print(f"[{e.code}] {e.message}")   # [404] 用户 (id=999) 不存在
except AppError as e:
    print(f"[{e.code}] {e.message}")   # 捕获所有业务异常
```

## 六、后端异常处理最佳实践

### 1. 精确捕获，不要用裸 except

```python
# ❌ 太宽泛，隐藏了真正的 bug
try:
    process()
except:
    pass

# ✅ 精确捕获预期的异常
try:
    process()
except ConnectionError:
    retry()
except ValueError as e:
    log.error(f"数据格式错误: {e}")
```

### 2. 不要吞掉异常

```python
# ❌ 异常被吞了，出了 bug 根本找不到原因
try:
    save_to_db(data)
except Exception:
    pass

# ✅ 至少记录日志
try:
    save_to_db(data)
except Exception as e:
    logger.error(f"保存失败: {e}", exc_info=True)
    raise    # 或者重新抛出
```

### 3. 在合适的层级处理异常

```python
# 底层函数：抛出异常，不处理
def query_user(user_id):
    user = db.find(user_id)
    if not user:
        raise NotFoundError("用户", user_id)
    return user

# 中间层：转换异常或添加上下文
def get_user_profile(user_id):
    try:
        user = query_user(user_id)
        return build_profile(user)
    except NotFoundError:
        raise    # 原样抛出，让上层处理
    except DatabaseError as e:
        raise AppError(f"查询失败: {e}", code=500)

# 最外层（路由/控制器）：捕获并返回合适的响应
@app.get("/users/{user_id}")
def get_user(user_id: int):
    try:
        return get_user_profile(user_id)
    except NotFoundError as e:
        return {"error": e.message}, 404
    except AppError as e:
        return {"error": e.message}, e.code
```

## 七、常见坑

### 坑1：except 的顺序

```python
# ❌ 父类在前，子类的 except 永远不会执行
try:
    open("不存在.txt")
except Exception:          # 先匹配到这个
    print("通用异常")
except FileNotFoundError:  # 永远不会执行！
    print("文件不存在")

# ✅ 子类在前，父类在后
try:
    open("不存在.txt")
except FileNotFoundError:  # 先匹配具体的
    print("文件不存在")
except Exception:          # 兜底
    print("其他异常")
```

### 坑2：在 except 中丢失原始异常

```python
# ❌ 原始异常信息丢失
try:
    process()
except ValueError:
    raise RuntimeError("处理失败")    # 原始的 ValueError 信息丢了

# ✅ 用 from 保留异常链
try:
    process()
except ValueError as e:
    raise RuntimeError("处理失败") from e    # 保留原始异常
```

### 坑3：finally 中的 return 会吞掉异常

```python
# ❌ finally 中的 return 会覆盖 try 中的 return 和异常
def bad():
    try:
        raise ValueError("错误")
    finally:
        return "finally 的值"   # 异常被吞掉了！

print(bad())   # "finally 的值"（没有异常！）

# ✅ finally 中不要 return
```
