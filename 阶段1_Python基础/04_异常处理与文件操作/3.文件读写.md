# 文件读写

## 一、为什么需要文件操作？

后端开发中，文件操作是基础能力：

- 读取配置文件（`.env`、`.yaml`、`.json`）
- 写入日志文件
- 处理用户上传的文件（CSV、Excel、图片）
- 导入/导出数据
- 缓存数据到本地

---

## 二、open() 基础

### 基本用法

```python
# 打开 → 操作 → 关闭（必须配合 with 使用）
with open("data.txt", "r", encoding="utf-8") as f:
    content = f.read()
    print(content)
# 离开 with 块，文件自动关闭
```

### 打开模式

| 模式 | 说明 | 文件不存在时 |
|------|------|-------------|
| `"r"` | 只读（默认） | 报错 FileNotFoundError |
| `"w"` | 写入（覆盖） | 自动创建 |
| `"a"` | 追加 | 自动创建 |
| `"x"` | 独占创建 | 文件已存在则报错 |
| `"rb"` | 二进制读 | 报错 |
| `"wb"` | 二进制写 | 自动创建 |

> ⚠️ **`"w"` 模式会清空文件内容！** 追加内容用 `"a"`。

### encoding 参数

```python
# ⚠️ Windows 默认编码是 GBK，不指定 encoding 读中文文件会乱码
with open("中文.txt", "r", encoding="utf-8") as f:   # 必须指定 utf-8
    content = f.read()
```

> 📝 **规则**：打开文本文件时，**永远显式指定 `encoding="utf-8"`**。

## 三、读取文件的三种方式

```python
# 方式一：read() — 一次性读取全部内容
with open("data.txt", "r", encoding="utf-8") as f:
    content = f.read()      # 返回字符串
    print(content)

# 方式二：readline() — 逐行读取
with open("data.txt", "r", encoding="utf-8") as f:
    line = f.readline()     # 读一行
    while line:
        print(line.strip())  # strip() 去掉换行符
        line = f.readline()

# 方式三：readlines() — 读取所有行，返回列表
with open("data.txt", "r", encoding="utf-8") as f:
    lines = f.readlines()   # ['第一行\n', '第二行\n', ...]
    for line in lines:
        print(line.strip())

# 方式四（推荐）：直接遍历文件对象 — 逐行读取，最省内存
with open("data.txt", "r", encoding="utf-8") as f:
    for line in f:           # 文件对象本身就是迭代器
        print(line.strip())
```

### 选择指南

| 方式 | 内存 | 适用场景 |
|------|------|----------|
| `read()` | 全部加载 | 小文件（<10MB） |
| `readline()` | 一行 | 需要逐行控制 |
| `readlines()` | 全部加载 | 需要随机访问行 |
| `for line in f` | 一行 | 大文件（推荐） |

## 四、写入文件

```python
# 写入（覆盖）
with open("output.txt", "w", encoding="utf-8") as f:
    f.write("第一行\n")
    f.write("第二行\n")

# 追加
with open("output.txt", "a", encoding="utf-8") as f:
    f.write("追加的内容\n")

# 写入多行
lines = ["行1\n", "行2\n", "行3\n"]
with open("output.txt", "w", encoding="utf-8") as f:
    f.writelines(lines)    # 不会自动加换行符！
```

## 五、pathlib — 现代路径操作

`pathlib` 是 Python 3.4+ 引入的面向对象路径库，比 `os.path` 更优雅：

### 基本操作

```python
from pathlib import Path

# 创建路径对象
p = Path("data/users")

# 路径拼接（用 / 运算符）
config = Path("project") / "config" / "settings.json"
print(config)   # project/config/settings.json

# 获取路径信息
p = Path("/home/user/project/app/main.py")
print(p.name)       # main.py（文件名）
print(p.stem)       # main（不含扩展名）
print(p.suffix)     # .py（扩展名）
print(p.parent)     # /home/user/project/app（父目录）
print(p.parts)      # ('/', 'home', 'user', 'project', 'app', 'main.py')
```

### 文件/目录操作

```python
from pathlib import Path

p = Path("data")

# 判断
p.exists()       # 是否存在
p.is_file()      # 是否是文件
p.is_dir()       # 是否是目录

# 创建目录
p.mkdir(parents=True, exist_ok=True)   # parents=True 递归创建，exist_ok=True 已存在不报错

# 读写文件（pathlib 内置的简便方法）
config = Path("config.txt")
config.write_text("debug=true\n", encoding="utf-8")     # 写入
content = config.read_text(encoding="utf-8")              # 读取

# 遍历目录
for file in Path("src").glob("*.py"):       # 当前目录下所有 .py 文件
    print(file)

for file in Path("src").rglob("*.py"):      # 递归查找所有 .py 文件
    print(file)

# 删除文件
Path("temp.txt").unlink(missing_ok=True)    # missing_ok=True 文件不存在不报错
```

### os.path vs pathlib 对比

```python
import os
from pathlib import Path

# os.path 写法
path = os.path.join("data", "users", "config.json")
dirname = os.path.dirname(path)
basename = os.path.basename(path)
exists = os.path.exists(path)

# pathlib 写法（更简洁）
path = Path("data") / "users" / "config.json"
dirname = path.parent
basename = path.name
exists = path.exists()
```

> 📝 **推荐使用 pathlib**，代码更可读，IDE 自动补全更好。

## 六、后端实际场景

### 读取配置文件

```python
from pathlib import Path

# 读取 .env 风格的配置
def load_env(filepath: str) -> dict:
    config = {}
    env_path = Path(filepath)
    if not env_path.exists():
        return config
    for line in env_path.read_text(encoding="utf-8").splitlines():
        line = line.strip()
        if line and not line.startswith("#") and "=" in line:
            key, value = line.split("=", 1)
            config[key.strip()] = value.strip()
    return config

# config = load_env(".env")
```

### 写入日志文件

```python
from pathlib import Path
from datetime import datetime

def write_log(message: str, log_file: str = "app.log"):
    timestamp = datetime.now().strftime("%Y-%m-%d %H:%M:%S")
    log_line = f"[{timestamp}] {message}\n"
    with open(log_file, "a", encoding="utf-8") as f:
        f.write(log_line)

# write_log("用户 zhangsan 登录成功")
# write_log("订单 10086 创建失败: 库存不足")
```

### 处理上传文件

```python
from pathlib import Path

def save_upload(filename: str, content: bytes, upload_dir: str = "uploads"):
    """保存上传文件"""
    upload_path = Path(upload_dir)
    upload_path.mkdir(parents=True, exist_ok=True)

    file_path = upload_path / filename
    file_path.write_bytes(content)
    return str(file_path)
```

## 七、常见坑

### 坑1：忘记指定 encoding

```python
# ❌ Windows 默认 GBK，读 UTF-8 文件会乱码或报错
with open("中文.txt") as f:
    content = f.read()   # UnicodeDecodeError 或乱码

# ✅ 永远指定 encoding
with open("中文.txt", encoding="utf-8") as f:
    content = f.read()
```

### 坑2：用 "w" 模式误覆盖文件

```python
# ❌ "w" 会清空文件！
with open("important.log", "w") as f:
    f.write("新内容")   # 原来的日志全没了！

# ✅ 追加用 "a"
with open("important.log", "a") as f:
    f.write("新内容")
```

### 坑3：不用 with，忘记关闭文件

```python
# ❌ 文件没关闭，可能导致数据丢失或文件锁
f = open("data.txt", "w")
f.write("数据")
# 忘了 f.close()

# ✅ 用 with 自动关闭
with open("data.txt", "w") as f:
    f.write("数据")
```

### 坑4：路径分隔符在 Windows 和 Linux 不同

```python
# ❌ 硬编码路径分隔符
path = "data\\users\\config.json"    # 只在 Windows 有效

# ✅ 用 pathlib 自动适配
path = Path("data") / "users" / "config.json"   # 跨平台
```
