# os 与 sys

## 一、概念解释

`os` 和 `sys` 都是 Python 的标准库模块，用来和“运行环境”打交道：

- `os`：偏向操作系统能力，比如环境变量、目录路径、文件夹遍历、进程相关信息
- `sys`：偏向 Python 解释器本身，比如命令行参数、模块搜索路径、退出程序、标准输入输出

它们都属于“程序和外部世界交互”的基础工具。

---

## 二、为什么需要？

写 Python 不只是写算法，后端开发更常见的是：

- 读取环境变量中的配置
- 判断文件路径是否存在
- 拼接跨平台路径
- 接收命令行参数
- 让程序在异常情况下安全退出

这些都离不开 `os` 和 `sys`。

如果不会这两个模块，你很难写出真正能在本地、测试、生产环境里运行的程序。

---

## 三、实际场景

### 场景 1：读取环境变量

后端项目通常不会把密码、密钥直接写死在代码里，而是放在环境变量中：

```python
import os

db_host = os.getenv("DB_HOST", "localhost")
db_port = os.getenv("DB_PORT", "3306")
```

### 场景 2：处理命令行参数

脚本经常需要接收运行参数：

```python
import sys

print(sys.argv)
```

例如运行：

```bash
python backup.py prod
```

那么 `sys.argv` 中就会包含脚本名和 `prod`。

### 场景 3：构造路径

```python
import os

log_path = os.path.join("logs", "app.log")
```

这样写比手拼 `"logs/app.log"` 更安全，能兼容 Windows 和 Linux。

## 四、核心用法

### 1. 读取环境变量

```python
import os

api_key = os.getenv("API_KEY")
debug = os.getenv("DEBUG", "false")
```

- `os.getenv("KEY")`：取不到时返回 `None`
- `os.getenv("KEY", "默认值")`：取不到时返回默认值

### 2. 操作当前工作目录

```python
import os

print(os.getcwd())   # 当前工作目录
os.chdir("demo")     # 切换工作目录
```

### 3. 遍历目录

```python
import os

for name in os.listdir("."):
    print(name)
```

### 4. 常用 `os.path`

```python
import os

path = os.path.join("data", "users.json")

print(os.path.exists(path))   # 是否存在
print(os.path.isfile(path))   # 是否文件
print(os.path.isdir(path))    # 是否目录
print(os.path.abspath(path))  # 绝对路径
```

### 5. 命令行参数 `sys.argv`

```python
import sys

print(sys.argv)
```

规则：

- `sys.argv[0]`：脚本名
- `sys.argv[1:]`：用户传入的参数

### 6. Python 模块搜索路径

```python
import sys

for p in sys.path:
    print(p)
```

这决定了 Python 去哪里找 `import` 的模块。

### 7. 退出程序

```python
import sys

if not config_ok:
    print("配置错误")
    sys.exit(1)
```

- `sys.exit(0)`：正常退出
- `sys.exit(1)`：异常退出

## 五、常见坑

### 坑 1：把路径写死

```python
# ❌ 跨平台风险
path = "data\\users.json"

# ✅ 更稳妥
path = os.path.join("data", "users.json")
```

### 坑 2：直接取环境变量导致 `None`

```python
db_host = os.getenv("DB_HOST")
```

如果变量没配置，后面很可能报错。更稳妥的方式是给默认值，或者显式校验。

### 坑 3：误用 `sys.argv`

```python
import sys

mode = sys.argv[1]
```

如果用户没有传参数，这里会直接触发 `IndexError`。应该先判断长度：

```python
mode = sys.argv[1] if len(sys.argv) > 1 else "dev"
```

### 坑 4：混淆“当前工作目录”和“脚本所在目录”

`os.getcwd()` 返回的是“程序运行时所在目录”，不一定是脚本文件本身所在的位置。后端项目里这点很常见，路径问题经常就出在这里。

---

## 六、后端开发中的位置

在后端项目里，`os` 和 `sys` 最常见的用途是：

- 读取 `DB_HOST`、`REDIS_HOST`、`API_KEY` 等环境变量
- 根据命令行参数切换开发/测试/生产模式
- 找配置文件、日志文件、上传目录
- 在启动前检查环境并决定是否退出

一句话记忆：

**`os` 管操作系统环境，`sys` 管 Python 运行环境。**
