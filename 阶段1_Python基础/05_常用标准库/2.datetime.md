# datetime

## 一、概念解释

`datetime` 是 Python 处理日期和时间的标准库模块。

它主要解决几类问题：

- 获取当前时间
- 表示一个具体时间点
- 做时间加减
- 时间和字符串互相转换
- 处理后端中的日志时间、过期时间、定时任务时间

`datetime` 模块中最常用的几个类型是：

- `date`：只有日期
- `time`：只有时间
- `datetime`：日期 + 时间
- `timedelta`：时间差

---

## 二、为什么需要？

后端开发几乎离不开时间：

- 用户注册时间、订单创建时间
- JWT 过期时间
- 日志时间戳
- 定时任务执行时间
- 查询“最近 7 天数据”

如果不会 `datetime`，这些常见场景都会写得很脆弱。

## 三、实际场景

### 场景 1：记录创建时间

```python
from datetime import datetime

created_at = datetime.now()
```

### 场景 2：计算过期时间

```python
from datetime import datetime, timedelta

expire_at = datetime.now() + timedelta(hours=2)
```

### 场景 3：时间字符串解析

```python
from datetime import datetime

dt = datetime.strptime("2026-05-07 20:30:00", "%Y-%m-%d %H:%M:%S")
```

## 四、核心用法

### 1. 获取当前时间

```python
from datetime import datetime, date

print(datetime.now())   # 当前日期时间
print(date.today())     # 今天日期
```

### 2. 创建指定时间

```python
from datetime import datetime

dt = datetime(2026, 5, 7, 21, 30, 0)
print(dt)
```

### 3. 时间格式化为字符串

```python
from datetime import datetime

now = datetime.now()
text = now.strftime("%Y-%m-%d %H:%M:%S")
print(text)
```

常见格式符：

- `%Y`：四位年份
- `%m`：两位月份
- `%d`：两位日期
- `%H`：24 小时制小时
- `%M`：分钟
- `%S`：秒

### 4. 字符串解析为时间

```python
from datetime import datetime

dt = datetime.strptime("2026-05-07 21:30:00", "%Y-%m-%d %H:%M:%S")
```

### 5. 时间加减

```python
from datetime import datetime, timedelta

now = datetime.now()
after_7_days = now + timedelta(days=7)
before_30_minutes = now - timedelta(minutes=30)
```

### 6. 计算时间差

```python
from datetime import datetime

start = datetime(2026, 5, 1, 8, 0, 0)
end = datetime(2026, 5, 7, 10, 30, 0)
delta = end - start

print(delta.days)
print(delta.total_seconds())
```

## 五、常见坑

### 坑 1：把时间当字符串处理

```python
# ❌ 不要直接比较字符串表示“时间计算”
"2026-05-10" > "2026-05-2"
```

时间应该先转成 `datetime` 再比较或运算。

### 坑 2：`strftime` 和 `strptime` 混淆

- `strftime`：time -> string
- `strptime`：string -> time

记忆方式：

- `f` = format，格式化输出
- `p` = parse，解析输入

### 坑 3：忽略时区问题

学习阶段先掌握基础即可，但真实后端里时区非常重要。数据库、服务器、前端显示时区不一致时，很容易出现“同一条数据时间不一样”的问题。

### 坑 4：误用 `timedelta.seconds`

`timedelta.seconds` 只表示“去掉整天后的秒数”，不是总秒数。总秒数应该用：

```python
delta.total_seconds()
```

---

## 六、后端开发中的位置

`datetime` 在后端里最常见的用途：

- 数据库记录 `created_at` / `updated_at`
- JWT、验证码、缓存的过期时间
- 日志时间格式化
- 查询时间范围数据

一句话记忆：

**`datetime` 负责表示时间，`timedelta` 负责计算时间差。**
