# re 正则表达式

## 一、概念解释

`re` 是 Python 处理正则表达式的标准库模块。

正则表达式本质上是一套“字符串匹配规则”，适合做：

- 格式校验
- 文本搜索
- 文本提取
- 文本替换

它尤其适合处理“有规律但不固定”的字符串。

---

## 二、为什么需要？

后端开发里常见很多文本规则问题：

- 校验手机号、邮箱、用户名
- 从日志中提取 IP、时间、状态码
- 批量替换敏感信息
- 从文本中提取订单号、URL、版本号

如果只用字符串切片和 `split()`，一旦格式稍复杂就会很难维护，这时就轮到 `re` 上场。

---

## 三、实际场景

### 场景 1：校验用户名

```python
import re

pattern = r"^[a-zA-Z0-9_]{4,16}$"
print(bool(re.match(pattern, "user_01")))
```

### 场景 2：从日志提取状态码

```python
import re

log = "GET /api/users 200 15ms"
match = re.search(r"\\b(\\d{3})\\b", log)
if match:
    print(match.group(1))
```

### 场景 3：批量脱敏手机号

```python
import re

text = "联系方式：13812345678"
result = re.sub(r"(\\d{3})\\d{4}(\\d{4})", r"\\1****\\2", text)
```

## 四、核心用法

### 1. `re.match`

从字符串开头开始匹配：

```python
import re

print(re.match(r"abc", "abcdef"))
print(re.match(r"abc", "zabc"))   # None
```

### 2. `re.search`

在整个字符串中寻找第一个匹配：

```python
import re

print(re.search(r"abc", "zabcdef"))
```

### 3. `re.findall`

找出所有匹配结果：

```python
import re

nums = re.findall(r"\\d+", "订单123，金额456，库存789")
print(nums)
```

### 4. `re.sub`

按规则替换：

```python
import re

text = re.sub(r"\\d+", "#", "a1b22c333")
print(text)
```

### 5. 分组

```python
import re

text = "user:alice, age:20"
match = re.search(r"user:(\\w+), age:(\\d+)", text)
if match:
    print(match.group(1))   # alice
    print(match.group(2))   # 20
```

### 6. 常见元字符

| 表达式 | 含义 |
|--------|------|
| `.` | 任意字符（默认不含换行） |
| `\\d` | 数字 |
| `\\w` | 字母数字下划线 |
| `\\s` | 空白字符 |
| `+` | 前一个模式出现 1 次或多次 |
| `*` | 前一个模式出现 0 次或多次 |
| `?` | 前一个模式出现 0 次或 1 次 |
| `{n}` | 恰好 n 次 |
| `{m,n}` | m 到 n 次 |
| `^` | 字符串开头 |
| `$` | 字符串结尾 |

## 五、常见坑

### 坑 1：忘记原始字符串 `r""`

```python
# ❌ 容易把反斜杠转义错
pattern = "\\d+"

# ✅ 更推荐
pattern = r"\\d+"
```

### 坑 2：`match` 和 `search` 混淆

- `match`：只看开头
- `search`：全文搜索

### 坑 3：正则写得过于复杂

正则很强，但不是越复杂越好。能用简单字符串方法解决的问题，不一定非要上正则。正则适合“规则明显、格式模式化”的文本。

### 坑 4：以为正则能完全可靠校验所有格式

比如邮箱、URL、HTML，这些规则真实情况很复杂。学习阶段可以用简化规则，但工作里不要过度自信。

---

## 六、后端开发中的位置

`re` 在后端里最常见的用途：

- 参数格式校验
- 日志文本解析
- 数据清洗与脱敏
- 从文本中提取关键字段

一句话记忆：

**正则不是为了“看起来高级”，而是为了高效处理有模式的文本。**
