# collections

## 一、概念解释

`collections` 提供了一些比内置容器更顺手的数据结构。

在学习阶段最常用的是：

- `Counter`：计数器
- `defaultdict`：带默认值的字典

它们都能减少很多重复判断代码，让容器操作更简洁。

---

## 二、为什么需要？

后端里经常要做这些事情：

- 统计日志中每种状态码出现多少次
- 统计单词频率、用户访问次数
- 按分类聚合数据
- 构造“如果键不存在就自动初始化”的映射结构

这些需求如果只用普通 `dict`，代码会很啰嗦；用 `collections` 会自然很多。

---

## 三、实际场景

### 场景 1：访问次数统计

```python
from collections import Counter

visits = ["u1", "u2", "u1", "u3", "u1", "u2"]
counter = Counter(visits)
print(counter)
```

### 场景 2：按分类聚合

```python
from collections import defaultdict

grouped = defaultdict(list)
grouped["python"].append("装饰器")
grouped["python"].append("生成器")
```

---

## 四、核心用法

### 1. `Counter`

```python
from collections import Counter

counter = Counter("banana")
print(counter)
print(counter["a"])
print(counter.most_common(2))
```

适合：

- 词频统计
- 状态码统计
- 标签计数

### 2. `defaultdict`

```python
from collections import defaultdict

data = defaultdict(list)
data["backend"].append("FastAPI")
data["backend"].append("Redis")
```

常见默认工厂：

- `list`
- `dict`
- `set`
- `int`

### 3. 对比普通 `dict`

```python
# 普通 dict
grouped = {}
for key, value in items:
    if key not in grouped:
        grouped[key] = []
    grouped[key].append(value)

# defaultdict
from collections import defaultdict

grouped = defaultdict(list)
for key, value in items:
    grouped[key].append(value)
```

## 五、常见坑

### 坑 1：`defaultdict` 会“读一下就创建键”

```python
from collections import defaultdict

d = defaultdict(list)
print(d["missing"])   # 会自动创建 "missing"
```

这很方便，但也可能在不经意间污染数据。

### 坑 2：把 `Counter` 当排序结构

`Counter` 本质还是字典，不会自动按频次排序。需要看高频项时，用：

```python
counter.most_common(3)
```

### 坑 3：忘了 `defaultdict` 转回普通字典

有些序列化或输出场景下，最好转成普通 `dict`：

```python
normal_dict = dict(my_defaultdict)
```

---

## 六、后端开发中的位置

`collections` 在后端开发里非常常见：

- 用 `Counter` 做访问统计、错误统计、标签统计
- 用 `defaultdict` 做分组、聚合、反向索引

一句话记忆：

**`Counter` 负责“数一数”，`defaultdict` 负责“省掉不存在键的初始化”。**
