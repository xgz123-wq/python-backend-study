# functools

## 一、概念解释

`functools` 是 Python 中和“函数增强”有关的标准库模块。

学习阶段最重要的 3 个点是：

- `partial`：提前固定一部分参数
- `lru_cache`：给函数加缓存
- `wraps`：写装饰器时保留原函数元信息

---

## 二、为什么需要？

后端里函数经常需要：

- 复用一部分固定参数
- 缓存重复计算结果
- 写装饰器做日志、鉴权、计时、重试

`functools` 正好就是做这些事的。

---

## 三、实际场景

### 场景 1：固定日志前缀

```python
from functools import partial

def log(level, message):
    print(f"[{level}] {message}")

info = partial(log, "INFO")
info("服务启动成功")
```

### 场景 2：缓存重复查询

```python
from functools import lru_cache

@lru_cache(maxsize=128)
def fib(n):
    if n < 2:
        return n
    return fib(n - 1) + fib(n - 2)
```

### 场景 3：装饰器保留函数名

```python
from functools import wraps

def timer(func):
    @wraps(func)
    def wrapper(*args, **kwargs):
        return func(*args, **kwargs)
    return wrapper
```

## 四、核心用法

### 1. `partial`

```python
from functools import partial

def power(base, exponent):
    return base ** exponent

square = partial(power, exponent=2)
cube = partial(power, exponent=3)
```

作用：把“通用函数”变成“更具体的函数”。

### 2. `lru_cache`

```python
from functools import lru_cache

@lru_cache(maxsize=128)
def slow_add(a, b):
    print("真实计算中...")
    return a + b
```

同样参数再次调用时，会直接返回缓存结果。

### 3. `wraps`

```python
from functools import wraps

def my_decorator(func):
    @wraps(func)
    def wrapper(*args, **kwargs):
        return func(*args, **kwargs)
    return wrapper
```

如果不用 `@wraps(func)`，被装饰函数的：

- `__name__`
- `__doc__`

这些元信息会变成 `wrapper` 的，而不是原函数自己的。

## 五、常见坑

### 坑 1：缓存不适合有副作用的函数

```python
@lru_cache
def get_time():
    return datetime.now()
```

这类函数每次都应该变化，缓存后结果反而错了。

### 坑 2：缓存参数必须可哈希

```python
@lru_cache
def func(data):
    ...
```

如果传入的是 `list`、`dict` 这类不可哈希对象，会报错。

### 坑 3：忘记 `wraps`

自己写装饰器时，如果不用 `wraps`，调试、日志、文档工具都会更难受，因为函数元信息丢了。

### 坑 4：把 `partial` 理解成“立刻执行”

`partial` 返回的是一个“新的函数”，不是执行结果。

---

## 六、后端开发中的位置

`functools` 在后端里常见用途：

- `partial`：固定日志级别、固定配置参数
- `lru_cache`：缓存配置读取、热点纯函数计算
- `wraps`：写日志、鉴权、性能统计装饰器

一句话记忆：

**`partial` 固定参数，`lru_cache` 缓存结果，`wraps` 保护函数身份。**
