# 常用标准库 - Demo 学习指南

## 学习顺序

本章建议按编号顺序学习：先从系统交互开始，再进入时间处理、文本规则、数据结构优化，最后到函数增强工具。它们都是后端开发中“天天会碰到，但初学时容易低估价值”的基础模块。

---

### Demo 1: os 与 sys（程序和运行环境的接口）
**文件**: `1_os_sys.py`

演示环境变量读取、命令行参数、当前工作目录、路径拼接、目录遍历，以及启动脚本时如何做参数和环境检查。这是最贴近真实后端启动流程的一组基础能力。

**运行方式**: `python 1_os_sys.py`
**前置依赖**: 无额外依赖
**测试方式**: 观察控制台输出，也可尝试 `python 1_os_sys.py prod`

### Demo 2: datetime（后端里的时间计算器）

## 核心要点

1. `os` 和 `sys` 负责把程序接到外部环境上，是脚本和后端服务启动时的基础能力。
2. `datetime` 解决时间表示和计算问题，后端里经常用于日志、过期时间和数据筛选。
3. `re` 擅长处理“有规则的文本”，适合校验、提取和脱敏。
4. `collections` 让统计和分组代码更短、更自然。
5. `functools` 是函数增强工具箱，后面学装饰器、缓存、框架源码时都会反复见到。

## 常见问题

| 问题 | 解决方案 |
|------|----------|
| `os.getenv()` 取到 `None` | 给默认值或在启动时显式校验 |
| `sys.argv[1]` 报 `IndexError` | 先判断 `len(sys.argv)` |
| `strftime` / `strptime` 老是混 | 记住 `f=format`，`p=parse` |
| 正则写得看不懂 | 从小规则拆起，先验证再组合 |
| `defaultdict` 莫名多了键 | 读取不存在键时会自动创建 |
| `lru_cache` 报不可哈希错误 | 不要传 `list`、`dict` 这类可变对象 |
| 自己写的装饰器函数名变成 `wrapper` | 使用 `@wraps(func)` |
