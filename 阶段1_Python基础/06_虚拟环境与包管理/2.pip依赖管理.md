# pip 依赖管理

## 一、概念解释

`pip` 是 Python 官方的包管理工具，用来安装、更新、卸载第三方包，以及管理项目的依赖列表。

大多数情况下，你通过 `pip install 包名` 从 [PyPI](https://pypi.org)（Python 包索引）下载安装。

---

## 二、为什么需要依赖管理？

一个项目依赖十几个甚至几十个第三方包，你需要一种方式：
1. **记录**：告诉别人你的项目需要哪些包、什么版本
2. **复现**：让其他人能一命令把环境装好
3. **固定**：防止包升级导致代码跑不了

这就是 `requirements.txt` 的价值。

---

## 三、实际场景

- 新同事接手项目：`pip install -r requirements.txt` 一步到位
- CI/CD 每次构建：从 requirements.txt 装干净依赖
- Docker 镜像构建：只装必要依赖，减小镜像体积
- 部署到服务器：固定版本号，避免环境差异

## 四、核心用法

### 4.1 安装包

```bash
pip install fastapi          # 安装最新版
pip install fastapi==0.110.0  # 安装指定版本
pip install "fastapi>=0.100"  # 安装满足版本约束的包
```

### 4.2 卸载包

```bash
pip uninstall fastapi
```

### 4.3 查看已安装的包

```bash
pip list              # 列出所有已安装的包和版本
pip show fastapi      # 查看某个包的详细信息（版本、依赖、位置）
```

### 4.4 生成 requirements.txt

```bash
# 导出当前环境所有包（含传递依赖）
pip freeze > requirements.txt

# 查看内容
cat requirements.txt
```

`pip freeze` 的输出格式：
```
annotated-types==0.6.0
anyio==4.3.0
fastapi==0.110.0
pydantic==2.6.4
...
```

### 4.5 从 requirements.txt 安装

```bash
pip install -r requirements.txt
```

### 4.6 升级包

```bash
pip install --upgrade fastapi
pip install -U fastapi          # 简写
```

### 4.7 换国内镜像源（加速下载）

```bash
# 临时使用清华源
pip install fastapi -i https://pypi.tuna.tsinghua.edu.cn/simple

# 永久配置（推荐）
pip config set global.index-url https://pypi.tuna.tsinghua.edu.cn/simple
```

## 五、requirements.txt 的两种写法

### 精确锁定版本（推荐用于部署）

```
fastapi==0.110.0
uvicorn==0.29.0
pydantic==2.6.4
sqlalchemy==2.0.28
```

优点：完全可复现，不受包升级影响  
缺点：需要定期手动更新

### 约束版本范围（用于库开发）

```
fastapi>=0.100.0,<1.0.0
uvicorn>=0.20.0
```

优点：允许小版本升级，获得 bug 修复  
缺点：不同时间安装可能得到不同版本

**项目开发推荐：精确锁定版本**，部署不出意外。

## 六、常见坑

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| `pip: command not found` | 没激活虚拟环境 / 路径未配置 | 激活虚拟环境后再 pip |
| 下载很慢 | 默认从国外源下载 | 配置国内镜像源 |
| `pip freeze` 包太多 | 包含了所有传递依赖 | 可以只手写直接依赖；或用 `pip-tools` 分离 |
| 版本冲突 `ResolutionImpossible` | 两个包要求同一包的不兼容版本 | 逐一排查冲突，升降级其中一个包 |
| 安装后 `import` 报错 | 装到了全局而非虚拟环境 | 检查是否激活了虚拟环境 |
