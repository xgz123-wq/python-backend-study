# 了解 Poetry / PDM

## 一、概念解释

Poetry 和 PDM 是比 `pip + requirements.txt` 更现代化的 **Python 依赖管理工具**，它们能同时处理：

- 依赖安装与卸载
- 依赖版本锁定（lock file）
- 虚拟环境管理
- 项目打包与发布

类比：`pip + venv + requirements.txt` 是三个分散的工具，Poetry/PDM 是把它们合成一个统一工具。

---

## 二、为什么需要了解？

`pip + requirements.txt` 已经够用，但它有两个痛点：

1. **没有 lock file**：`requirements.txt` 写 `fastapi>=0.100` 时，不同时间安装会得到不同版本
2. **手动维护**：`pip freeze > requirements.txt` 混入了所有传递依赖，难以分辨哪些是直接依赖

Poetry/PDM 的方案：
- `pyproject.toml`：记录**直接依赖**（类似 npm 的 `package.json`）
- `poetry.lock` / `pdm.lock`：**精确锁定**所有包的完整版本树（类似 npm 的 `package-lock.json`）

---

## 三、Poetry vs PDM vs pip

| 特性 | pip + venv | Poetry | PDM |
|------|-----------|--------|-----|
| 虚拟环境管理 | 手动 | 自动 | 自动 |
| 依赖锁定 | 弱（freeze）| ✅ lock file | ✅ lock file |
| 直接/传递依赖分离 | ❌ | ✅ | ✅ |
| 项目打包 | 需额外工具 | ✅ 内置 | ✅ 内置 |
| 学习成本 | 低 | 中 | 中 |
| 适合场景 | 快速脚本、简单项目 | 应用/库开发 | 应用/库开发 |

## 四、Poetry 核心用法（了解即可）

### 4.1 安装

```bash
pip install poetry
```

### 4.2 创建新项目

```bash
poetry new myproject
```

或在现有项目初始化：

```bash
poetry init
```

生成 `pyproject.toml`：

```toml
[tool.poetry]
name = "myproject"
version = "0.1.0"
description = ""

[tool.poetry.dependencies]
python = "^3.11"
fastapi = "^0.110.0"
sqlalchemy = "^2.0"

[tool.poetry.group.dev.dependencies]
pytest = "^8.0"
ruff = "^0.3"
```

### 4.3 常用命令

```bash
poetry install          # 安装所有依赖（根据 lock file）
poetry add fastapi      # 添加依赖
poetry add pytest --group dev  # 添加开发依赖
poetry remove fastapi   # 移除依赖
poetry update           # 更新依赖到最新兼容版本
poetry shell            # 进入虚拟环境
poetry run python main.py  # 在虚拟环境内运行命令
```

## 五、PDM 核心用法（了解即可）

PDM 更符合 Python 官方标准（PEP 517/518），与工具链兼容性更好。

```bash
pip install pdm

pdm init                # 初始化项目
pdm add fastapi         # 添加依赖
pdm add -d pytest       # 添加开发依赖
pdm install             # 安装所有依赖
pdm run python main.py  # 运行命令
```

## 六、现阶段建议

| 阶段 | 推荐工具 |
|------|---------|
| 学习阶段（现在）| `pip + venv + requirements.txt`，简单直接 |
| 项目开发阶段 | 考虑引入 Poetry 或 PDM |
| 团队协作 / 开源项目 | 强烈推荐 Poetry / PDM，lock file 避免"在我机器上能跑" |

> 学习重点是理解**为什么需要依赖管理工具**，能看懂 `pyproject.toml` 和 lock file，知道有这些工具存在即可。进入项目实战时再深入使用。
