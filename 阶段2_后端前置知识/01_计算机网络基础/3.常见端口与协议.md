# 常见端口与协议

> 端口 = 服务器上的"门牌号"。IP 定位到哪台机器，端口定位到哪个服务。

---

## 一、概念解释

### 什么是端口？

一台服务器上可以同时运行多个服务（Web 服务、数据库、缓存等），**端口号**用来区分这些服务：

```
IP 地址：确定哪台机器      （相当于大楼地址）
端口号：确定哪个服务        （相当于房间号）
```

完整的网络地址 = `IP:端口`，例如 `192.168.1.100:8000`

### 端口号范围

| 范围 | 名称 | 说明 |
|------|------|------|
| 0 ~ 1023 | 知名端口 | 分配给标准服务（HTTP=80, HTTPS=443），需要管理员权限 |
| 1024 ~ 49151 | 注册端口 | 分配给常用应用（MySQL=3306, Redis=6379） |
| 49152 ~ 65535 | 动态/私有端口 | 操作系统临时分配给客户端连接 |

## 二、为什么需要了解端口？

| 场景 | 为什么需要 |
|------|-----------|
| 启动服务 | `uvicorn app:main --port 8000`，必须知道端口概念 |
| 排查冲突 | "端口已被占用"是最常见的启动失败原因之一 |
| 防火墙配置 | 部署时需要开放特定端口 |
| 服务间通信 | 后端连数据库、连 Redis，都需要指定端口 |

## 三、后端必知端口清单

### 核心端口（必须记住）

| 端口 | 协议/服务 | 说明 | 后端使用场景 |
|------|----------|------|-------------|
| **80** | HTTP | Web 服务默认端口 | Nginx 监听 |

## 四、协议与端口的对应关系

### 应用层协议一览

```
应用层协议          传输层协议      默认端口      后端相关度
─────────────────────────────────────────────────────
HTTP               TCP             80           ⭐⭐⭐
HTTPS              TCP             443          ⭐⭐⭐
DNS                UDP（主）       53           ⭐⭐
MySQL 协议         TCP             3306         ⭐⭐⭐
Redis 协议（RESP） TCP             6379         ⭐⭐⭐
SSH                TCP             22           ⭐⭐⭐
FTP                TCP             21           ⭐
SMTP               TCP             25/465/587   ⭐⭐
WebSocket          TCP             80/443       ⭐⭐⭐
```

### 关键理解：协议分层

```python
# 你写的代码（应用层）
response = requests.get("http://api.example.com/users")

# 底层实际发生的事情：
# HTTP 协议（应用层）→ 构造请求报文
# TCP 协议（传输层）→ 建立连接、可靠传输
# IP 协议（网络层）→ 寻址路由到目标服务器
# 以太网（链路层）→ 物理传输
```

## 五、实际场景

### 场景 1：端口冲突排查

```bash
# 查看某端口被哪个进程占用

# Windows
netstat -ano | findstr :8000

# Linux/Mac
lsof -i :8000
# 或
ss -tlnp | grep 8000

# 解决方案：
# 1. 关闭占用端口的进程
# 2. 换一个端口启动服务：uvicorn app:main --port 8001
```

### 场景 2：后端服务常见配置

```python
# FastAPI 开发服务器
# uvicorn main:app --host 0.0.0.0 --port 8000

# 数据库连接
DATABASE_URL = "mysql://user:password@localhost:3306/mydb"

# Redis 连接
REDIS_URL = "redis://localhost:6379/0"

# 生产部署（Nginx 反向代理）
# 外部访问：443 → Nginx → 内部转发：8000 → FastAPI
```

### 场景 3：Docker 端口映射

```yaml
# docker-compose.yml
services:
  mysql:
    image: mysql:8.0
    ports:
      - "3306:3306"    # 宿主机端口:容器端口
  redis:
    image: redis:7
    ports:
      - "6379:6379"
  api:
    build: .
    ports:
      - "8000:8000"
```

## 六、常见误区

| 误区 | 纠正 |
|------|------|
| "一个端口只能一个进程" | 正确，但 SO_REUSEPORT 可以让多个进程监听同一端口（负载均衡） |
| "修改端口就安全了" | 改端口只是"隐蔽"，不是"安全"，真正的安全靠认证和防火墙 |
| "0.0.0.0 是无效地址" | 0.0.0.0 表示监听所有网络接口，开发常用。127.0.0.1 只监听本地 |
| "端口号可以随便选" | 1024 以下需要管理员权限，且不要和已知服务冲突 |

### 0.0.0.0 vs 127.0.0.1

```
127.0.0.1（localhost）
→ 只接受来自本机的连接
→ 其他机器无法访问
→ 适合：纯本地开发

0.0.0.0
→ 接受来自所有网络接口的连接
→ 局域网内其他设备可以访问
→ 适合：Docker 内、需要外部访问时
```

## 七、面试高频题

**Q：HTTP 和 HTTPS 的默认端口？**
A：HTTP 是 80，HTTPS 是 443。

**Q：MySQL、Redis、PostgreSQL 的默认端口？**
A：MySQL 3306，Redis 6379，PostgreSQL 5432。

**Q：0.0.0.0 和 127.0.0.1 的区别？**
A：127.0.0.1 只监听回环地址（本机访问），0.0.0.0 监听所有网络接口（允许外部访问）。

**Q：为什么 1024 以下的端口需要管理员权限？**
A：这些是"知名端口"，由 IANA 分配给标准服务，限制权限防止普通用户冒充系统服务。
