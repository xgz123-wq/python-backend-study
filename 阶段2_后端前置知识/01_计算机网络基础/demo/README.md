## Demo 详细说明

### Demo 1: TCP/IP 协议演示（体验可靠传输 vs 快速传输）

**文件**: `1_tcp_ip.py`

使用 `socket` 模块分别实现 TCP 和 UDP 通信，直观对比两者的区别。TCP 需要先建立连接（三次握手），UDP 直接发送。还演示了 TCP 连接的五元组概念。

**运行方式**: `python 1_tcp_ip.py`
**前置依赖**: 无额外依赖
**测试方式**: 观察控制台输出，对比 TCP（有连接）和 UDP（无连接）的通信流程

### Demo 2: DNS 解析演示（理解域名到 IP 的翻译过程）

**文件**: `2_dns.py`

用 Python 进行实际的 DNS 查询：域名转 IP、一个域名多个 IP、DNS 缓存对查询速度的影响、反向 DNS 查询。还模拟了完整的 DNS 解析流程步骤。

**运行方式**: `python 2_dns.py`
**前置依赖**: 无额外依赖（需要网络连接）
**测试方式**: 观察控制台输出，注意 DNS 缓存对查询耗时的影响

### Demo 3: 常见端口与协议演示（端口实操）

**文件**: `3_ports_protocols.py`

扫描本机常见服务端口的开放状态、演示端口冲突错误、对比 `0.0.0.0` 和 `127.0.0.1` 的行为差异、提供端口连通性测试工具。

**运行方式**: `python 3_ports_protocols.py`
**前置依赖**: 无额外依赖
**测试方式**: 观察控制台输出，如果安装了 MySQL/Redis 会看到对应端口显示「开放」

## 核心要点

1. **TCP 面向连接**：通信前三次握手，通信后四次挥手，保证数据可靠有序传输
2. **UDP 无连接**：直接发送，速度快但不保证送达，适合 DNS/视频/游戏
3. **DNS 是域名到 IP 的翻译系统**：多级缓存加速，解析流程逐级向上查询
4. **端口区分服务**：IP 定位机器，端口定位服务。后端必记：80/443/3306/6379/8000
5. **0.0.0.0 vs 127.0.0.1**：前者监听所有网络接口，后者只监听本机回环

## 常见问题

| 问题 | 解决方案 |
|------|---------|
| `ConnectionRefusedError` | 目标端口没有服务在监听，检查服务是否启动 |
| `Address already in use` | 端口被占用，用 `netstat -ano \| findstr :端口号` 查找占用进程 |
| DNS 查询超时 | 检查网络连接，尝试更换 DNS 服务器（如 8.8.8.8） |
| Demo 1 运行卡住 | socket 阻塞等待连接，重新运行即可 |
