# HTTP 请求与响应结构

> HTTP 是浏览器、App、后端服务之间最常见的“对话格式”。理解请求和响应结构，是学习 API 的入口。

---

## 一、概念解释

### 什么是 HTTP？

**HTTP**（HyperText Transfer Protocol，超文本传输协议）是应用层协议，用来规定客户端和服务器如何交换数据。

```text
客户端（浏览器 / Postman / Python 程序）
        │  HTTP Request
        ▼
服务器（Nginx / FastAPI / Django）
        │  HTTP Response
        ▼
客户端接收结果
```

HTTP 本身不负责“把数据可靠送到对方”，可靠传输由底层 TCP 负责。HTTP 负责定义：

- 请求长什么样
- 响应长什么样
- 方法、状态码、Header、Body 怎么表达

---

## 二、为什么需要 HTTP？

| 场景 | 为什么需要 |
|------|------------|
| 写 RESTful API | API 本质就是接收 HTTP 请求并返回 HTTP 响应 |
| 调试接口 | 必须看懂请求方法、URL、Header、Body、状态码 |
| 前后端联调 | 前端发送 JSON，后端解析 JSON，靠 HTTP 约定格式 |
| 排查线上问题 | 通过状态码和响应体判断错误来自客户端还是服务端 |

---

## 三、HTTP 请求结构

一个 HTTP 请求通常由 4 部分组成：

```http
POST /api/users?page=1 HTTP/1.1
Host: example.com
Content-Type: application/json
Authorization: Bearer token

{"name": "Alice", "age": 20}
```

| 部分 | 示例 | 说明 |
|------|------|------|
| 请求行 | `POST /api/users?page=1 HTTP/1.1` | 方法 + 路径 + 协议版本 |
| 请求头 | `Content-Type: application/json` | 元数据，例如格式、认证、来源 |
| 空行 | 空白行 | 分隔 Header 和 Body |
| 请求体 | `{"name": "Alice"}` | 提交给服务器的数据 |

### 请求行拆解

```http
POST /api/users?page=1 HTTP/1.1
```

- `POST`：请求方法，表示创建资源
- `/api/users?page=1`：请求路径和查询参数
- `HTTP/1.1`：协议版本

### URL 的常见组成

```text
https://api.example.com:443/users/1?verbose=true#profile
│       │               │   │        │            │
协议    域名            端口 路径     查询参数       片段
```

后端通常能收到：路径、查询参数、Header、Body。`#profile` 片段只在浏览器端使用，不会发送给服务器。

## 四、HTTP 响应结构

一个 HTTP 响应也通常由 4 部分组成：

```http
HTTP/1.1 200 OK
Content-Type: application/json
Content-Length: 35

{"id": 1, "name": "Alice", "age": 20}
```

| 部分 | 示例 | 说明 |
|------|------|------|
| 状态行 | `HTTP/1.1 200 OK` | 协议版本 + 状态码 + 状态文本 |
| 响应头 | `Content-Type: application/json` | 响应元数据 |
| 空行 | 空白行 | 分隔 Header 和 Body |
| 响应体 | `{"id": 1}` | 返回给客户端的数据 |

### 状态行拆解

```http
HTTP/1.1 200 OK
```

- `HTTP/1.1`：协议版本
- `200`：状态码，表示成功
- `OK`：状态文本，给人看的说明

## 五、实际场景

### 场景 1：前端登录请求

```http
POST /api/login HTTP/1.1
Host: api.example.com
Content-Type: application/json

{"username": "alice", "password": "******"}
```

服务器响应：

```http
HTTP/1.1 200 OK
Content-Type: application/json

{"access_token": "...", "token_type": "bearer"}
```

### 场景 2：查询用户列表

```http
GET /api/users?page=1&page_size=20 HTTP/1.1
Host: api.example.com
Accept: application/json
```

服务器响应：

```json
{
    "items": [
        {"id": 1, "name": "Alice"}
    ],
    "total": 1
}
```

## 六、常见误区

| 误区 | 纠正 |
|------|------|
| “HTTP 是连接” | HTTP 是应用层协议，底层连接通常由 TCP 建立 |
| “GET 不能带参数” | GET 常通过 URL 查询参数传参，只是不建议带请求体 |
| “响应成功一定是业务成功” | HTTP `200` 只表示请求被正常处理，业务是否成功还要看响应体中的业务 code |
| “URL 的 `#` 会传给后端” | 不会，片段标识符只在客户端使用 |

## 七、面试高频题

**Q：HTTP 请求由哪几部分组成？**  
A：请求行、请求头、空行、请求体。

**Q：HTTP 响应由哪几部分组成？**  
A：状态行、响应头、空行、响应体。

**Q：HTTP 和 TCP 的关系？**  
A：HTTP 是应用层协议，规定数据格式；TCP 是传输层协议，负责可靠传输。HTTP 通常运行在 TCP 之上。
