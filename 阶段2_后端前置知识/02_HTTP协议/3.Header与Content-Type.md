# Header 与 Content-Type

> Header 是 HTTP 的“说明书”，Content-Type 告诉对方 Body 里装的是什么格式。

---

## 一、概念解释

### Header 是什么？

HTTP Header 是请求或响应中的元数据，描述这次通信的附加信息。

```http
Content-Type: application/json
Accept: application/json
User-Agent: Mozilla/5.0
Authorization: Bearer token
```

Header 不直接表示业务数据，但会影响服务器如何解析请求、客户端如何处理响应。

### Content-Type 是什么？

`Content-Type` 表示请求体或响应体的数据格式。

```http
Content-Type: application/json

{"name": "Alice"}
```

服务器看到 `application/json`，才知道应该按 JSON 解析 Body。

---

## 二、为什么需要 Header？

| 场景 | Header 的作用 |
|------|---------------|
| 提交 JSON | `Content-Type: application/json` 告诉后端按 JSON 解析 |
| 接收 JSON | `Accept: application/json` 告诉服务端客户端希望得到 JSON |
| 用户认证 | `Authorization: Bearer <token>` 携带访问令牌 |
| 浏览器缓存 | `Cache-Control` / `ETag` 控制缓存行为 |
| 跨域请求 | `Origin` / `Access-Control-*` 参与 CORS 判断 |

---

## 三、常见请求 Header

| Header | 作用 | 示例 |
|--------|------|------|
| `Host` | 目标主机 | `api.example.com` |
| `User-Agent` | 客户端信息 | `Mozilla/5.0` |
| `Accept` | 客户端可接受的响应类型 | `application/json` |
| `Content-Type` | 请求体格式 | `application/json` |
| `Authorization` | 认证信息 | `Bearer eyJ...` |
| `Cookie` | 浏览器携带的 Cookie | `session_id=abc` |
| `Origin` | 跨域请求来源 | `https://frontend.example.com` |

---

## 四、常见响应 Header

| Header | 作用 | 示例 |
|--------|------|------|
| `Content-Type` | 响应体格式 | `application/json; charset=utf-8` |
| `Content-Length` | 响应体字节长度 | `128` |
| `Set-Cookie` | 让浏览器保存 Cookie | `session_id=abc; HttpOnly` |
| `Cache-Control` | 缓存策略 | `no-store` |
| `Location` | 重定向地址 | `/login` |
| `Access-Control-Allow-Origin` | 允许跨域来源 | `https://frontend.example.com` |

---

## 五、常见 Content-Type

| Content-Type | 含义 | 常见场景 |
|--------------|------|----------|
| `application/json` | JSON 数据 | RESTful API 最常用 |
| `application/x-www-form-urlencoded` | 表单键值对 | 传统 HTML 表单 |
| `multipart/form-data` | 多部分表单 | 文件上传 |
| `text/plain` | 纯文本 | 文本响应 |
| `text/html` | HTML 页面 | 浏览器页面 |
| `application/octet-stream` | 二进制流 | 文件下载 |

### JSON 请求示例

```http
POST /users HTTP/1.1
Content-Type: application/json

{"name": "Alice", "age": 20}
```

### 表单请求示例

```http
POST /login HTTP/1.1
Content-Type: application/x-www-form-urlencoded

username=alice&password=123456
```

### 文件上传示例

```http
POST /upload HTTP/1.1
Content-Type: multipart/form-data; boundary=----abc

------abc
Content-Disposition: form-data; name="file"; filename="a.txt"
Content-Type: text/plain

hello
------abc--
```

## 六、实际场景

### 场景：FastAPI 接收 JSON

如果前端发送 JSON，但 Header 写错：

```http
Content-Type: text/plain

{"name": "Alice"}
```

后端可能无法按 JSON 正确解析，导致参数校验失败。

正确写法：

```http
Content-Type: application/json

{"name": "Alice"}
```

## 七、常见误区

| 误区 | 纠正 |
|------|------|
| “Header 不重要” | Header 决定 Body 怎么解析、是否认证、是否跨域、是否缓存 |
| “Accept 和 Content-Type 一样” | `Accept` 表示想接收什么；`Content-Type` 表示自己发送的 Body 是什么 |
| “JSON 放 Body 就行” | 还必须设置 `Content-Type: application/json` |
| “Token 可以放 URL” | 不推荐，URL 容易进入日志；应放 `Authorization` Header |

## 八、面试高频题

**Q：Content-Type 和 Accept 的区别？**  
A：`Content-Type` 描述当前请求体或响应体的格式；`Accept` 描述客户端希望接收的响应格式。

**Q：文件上传一般用什么 Content-Type？**  
A：`multipart/form-data`。

**Q：JWT Token 通常放在哪里？**  
A：放在 `Authorization: Bearer <token>` 请求头中。
