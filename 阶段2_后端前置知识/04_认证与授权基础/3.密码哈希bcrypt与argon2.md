# 密码哈希：bcrypt 与 argon2

> 永远不要明文存储密码。密码哈希是用户数据安全的最后防线。

---

## 一、概念解释

### 什么是密码哈希？

**哈希**（Hash）是一种单向函数：输入任意数据，输出固定长度的摘要，且无法从摘要反推出原始数据。

```
明文密码: "123456"
         ↓ bcrypt
哈希值:  "$2b$12$EixZaYVK1fsbw1ZfbX3OXePaWxn96p36WQoeG6Lruj3vjPGga31lW"

哈希特性：
✓ 不可逆：无法从哈希值得到 "123456"
✓ 加盐：每次哈希结果都不同（即使输入相同）
✓ 验证：把明文再次哈希，与存储哈希比对
```

### 为什么普通 MD5/SHA1 不够？

**彩虹表攻击**：攻击者预先计算大量常见密码的哈希值，建成一张查询表：

```
MD5("123456")    = "e10adc3949ba59abbe56e057f20f883e"
MD5("password")  = "5f4dcc3b5aa765d61d8327deb882cf99"
...
（预先计算几百亿条记录）

数据库泄露后，攻击者拿到 MD5 哈希，查表秒查出原始密码！
```

**bcrypt/argon2 如何防御**：
1. **随机盐（Salt）**：每次哈希前生成随机盐，加到密码前再哈希
   - 即使两个用户密码相同，哈希值也不同
   - 使彩虹表完全失效（要预计算就得针对每个盐预计算，计算量爆炸）
2. **计算成本（Cost Factor）**：故意设计得慢，让暴力破解不可行
   - 登录时验证一次：几百毫秒，用户可接受
   - 攻击者每秒只能尝试几十个密码，无法暴力破解

## 二、bcrypt 详解

### 1. bcrypt 的哈希格式

```
$2b$12$EixZaYVK1fsbw1ZfbX3OXePaWxn96p36WQoeG6Lruj3vjPGga31lW
│  │  │                    │
│  │  └─ 22位盐值           └─ 31位哈希值
│  └─ cost factor（12 表示 2^12 = 4096 轮迭代）
└─ 版本号（2b 是当前标准版本）
```

### 2. cost factor（工作因子）

| cost | 哈希时间（现代 CPU） | 说明 |
|------|-------------------|------|
| 10 | ~100ms | 最低推荐值，生产环境可接受 |
| 12 | ~300ms | 推荐值，安全与性能平衡 |
| 14 | ~1s | 安全要求高的场景 |
| 16 | ~3s | 极高安全要求（如密钥派生） |

### 3. Python 中使用 bcrypt

```python
import bcrypt

# 生成哈希（注册/修改密码时）
def hash_password(plain_password: str) -> str:
    """
    hash_password("123456") 
    → "$2b$12$..."（每次都不同，因为盐是随机的）
    """
    salt = bcrypt.gensalt(rounds=12)           # 生成随机盐，rounds=cost factor
    hashed = bcrypt.hashpw(plain_password.encode("utf-8"), salt)
    return hashed.decode("utf-8")             # bytes → str，方便存数据库

# 验证密码（登录时）
def verify_password(plain_password: str, hashed_password: str) -> bool:
    """
    自动从哈希值中提取盐，对明文同样处理后比对
    """
    return bcrypt.checkpw(
        plain_password.encode("utf-8"),
        hashed_password.encode("utf-8")
    )

# 使用示例
hashed = hash_password("my_secret_password")
print(hashed)  # $2b$12$...（每次不同！）

print(verify_password("my_secret_password", hashed))   # True
print(verify_password("wrong_password", hashed))        # False
```

## 三、argon2 详解

### 1. argon2 vs bcrypt

argon2 是 2015 年密码哈希竞赛的冠军算法，专门为抵抗 GPU/ASIC 暴力破解而设计。

| 维度 | bcrypt | argon2 |
|------|--------|--------|
| 设计年代 | 1999 年 | 2015 年 |
| 内存占用 | 固定（很小） | 可配置（推荐高内存） |
| 抗 GPU 攻击 | 一般 | 强（高内存使 GPU 并行无效） |
| 参数灵活性 | 只有 cost | time、memory、parallelism |
| 推荐场景 | 大多数 Web 应用 | 高安全要求、新项目 |
| 国际认可 | 广泛使用 | NIST 推荐 |

### 2. argon2 的三种变体

| 变体 | 说明 |
|------|------|
| argon2d | 抗 GPU，但不抗侧信道攻击（不用于密码哈希） |
| argon2i | 抗侧信道攻击（原密码哈希推荐） |
| **argon2id** | 混合 d 和 i，**当前推荐** |

### 3. Python 中使用 argon2

```python
from argon2 import PasswordHasher
from argon2.exceptions import VerifyMismatchError

# 创建 PasswordHasher（可自定义参数）
ph = PasswordHasher(
    time_cost=2,        # 迭代次数（时间成本）
    memory_cost=65536,  # 内存占用：64MB（抗 GPU 攻击的关键）
    parallelism=2,      # 并行度
    hash_len=32,        # 哈希长度
    salt_len=16,        # 盐长度
)

# 生成哈希
def hash_password(plain_password: str) -> str:
    return ph.hash(plain_password)
    # → "$argon2id$v=19$m=65536,t=2,p=2$..."

# 验证密码
def verify_password(plain_password: str, hashed_password: str) -> bool:
    try:
        return ph.verify(hashed_password, plain_password)
    except VerifyMismatchError:
        return False

# 检查是否需要重新哈希（参数升级后）
def needs_rehash(hashed_password: str) -> bool:
    return ph.check_needs_rehash(hashed_password)

# 使用示例
hashed = hash_password("my_secret_password")
print(verify_password("my_secret_password", hashed))   # True
print(verify_password("wrong", hashed))                 # False
```

## 四、在 FastAPI/后端中的完整实践

### 推荐方案：passlib 统一接口

`passlib` 是 Python 最流行的密码哈希库，统一封装了 bcrypt、argon2 等多种算法：

```python
from passlib.context import CryptContext

# 推荐配置：bcrypt 作为主算法，deprecated="auto" 自动处理旧哈希
pwd_context = CryptContext(schemes=["bcrypt"], deprecated="auto")

def hash_password(plain_password: str) -> str:
    """注册/修改密码时调用"""
    return pwd_context.hash(plain_password)

def verify_password(plain_password: str, hashed_password: str) -> bool:
    """登录时验证密码"""
    return pwd_context.verify(plain_password, hashed_password)
```

### 完整用户认证流程

```python
from fastapi import FastAPI, HTTPException
from pydantic import BaseModel
from passlib.context import CryptContext

app = FastAPI()
pwd_context = CryptContext(schemes=["bcrypt"], deprecated="auto")

# 模拟数据库（实际用 SQLAlchemy）
fake_db = {}

class UserRegister(BaseModel):
    username: str
    password: str

class UserLogin(BaseModel):
    username: str
    password: str

@app.post("/register")
def register(user: UserRegister):
    if user.username in fake_db:
        raise HTTPException(status_code=400, detail="用户已存在")
    
    # 存储哈希密码，绝不存明文！
    hashed_password = pwd_context.hash(user.password)
    fake_db[user.username] = {"password_hash": hashed_password}
    return {"message": "注册成功"}

@app.post("/login")
def login(user: UserLogin):
    db_user = fake_db.get(user.username)
    if not db_user:
        raise HTTPException(status_code=401, detail="用户名或密码错误")
    
    # 验证密码（不要泄露是用户名还是密码错误，防止用户名枚举攻击）
    if not pwd_context.verify(user.password, db_user["password_hash"]):
        raise HTTPException(status_code=401, detail="用户名或密码错误")
    
    return {"message": "登录成功"}
```

## 五、常见误区

| 误区 | 纠正 |
|------|------|
| "MD5 加盐就安全了" | 手动加盐的 MD5 依然很快，GPU 一秒可以尝试数十亿次。必须用慢哈希算法 |
| "SHA256 比 MD5 更安全" | 对于密码存储，SHA256 和 MD5 面临同样的暴力破解问题——都太快了 |
| "bcrypt 有效期 2 的 12 次方次迭代，cost=12 就是 12 次" | 错！cost=12 是指数，实际是 2^12 = 4096 次迭代 |
| "每次登录都重新哈希验证很慢" | 设计如此，每次验证几百毫秒对用户体验几乎无感，但让攻击者暴力破解代价极高 |
| "可以把原始密码打印到日志" | 绝对不行！日志可能被多人查看，禁止在任何日志中记录明文密码 |

## 六、面试高频题

**Q：为什么 MD5 存储密码不安全？**
A：两个原因：1）确定性哈希，相同输入输出相同，可被彩虹表攻击；2）速度极快，现代 GPU 每秒可尝试数百亿次。bcrypt/argon2 通过随机盐消灭彩虹表，通过故意慢哈希阻止暴力破解。

**Q：bcrypt 的 cost factor 是什么？应该设多少？**
A：控制哈希计算的迭代次数（2^cost 次），值越大越慢越安全。推荐 12：在现代 CPU 上约 300ms，对用户体验影响可接受，但让攻击者暴力破解代价极高。随 CPU 性能提升，可以逐步调高。

**Q：argon2 比 bcrypt 好在哪里？**
A：argon2 支持配置内存占用（bcrypt 内存很小），高内存设置使得 GPU 并行破解失效——GPU 有很多核心但内存有限，argon2 通过占用大量内存让 GPU 无法有效并行。新项目推荐使用 argon2id。
