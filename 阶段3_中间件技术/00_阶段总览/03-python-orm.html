📌 本章总览

📌 本章为什么要学 Python ORM？▼

ORM 让后端不用拼 SQL 也能操作数据库——SQLAlchemy 是 Python 同步生态的事实标准，Tortoise 把 ORM 带进 async/await 世界，Alembic 让表结构变更也能版本化。读完本章，你能选对 ORM、写出非阻塞的异步查询、用迁移文件管控生产数据库变更。

第一节 · SQLAlchemy 核心用法

SQLAlchemy 把表映射成 Python 类，让后端 CRUD 既安全又少写 SQL。
🎯 学习目标▼
  • 理解 ORM = 表与 Python 类的映射。
  • 掌握 SQLAlchemy 2.0 Model、Session、CRUD。
  • 知道 ORM 和原生 SQL 的边界。
后端场景：日常后台管理、用户系统、业务 CRUD 多数可用 ORM 提升开发效率。
📖 核心概念▼
Table users  → class User
Row record   → User(id=1, username='alice')
CRUD SQL     → session.add / select / update / delete
维度原始 SQLORM
安全手动参数化自动参数化
体验字符串拼接对象属性和 IDE 补全
复杂查询灵活复杂 JOIN 可混用原生 SQL
💻 代码示例▼

来源：demo/1_sqlalchemy_demo.py

class User(Base):
    __tablename__ = "users"
    id: Mapped[int] = mapped_column(Integer, primary_key=True)
    username: Mapped[str] = mapped_column(String(50), unique=True)

with SessionLocal() as session:
    user = User(username="zhangsan", email="zs@example.com")
    session.add(user)
    session.commit()
    session.refresh(user)
⚠️ 常见误区▼
误区纠正
所有查询都必须 ORM复杂聚合/JOIN 可使用原生 SQL。
add 后立即有数据库 ID需要 commit/flush 后 refresh。

第二节 · Tortoise ORM 异步

Tortoise ORM 让数据库访问也能 async/await，配合 FastAPI 不再阻塞事件循环。
🎯 学习目标▼
  • 理解异步 ORM 解决事件循环阻塞。
  • 掌握 async CRUD 与字段定义。
  • 知道 FastAPI 场景下的初始化方式。
📖 核心概念▼
同步 ORM：请求1 等 DB → 请求2 等待
异步 ORM：请求1 发出查询并让出 → 请求2 可继续发查询

Tortoise ORM API 风格接近 Django，完全基于 async/await，适合 FastAPI 等异步 Web 框架。

💻 代码示例▼

来源：demo/2_tortoise_orm_demo.py

class User(Model):
    id = fields.IntField(primary_key=True)
    username = fields.CharField(max_length=50, unique=True)

user = await User.create(username="zhangsan", email="zs@example.com")
users = await User.filter(is_active=True).order_by("-created_at").limit(10)
⚠️ 常见误区▼
误区纠正
异步 ORM 一定更快优势在 I/O 并发，不会让单条 SQL 本身变快。
忘记 await异步 CRUD 必须 await，否则只是协程对象。

第三节 · Alembic 数据库迁移

Alembic 用迁移文件管理表结构版本，让上线和回滚都可追溯。
🎯 学习目标▼
  • 理解迁移文件记录表结构版本。
  • 掌握 revision、upgrade、downgrade 工作流。
  • 知道 Alembic 与 SQLAlchemy metadata 的关系。
📖 核心概念▼
Model 变更 → alembic revision --autogenerate → 迁移文件
迁移文件 → alembic upgrade head → 数据库结构同步
出问题 → alembic downgrade -1 → 回滚
💻 代码示例▼

来源：demo/3_alembic_demo.py

alembic revision --autogenerate -m "initial tables"
alembic upgrade head
alembic downgrade -1

# env.py
target_metadata = Base.metadata
⚠️ 常见误区▼
误区纠正
自动生成后直接上生产必须人工审查迁移文件。
迁移只需要 upgrade生产回滚需要可用的 downgrade。
