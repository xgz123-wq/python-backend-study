3四大中间件对照速查

架构师视角：4 个组件在系统中的位置和职责

中间件层核心职责典型场景替代方案
MySQL持久层数据落地、ACID 事务用户表、订单表、配置表PostgreSQL / Oracle / TiDB
Redis缓存层内存 KV、热点加速会话缓存、排行榜、限流、分布式锁Memcached / KeyDB
ORM应用层对象 ↔ 表的映射用 Python 类代替 SQLSQLAlchemy / Tortoise / Django ORM
MQ通信层异步解耦、削峰填谷发邮件、订单超时、跨服务调用Celery + Redis / RabbitMQ / Kafka

4环境准备（docker 一键启动）

本地不需要装 MySQL / Redis，直接用 docker-compose 拉起

中间件docker-compose 片段端口
MySQL 8mysql:8.03306
Redis 7redis:7-alpine6379
RabbitMQrabbitmq:3-management5672 / 15672
⚠️ 安装注意

MySQL 8 默认字符集是 utf8mb4，不是 utf8。Redis 6+ 开始需要 requirepass 显式设置密码。所有中间件生产环境必须 绑定内网 + 鉴权 + 持久化，不要用默认配置裸奔。
