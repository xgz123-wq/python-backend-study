# 0. MySQL 环境与基础操作

> 入门预备课：在学 SQL 语法之前，先把 Linux 下操控 MySQL 的基本功打牢。

---

## 一、概念解释

MySQL 是最流行的开源关系型数据库，广泛用于 Web 后端。在实际工作中，后端工程师几乎每天都要通过命令行连接数据库排查问题，掌握 Linux 终端操作 MySQL 是基本功。

本课使用的环境：
| 配置项 | 值 |
|--------|-----|
| 容器名 | `mysql-study` |
| 用户名 | `root` |
| 密码   | `123456` |
| 数据库 | `mysql_study` |
| 端口   | `3306` |

## 二、首次环境搭建（只需执行一次）

如果你的虚拟机/服务器上还没有 `mysql-study` 容器，先执行以下命令创建：

```bash
# 进入 demo 目录
cd 阶段3_中间件技术/01_MySQL/demo

# 启动 MySQL 容器（后台运行）
docker-compose up -d

# 等待 healthy 状态（约 30 秒）
docker-compose ps
# 输出应显示：(healthy)

# 导入初始化数据
docker exec -i mysql-study mysql -u root -p123456 mysql_study < init.sql
```

> **说明**：`docker-compose.yml` 中定义了容器名 `mysql-study`、密码 `123456`、数据库 `mysql_study`，与本文档所有命令完全匹配。

## 三、Docker 容器管理

### 查看容器状态

```bash
# 查看所有运行中的容器
docker ps

# 查看所有容器（含已停止）
docker ps -a

# 只查找 mysql-study 容器
docker ps -a | grep mysql-study
```

### 启动/停止容器

```bash
# 启动容器
docker start mysql-study

# 停止容器
docker stop mysql-study

# 重启容器
docker restart mysql-study
```

### 进入容器 Shell（了解即可）

```bash
# 进入容器的 bash
docker exec -it mysql-study bash

# 退出容器 Shell
exit
```

## 四、连接 MySQL

### 方式一：通过 docker exec 直接进入 MySQL（推荐）

```bash
# 连接并选中 mysql_study 数据库
docker exec -it mysql-study mysql -u root -p123456 mysql_study

# 说明：
# -it           → 交互式终端
# mysql-study   → 容器名
# mysql         → 容器内的 mysql 客户端命令
# -u root       → 用户名
# -p123456      → 密码（注意 -p 和密码之间没有空格）
# mysql_study   → 登录后直接进入此数据库
```

> **安全说明**：密码直接写在命令行会被保存在 Shell 历史记录中，生产环境请使用 `mysql -u root -p`（不带密码，回车后再输入）。本地学习环境可以直接带密码，方便快捷。

### 方式二：执行 SQL 文件（批量导入）

```bash
# 将 init.sql 导入 mysql_study 数据库
docker exec -i mysql-study mysql -u root -p123456 mysql_study < init.sql

# 说明：
# -i（不加 t）→ 非交互模式，允许从标准输入读取
# < init.sql  → 将文件内容重定向到 mysql 命令的输入
```

### 方式三：在容器内执行 SQL 文件

```bash
# 先将文件复制到容器中
docker cp init.sql mysql-study:/tmp/init.sql

# 进入容器并执行
docker exec -it mysql-study mysql -u root -p123456 mysql_study -e "source /tmp/init.sql"
```

## 五、MySQL 命令行基本操作

进入 MySQL 后（`mysql>` 提示符），常用命令：

### 4.1 数据库操作

```sql
-- 查看所有数据库
SHOW DATABASES;

-- 切换到 mysql_study 数据库
USE mysql_study;

-- 查看当前所在的数据库
SELECT DATABASE();

-- 创建新数据库（UTF8MB4 字符集）
CREATE DATABASE new_db
  CHARACTER SET utf8mb4
  COLLATE utf8mb4_unicode_ci;

-- 删除数据库（谨慎！不可恢复）
DROP DATABASE new_db;
```

### 4.2 表操作

```sql
-- 查看当前数据库所有表
SHOW TABLES;

-- 查看表结构（字段、类型、索引）
DESC users;
-- 或
DESCRIBE users;

-- 查看建表语句（含字符集、索引、外键）
SHOW CREATE TABLE users\G

-- 查看表的统计信息
SHOW TABLE STATUS LIKE 'users'\G
```

### 4.3 快捷命令

```sql
-- 查看 MySQL 版本
SELECT VERSION();

-- 查看当前时间
SELECT NOW();

-- 查看当前登录用户
SELECT USER();

-- 查看所有系统变量（了解即可）
SHOW VARIABLES LIKE 'character%';
-- 输出示例：
-- character_set_client    → utf8mb4
-- character_set_database  → utf8mb4

-- 查看存储引擎
SHOW ENGINES;
```

### 4.4 退出 MySQL

```sql
-- 两种方式都可以
EXIT;
QUIT;
-- 或 Ctrl+D
```

## 六、字符集说明

为什么要用 `utf8mb4` 而不是 `utf8`？

| 字符集 | 最大字节/字符 | 能否存 Emoji |
|--------|--------------|-------------|
| `utf8`（MySQL 专有）| 3 字节 | ❌ 不支持（Emoji 需要 4 字节） |
| `utf8mb4` | 4 字节 | ✅ 完整支持 |

**结论**：新项目一律用 `utf8mb4` + `utf8mb4_unicode_ci`（ci = case insensitive，大小写不敏感）。

```sql
-- 验证当前数据库字符集
SELECT DEFAULT_CHARACTER_SET_NAME, DEFAULT_COLLATION_NAME
FROM information_schema.SCHEMATA
WHERE SCHEMA_NAME = 'mysql_study';
```

## 七、SQL 书写规范

良好的 SQL 书写习惯让代码更易读、易维护：

### 6.1 大小写约定

```sql
-- ✅ 推荐：关键字大写，表名/字段名小写
SELECT id, username, email
FROM users
WHERE status = 'active';

-- ❌ 不推荐：全部小写混写
select ID, UserName from Users where Status='active';
```

### 6.2 缩进与换行

```sql
-- ✅ 推荐：多条件换行对齐
SELECT
    u.username,
    o.order_no,
    o.total_amount
FROM orders o
JOIN users u ON o.user_id = u.id
WHERE o.status = 'paid'
  AND o.total_amount > 1000
ORDER BY o.created_at DESC;
```

### 6.3 语句结尾

- SQL 语句以 `;` 结尾（命令行中必须，.sql 文件中也统一加）
- 命令行中如果忘记加 `;`，按 Enter 后 MySQL 会等待，继续输入 `;` 再回车即可

### 6.4 注释

```sql
-- 单行注释（推荐）

# 单行注释（MySQL 特有，不推荐，兼容性差）

/*
  多行注释
  用于注释掉大段 SQL
*/
```

## 八、常见坑

| 坑 | 说明 | 解决方案 |
|----|------|---------|
| 中文乱码 | 终端/连接字符集与数据库不一致 | 确保数据库和表均为 `utf8mb4` |
| 忘记 USE 数据库 | 执行 SQL 报 "No database selected" | 进入 MySQL 后先执行 `USE mysql_study;` |
| 密码带特殊字符 | `-p123456` 中如果密码含 `!` 等符号，Shell 会解析 | 用引号括起：`-p'123!456'` |
| 外键约束报错 | 删除父表数据时，子表有关联记录 | 先删子表数据，再删父表；或临时关闭外键检查 |
| 大小写敏感 | Linux 下库名/表名默认区分大小写 | 统一使用小写命名 |

## 九、实战：导入电商数据

按以下步骤验证环境是否正常：

```bash
# 第 1 步：确认容器运行
docker ps | grep mysql-study

# 第 2 步：导入初始化数据（在 demo/ 目录下执行）
docker exec -i mysql-study mysql -u root -p123456 mysql_study < init.sql

# 第 3 步：进入 MySQL 验证
docker exec -it mysql-study mysql -u root -p123456 mysql_study

# 第 4 步：在 MySQL 中执行
SHOW TABLES;
SELECT COUNT(*) FROM users;
SELECT COUNT(*) FROM products;
SELECT COUNT(*) FROM orders;
```

预期输出：
```
+------------+
| Tables in  |
+------------+
| categories |
| orders     |
| products   |
| users      |
+------------+

users: 10 条
products: 20 条
orders: 28 条
```
