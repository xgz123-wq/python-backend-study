# SQLAlchemy 核心用法

> SQLAlchemy 把表映射成 Python 类，让后端 CRUD 既安全又少写 SQL。

## 概念解释

**ORM（Object-Relational Mapping，对象关系映射）** 是一种将数据库表映射为 Python 类的技术：
- 一张数据库表 → 一个 Python 类
- 一行数据记录 → 一个 Python 对象
- CRUD 操作 → 调用对象方法，不手写 SQL

**SQLAlchemy** 是 Python 生态中最成熟的 ORM 框架，支持同步和异步两种模式，覆盖主流数据库（MySQL、PostgreSQL、SQLite 等）。

---

## 为什么需要 ORM？

| 对比维度 | 原始 SQL | ORM |
|---------|---------|-----|
| 代码风格 | 字符串拼接，易出错 | Python 对象，IDE 有补全 |
| 安全性 | 手动防 SQL 注入 | 自动参数化查询 |
| 可移植性 | 换数据库要改 SQL | 切换方言即可 |
| 复杂查询 | 灵活但繁琐 | 简单查询方便，复杂查询仍可用原生 SQL |

**踩坑原则**：ORM 适合日常 CRUD，复杂 JOIN/聚合查询建议混用原生 SQL。

---

## SQLAlchemy 两种使用风格

### 1. Core（底层 SQL 表达式层）
直接操作 Table 对象构造 SQL，接近原生 SQL，适合复杂查询。

### 2. ORM（对象映射层）⭐ 主流用法
定义 Model 类，用 Session 操作对象，本文重点介绍此方式。

### 同步连接（适合脚本、CLI 工具）

```python
from sqlalchemy import create_engine
from sqlalchemy.orm import sessionmaker, DeclarativeBase

# 数据库 URL 格式：dialect+driver://user:password@host:port/dbname
DATABASE_URL = "mysql+pymysql://root:123456@localhost:3306/mydb"

engine = create_engine(
    DATABASE_URL,
    pool_size=5,          # 连接池大小
    max_overflow=10,      # 超出 pool_size 时最多额外创建的连接数
    pool_pre_ping=True,   # 使用连接前先 ping 确认连接存活
    echo=False,           # True 时会打印所有 SQL（调试用）
)

# Session 工厂（每次请求创建一个 Session 实例）
SessionLocal = sessionmaker(bind=engine, autocommit=False, autoflush=False)

# 所有 Model 的基类
class Base(DeclarativeBase):
    pass
```

## 二、定义 Model（数据库表映射）

```python
from datetime import datetime
from sqlalchemy import String, Integer, Text, DateTime, Boolean, ForeignKey
from sqlalchemy.orm import Mapped, mapped_column, relationship

class User(Base):
    __tablename__ = "users"  # 对应数据库表名

    # mapped_column：SQLAlchemy 2.0 推荐的写法（类型安全）
    id: Mapped[int] = mapped_column(Integer, primary_key=True, autoincrement=True)
    username: Mapped[str] = mapped_column(String(50), unique=True, nullable=False, index=True)
    email: Mapped[str] = mapped_column(String(100), unique=True, nullable=False)
    hashed_password: Mapped[str] = mapped_column(String(255), nullable=False)
    is_active: Mapped[bool] = mapped_column(Boolean, default=True)
    created_at: Mapped[datetime] = mapped_column(DateTime, default=datetime.utcnow)

    # 关联关系：一个用户有多篇文章
    posts: Mapped[list["Post"]] = relationship("Post", back_populates="author")

    def __repr__(self) -> str:
        return f"<User id={self.id} username={self.username}>"


class Post(Base):
    __tablename__ = "posts"

    id: Mapped[int] = mapped_column(Integer, primary_key=True, autoincrement=True)
    title: Mapped[str] = mapped_column(String(200), nullable=False)
    content: Mapped[str] = mapped_column(Text, nullable=True)
    # 外键：关联 users 表的 id 字段
    author_id: Mapped[int] = mapped_column(Integer, ForeignKey("users.id"), nullable=False)
    created_at: Mapped[datetime] = mapped_column(DateTime, default=datetime.utcnow)

    # 关联关系：每篇文章属于一个用户
    author: Mapped["User"] = relationship("User", back_populates="posts")
```

## 三、CRUD 操作

### Session 的使用方式

```python
# 推荐：用 with 语句，自动处理关闭
with SessionLocal() as session:
    # 在这里执行数据库操作
    pass
```

### Create（创建）

```python
with SessionLocal() as session:
    user = User(
        username="zhangsan",
        email="zs@example.com",
        hashed_password="hashed_abc123",
    )
    session.add(user)       # 将对象加入 Session（此时还未写库）
    session.commit()        # 提交事务（真正写入数据库）
    session.refresh(user)   # 刷新对象，获取数据库自动生成的字段（如 id）
    print(user.id)          # 现在能拿到自动生成的 id

    # 批量插入
    users = [
        User(username=f"user{i}", email=f"user{i}@example.com", hashed_password="xxx")
        for i in range(10)
    ]
    session.add_all(users)
    session.commit()
```

### Read（查询）

```python
from sqlalchemy import select, and_, or_

with SessionLocal() as session:
    # 按主键查询（最简单）
    user = session.get(User, 1)

    # SELECT ... WHERE（推荐 select() 风格）
    stmt = select(User).where(User.username == "zhangsan")
    user = session.execute(stmt).scalar_one_or_none()
    # scalar_one()         → 恰好1条，否则报错
    # scalar_one_or_none() → 0或1条，0条返回 None
    # scalars().all()      → 返回列表

    # 多条件查询
    stmt = select(User).where(
        and_(User.is_active == True, User.email.like("%@example.com"))
    )
    users = session.execute(stmt).scalars().all()

    # 排序 + 分页
    stmt = (
        select(User)
        .where(User.is_active == True)
        .order_by(User.created_at.desc())
        .offset(0)    # 跳过前 N 条（page-1 * page_size）
        .limit(10)    # 取 N 条
    )
    users = session.execute(stmt).scalars().all()

    # 统计数量
    from sqlalchemy import func
    stmt = select(func.count()).select_from(User).where(User.is_active == True)
    count = session.execute(stmt).scalar()
```

### Update（更新）

```python
with SessionLocal() as session:
    # 方式1：查出来改属性再提交
    user = session.get(User, 1)
    if user:
        user.email = "new_email@example.com"
        session.commit()

    # 方式2：批量 UPDATE（不加载对象，直接 SQL UPDATE，性能更好）
    from sqlalchemy import update
    stmt = (
        update(User)
        .where(User.is_active == False)
        .values(is_active=True)
    )
    session.execute(stmt)
    session.commit()
```

### Delete（删除）

```python
with SessionLocal() as session:
    # 方式1：查出来再删
    user = session.get(User, 1)
    if user:
        session.delete(user)
        session.commit()

    # 方式2：直接 DELETE（不加载对象）
    from sqlalchemy import delete
    stmt = delete(User).where(User.id == 1)
    session.execute(stmt)
    session.commit()
```

## 四、关联查询（JOIN）

```python
with SessionLocal() as session:
    # 查询用户及其所有文章（懒加载 → 需要 joinedload 预加载）
    from sqlalchemy.orm import joinedload

    stmt = (
        select(User)
        .options(joinedload(User.posts))  # 预加载关联数据，避免 N+1 问题
        .where(User.id == 1)
    )
    user = session.execute(stmt).unique().scalar_one_or_none()
    if user:
        for post in user.posts:
            print(post.title)

    # 显式 JOIN 查询
    stmt = (
        select(User, Post)
        .join(Post, User.id == Post.author_id)
        .where(User.is_active == True)
    )
    results = session.execute(stmt).all()
    for user, post in results:
        print(f"{user.username}: {post.title}")
```

## 五、事务处理

```python
with SessionLocal() as session:
    try:
        # 转账示例：扣 A 的余额，加 B 的余额（必须原子执行）
        user_a = session.get(User, 1)
        user_b = session.get(User, 2)

        # ... 业务操作 ...

        session.commit()    # 全部成功才提交
    except Exception:
        session.rollback()  # 任一步失败则回滚
        raise
```

`with SessionLocal() as session` 配合 `autocommit=False` 时，如果 `with` 块正常结束但未调用 `commit()`，事务会被回滚。

## 六、常见坑

1. **N+1 查询问题**：循环中访问关联属性会触发 N 次额外 SQL。解决：用 `joinedload()` 预加载
2. **Session 跨请求复用**：每个请求应使用独立的 Session，不要共享（FastAPI 用 `Depends` 注入）
3. **忘记 commit**：`session.add()` 只是标记，不调用 `commit()` 数据不会写入
4. **复杂查询别硬撑 ORM**：多层嵌套 JOIN + 聚合时，直接用 `session.execute(text("..."))` 执行原生 SQL
5. **`echo=True` 只在开发调试用**：生产环境关闭，否则 SQL 日志刷屏
