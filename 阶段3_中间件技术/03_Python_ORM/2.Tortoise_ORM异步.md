# Tortoise ORM（异步）

> Tortoise ORM 让数据库访问也能 async/await，配合 FastAPI 不再阻塞事件循环。

## 概念解释

**Tortoise ORM** 是一个专为 Python **异步**场景设计的 ORM 框架，灵感来自 Django ORM，API 风格相似但完全基于 `async/await`。

在 FastAPI、Sanic 等异步 Web 框架中，如果使用 SQLAlchemy 同步版本会阻塞事件循环，导致并发性能下降。Tortoise ORM 天然异步，是 FastAPI 项目的主流选择之一。

---

## 为什么需要异步 ORM？

```
同步 ORM（阻塞）：
  请求1 → 等待数据库 100ms → 返回
  请求2 → （必须等请求1完成）→ 等待数据库 100ms → 返回
  总耗时：200ms

异步 ORM（非阻塞）：
  请求1 → 发出数据库查询 → 让出控制权
  请求2 → 发出数据库查询 → 让出控制权
  数据库1返回 → 处理请求1
  数据库2返回 → 处理请求2
  总耗时：~100ms（并发处理）
```

## 一、定义 Model

Tortoise ORM 的 Model 风格接近 Django，字段类型有专门的类。

```python
from tortoise import fields
from tortoise.models import Model
from datetime import datetime


class User(Model):
    # 表配置
    class Meta:
        table = "users"          # 对应数据库表名
        ordering = ["-created_at"]  # 默认排序（-表示降序）

    id = fields.IntField(primary_key=True)
    username = fields.CharField(max_length=50, unique=True)
    email = fields.CharField(max_length=100, unique=True)
    hashed_password = fields.CharField(max_length=255)
    is_active = fields.BooleanField(default=True)
    created_at = fields.DatetimeField(auto_now_add=True)  # 创建时自动设置
    updated_at = fields.DatetimeField(auto_now=True)       # 更新时自动更新

    # 反向关联（声明一对多关系的"多"那端由 ForeignKeyField 定义）
    posts: fields.ReverseRelation["Post"]

    def __str__(self) -> str:
        return f"User(id={self.id}, username={self.username})"


class Post(Model):
    class Meta:
        table = "posts"

    id = fields.IntField(primary_key=True)
    title = fields.CharField(max_length=200)
    content = fields.TextField(null=True)
    # 外键：关联 User 模型，on_delete=CASCADE 表示用户删除时文章也删除
    author = fields.ForeignKeyField(
        "models.User",
        related_name="posts",
        on_delete=fields.CASCADE,
    )
    created_at = fields.DatetimeField(auto_now_add=True)
```

### 常用字段类型

| 字段类 | Python 类型 | 说明 |
|--------|------------|------|
| `IntField` | `int` | 整数 |
| `BigIntField` | `int` | 大整数（ID 常用）|
| `CharField(max_length)` | `str` | 字符串（有长度限制）|
| `TextField` | `str` | 长文本 |
| `BooleanField` | `bool` | 布尔 |
| `FloatField` | `float` | 浮点数 |
| `DecimalField` | `Decimal` | 精确小数（金额必用）|
| `DatetimeField` | `datetime` | 日期时间 |
| `JSONField` | `dict/list` | JSON 数据 |
| `ForeignKeyField` | 关联对象 | 外键 |
| `ManyToManyField` | 关联集合 | 多对多 |

## 二、初始化连接

### 独立脚本中使用

```python
import asyncio
from tortoise import Tortoise

async def init():
    await Tortoise.init(
        db_url="mysql://root:123456@localhost:3306/mydb",
        modules={"models": ["your_app.models"]},  # 包含 Model 的模块路径
    )
    # 自动建表（开发用，生产用 aerich 迁移）
    await Tortoise.generate_schemas()

async def close():
    await Tortoise.close_connections()

async def main():
    await init()
    # ... 数据库操作 ...
    await close()

asyncio.run(main())
```

### 在 FastAPI 中使用（推荐方式）

```python
from contextlib import asynccontextmanager
from fastapi import FastAPI
from tortoise.contrib.fastapi import RegisterTortoise

TORTOISE_ORM = {
    "connections": {"default": "mysql://root:123456@localhost:3306/mydb"},
    "apps": {
        "models": {
            "models": ["app.models", "aerich.models"],  # aerich.models 用于迁移
            "default_connection": "default",
        }
    },
}

@asynccontextmanager
async def lifespan(app: FastAPI):
    async with RegisterTortoise(app, config=TORTOISE_ORM, generate_schemas=False):
        yield

app = FastAPI(lifespan=lifespan)
```

## 三、异步 CRUD 操作

### Create（创建）

```python
# 方式1：create()，一步完成创建并返回对象
user = await User.create(
    username="zhangsan",
    email="zs@example.com",
    hashed_password="hashed_abc",
)
print(user.id)  # 自动生成的主键

# 方式2：先实例化再 save()
user = User(username="lisi", email="ls@example.com", hashed_password="hashed_xyz")
await user.save()

# 批量创建
await User.bulk_create([
    User(username=f"user{i}", email=f"u{i}@example.com", hashed_password="x")
    for i in range(10)
])
```

### Read（查询）

```python
# 按主键查询
user = await User.get(id=1)         # 不存在时抛 DoesNotExist 异常
user = await User.get_or_none(id=1) # 不存在时返回 None（更安全）

# 条件过滤
users = await User.filter(is_active=True).all()

# 单条查询
user = await User.filter(username="zhangsan").first()

# 复杂条件（Q 对象）
from tortoise.expressions import Q

users = await User.filter(
    Q(username="zhangsan") | Q(email="zs@example.com")  # OR 条件
).all()

# 字段查找语法（__后缀）
users = await User.filter(
    email__endswith="@example.com",   # LIKE '%@example.com'
    created_at__gte=datetime(2024, 1, 1),  # >= 某日期
    username__in=["zhangsan", "lisi"],     # IN (...)
).all()

# 排序 + 分页
users = await User.filter(is_active=True).order_by("-created_at").offset(0).limit(10)

# 统计数量
count = await User.filter(is_active=True).count()

# 判断是否存在
exists = await User.filter(username="zhangsan").exists()

# 只查部分字段（减少数据传输）
users = await User.all().values("id", "username", "email")
# 返回 [{"id": 1, "username": "zhangsan", ...}, ...]

# values_list：返回元组列表
ids = await User.all().values_list("id", flat=True)
# 返回 [1, 2, 3, ...]
```

### Update（更新）

```python
# 方式1：查出来改属性再 save()
user = await User.get(id=1)
user.email = "new@example.com"
await user.save()

# 方式2：filter().update() 批量更新（不加载对象，性能好）
updated_count = await User.filter(is_active=False).update(is_active=True)
print(f"更新了 {updated_count} 条")
```

### Delete（删除）

```python
# 查出来再删
user = await User.get(id=1)
await user.delete()

# 批量删除
deleted_count = await User.filter(is_active=False).delete()
```

## 四、关联查询（预加载避免 N+1）

```python
# 问题：N+1 查询（每次访问 author 都触发额外 SQL）
posts = await Post.all()
for post in posts:
    print(post.author)  # ← 每次都查一次 users 表！N+1 问题

# 解决：prefetch_related 预加载
posts = await Post.all().prefetch_related("author")
for post in posts:
    print(post.author.username)  # 不再额外查询

# 多层预加载
users = await User.all().prefetch_related("posts")

# select_related：通过 JOIN 预加载（适合一对一/多对一）
posts = await Post.all().select_related("author")
```

## 五、事务

```python
from tortoise.transactions import in_transaction

async def transfer_demo():
    async with in_transaction():
        # 事务内所有操作要么全成功，要么全回滚
        await User.filter(id=1).update(balance=User.balance - 100)
        await User.filter(id=2).update(balance=User.balance + 100)
        # 如果这里抛出异常，整个事务自动回滚
```

## SQLAlchemy vs Tortoise ORM 对比

| 对比维度 | SQLAlchemy | Tortoise ORM |
|---------|-----------|--------------|
| 异步支持 | 需要 asyncio 扩展（较复杂）| 原生异步 |
| API 风格 | 偏 Python 原生风格 | 类 Django 风格（更易上手）|
| 成熟度 | 非常成熟（10+ 年）| 相对较新（2018+）|
| 迁移工具 | Alembic | Aerich |
| 生态 | 极其丰富 | 一般 |
| 适用场景 | 各类项目，尤其复杂查询 | FastAPI 等纯异步项目 |

**选型建议**：
- 同步项目（Flask/Django CLI）→ SQLAlchemy 同步版
- FastAPI 异步项目 → SQLAlchemy asyncio 扩展 **或** Tortoise ORM
- 追求简单易用 → Tortoise ORM
- 追求成熟稳定 → SQLAlchemy asyncio

## 常见坑

1. **忘记 `await`**：所有数据库操作都是协程，必须 `await`，否则不执行
2. **在同步函数中调用异步 ORM**：必须在 `async def` 中或用 `asyncio.run()`
3. **`get()` 不存在时抛异常**：生产代码优先用 `get_or_none()`
4. **N+1 问题**：循环中访问关联属性一定要用 `prefetch_related` 预加载
5. **`generate_schemas=True` 不适合生产**：开发阶段可以用，生产环境必须用 Aerich 迁移管理表结构
