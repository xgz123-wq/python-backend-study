第一节 · 路由与请求参数

FastAPI 核心第一讲：URL 与 HTTP 方法的映射规则，三种参数类型的差异与陷阱。

学会用一行装饰器，把 URL 映射到处理函数，顺手拿到参数校验。
⏱️ 预估阅读 8 分钟
🎯 学习目标 本节你要带走什么 ▶
  • 理解「路由 = URL + HTTP 方法 + 处理函数」三要素
  • 区分路径参数、查询参数、请求体三种入参位置
  • 掌握装饰器 6 个 HTTP 方法的语义与选择
  • 能用一行装饰器声明类型校验和默认值
  • 理解路径匹配顺序冲突是常见 bug 来源
与后端开发的关联：路由是 REST API 的"门牌号"。理解参数传递位置是设计接口契约（contract）的第一步——前端对接、后端校验、文档自动生成（/docs）都依赖这套规则。
📖 核心概念 三种参数 + 路由装饰器 ▶
核心定义 路由 = URL 路径 + HTTP 方法 + 处理函数；三者拼成一条「method path → handler」的映射规则。请求参数是 handler 接收的入参，按位置分三种。

1. 三种参数类型对比

类型位置URL 示例适用场景
路径参数URL 段GET /items/42唯一资源定位
查询参数?k=v&k2=v2GET /items/?limit=10过滤 / 分页 / 可选参数
请求体JSON bodyPOST /items/创建 / 更新资源

2. 路由装饰器 → HTTP 方法

@app.get("/items/")          # 查询列表     → 幂等
@app.get("/items/{id}")      # 查询单个     → 幂等
@app.post("/items/")         # 创建         → 非幂等
@app.put("/items/{id}")      # 全量更新     → 幂等
@app.patch("/items/{id}")    # 部分更新     → 非幂等
@app.delete("/items/{id}")   # 删除         → 幂等

3. 类型注解 = 自动校验

FastAPI 看 Python 类型注解自动做数据校验，类型不对就返回 422。默认值 = 该参数可选。

路径参数  item_id: int                     # 必须传整数
查询参数  skip: int = 0, limit: int = 10  # 可选，默认值
请求体    item: ItemCreate                 # Pydantic 验证

① 最小三参数演示（来自 1.路由与请求参数.md）

from fastapi import FastAPI
from pydantic import BaseModel

app = FastAPI()

# 路径参数：从 URL 里取
@app.get("/items/{item_id}")
async def get_item(item_id: int):
    return {"id": item_id}

# 查询参数：URL ?key=value
@app.get("/items/")
async def list_items(skip: int = 0, limit: int = 10):
    return {"skip": skip, "limit": limit}

# 请求体：JSON body，用 Pydantic 接收
class ItemCreate(BaseModel):
    name: str
    price: float

@app.post("/items/")
async def create_item(item: ItemCreate):
    return item

② 真实项目：商品 CRUD 路由（来自 routers/items.py）

from fastapi import APIRouter, Depends, HTTPException
from sqlalchemy.ext.asyncio import AsyncSession
from sqlalchemy import select

from app.deps import get_db
from app.models.item import Item
from app.schemas import ItemCreate, ItemUpdate, ItemResponse

router = APIRouter(prefix="/items", tags=["商品"])

# 查询列表：路径 + 查询参数（skip/limit）
@router.get("/", response_model=list[ItemResponse])
async def list_items(
    skip: int = 0,
    limit: int = 10,
    db: AsyncSession = Depends(get_db),
):
    result = await db.execute(select(Item).offset(skip).limit(limit))
    return result.scalars().all()

# 查询单个：路径参数 item_id
@router.get("/{item_id}", response_model=ItemResponse)
async def get_item(item_id: int, db: AsyncSession = Depends(get_db)):
    item = await db.get(Item, item_id)
    if not item:
        raise HTTPException(status_code=404, detail=f"Item {item_id} not found")
    return item

# 创建：请求体 Pydantic
@router.post("/", response_model=ItemResponse, status_code=201)
async def create_item(
    data: ItemCreate,
    db: AsyncSession = Depends(get_db),
):
    db_item = Item(**data.model_dump())
    db.add(db_item)
    await db.commit()
    await db.refresh(db_item)
    return db_item

⚠️ 常见误区 最常踩的坑 ▶
❌ 误区✅ 正确做法
路径参数顺序写反：
/items/{id} 写在 /items/me 前面，me 被当作 id 匹配，永远进不到 get_me		 固定路径（/items/me）必须写在动态路径（/items/{id}）之前
查询参数忘了给默认值，导致它是必填：
async def list(limit: int) 不传 limit 就报 422		 可选项必须给默认值：
async def list(skip: int = 0, limit: int = 10)
把敏感数据放在路径里：
GET /users/123/password/abc 密码会进 URL 日志		 敏感数据走请求体（POST/PUT），不要进 URL
路径匹配顺序：FastAPI 按注册顺序匹配路由，更具体的路径必须先注册。生产里 /items/me 这类"特殊"路径应单独放在 router 文件最前面。
