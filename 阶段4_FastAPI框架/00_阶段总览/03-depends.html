第三节 · 依赖注入 Depends

把「获取数据库连接」「校验 JWT」这些公共逻辑抽成函数，路由自动调用——告别每个路由重复样板代码。

把数据库连接、登录校验抽成函数，每个路由自动复用，不再复制粘贴。
⏱️ 预估阅读 7 分钟
🎯 学习目标 本节你要带走什么 ▶
  • 理解依赖注入 = 把「准备工作」抽成函数，框架自动调用
  • 会用 Depends(get_db) 拿到数据库 session
  • 区分 yield（带清理）与 return（无清理）依赖
  • 理解依赖可嵌套（A 依赖 B，路由依赖 A）
  • 能识别 JWT 鉴权场景下依赖链的设计模式
与后端开发的关联：依赖注入是「控制反转」的轻量实现。在后端 = 路由只需要关心业务，DB / 鉴权 / 限流等横切关注点由框架装配。没有 Depends，每个路由都是 30 行模板代码 + 3 行业务，复制粘贴十几次就漏改一处。
📖 核心概念 工作原理 + yield 与 return 的差别 ▶
核心定义 Depends(函数) = FastAPI 在调用 handler 之前先调用这个函数，把返回值作为参数传给 handler。函数里用 yield 时，yield 之后的代码在请求结束后自动执行（清理钩子）。

1. 工作流程

请求到达
   ↓
FastAPI 看到 handler 参数里有 Depends(...)
   ↓
先调用被依赖的函数：
   - 普通函数 → return 一个值
   - 生成器函数 → 执行到 yield 暂停，把值传给 handler
   ↓
handler 跑业务逻辑
   ↓
handler 返回响应
   ↓
如果依赖函数用了 yield，yield 之后的代码执行（如关闭 db）

2. yield vs return

写法清理钩子适用场景
return obj❌ 无纯计算、不需要释放资源（如 get_current_user）
yield obj✅ yield 之后的代码在请求后执行需要释放资源（数据库 session / 文件句柄 / 连接池）

3. 依赖链（依赖套依赖）

路由 → Depends(get_current_user)
            ↓
         get_current_user(token=Depends(oauth2_scheme), db=Depends(get_db))
            ↓                            ↓
       解析 JWT                拿数据库 session
            ↓                            ↓
         查 User 表 → 把 User 对象返回给路由

4. 没有 Depends 的「坏味道」

@app.get("/items/")
async def list_items():
    db = SessionLocal()
    try:
        result = db.query(Item).all()
        return result
    finally:
        db.close()

@app.post("/items/")
async def create_item(item: ItemCreate):
    db = SessionLocal()        # ← 复制 5 行
    try:
        ...
    finally:
        db.close()

→ 10 个路由就要 10 段「SessionLocal / try / finally / close」
💻 代码示例 从 demo/app/deps.py 摘录 ▶

① 最小对比（来自 3.依赖注入Depends.md）

# ❌ 没有依赖：每个路由都重复
@app.get("/items/")
async def list_items():
    db = SessionLocal()
    try:
        ...
    finally:
        db.close()

# ✅ 用依赖：抽出来，复用
async def get_db():
    db = AsyncSession(engine)
    try:
        yield db          # yield 让 FastAPI 在请求结束后自动 close
    finally:
        await db.close()

@app.get("/items/")
async def list_items(db: AsyncSession = Depends(get_db)):
    ...  # db 已经准备好了

② 真实项目 deps.py

from typing import AsyncGenerator
from sqlalchemy.ext.asyncio import AsyncSession
from app.database import AsyncSessionLocal


async def get_db() -> AsyncGenerator[AsyncSession, None]:
    """
    数据库 session 依赖

    用 yield（不是 return）= 请求结束后自动执行 finally，关闭 session。
    路由里写 db: AsyncSession = Depends(get_db) 就能拿到已经初始化好的 session。
    """
    async with AsyncSessionLocal() as session:
        yield session

③ 路由中使用（来自 routers/items.py）

from fastapi import APIRouter, Depends, HTTPException
from sqlalchemy.ext.asyncio import AsyncSession
from sqlalchemy import select

from app.deps import get_db
from app.models.item import Item
from app.schemas import ItemCreate, ItemUpdate, ItemResponse

router = APIRouter(prefix="/items", tags=["商品"])

# 每个路由都写 db: AsyncSession = Depends(get_db)
# FastAPI 自动调用 get_db()，把 yield 出来的 session 注入
@router.get("/", response_model=list[ItemResponse])
async def list_items(
    skip: int = 0,
    limit: int = 10,
    db: AsyncSession = Depends(get_db),
):
    result = await db.execute(select(Item).offset(skip).limit(limit))
    return result.scalars().all()

④ 鉴权依赖（典型场景）

from fastapi import Depends, HTTPException
from fastapi.security import OAuth2PasswordBearer

# tokenUrl：前端去哪里拿 token（OAuth2 标准的密码模式入口）
oauth2_scheme = OAuth2PasswordBearer(tokenUrl="/auth/token")

# 1. 拿 token 字符串（来自请求头 Authorization: Bearer xxx）
# 2. 拿 db session（依赖嵌套）
# 3. 查 user，返回 ORM 对象
async def get_current_user(
    token: str = Depends(oauth2_scheme),
    db: AsyncSession = Depends(get_db),
) -> User:
    payload = decode_token(token)
    user = await db.get(User, payload["sub"])
    if not user:
        raise HTTPException(status_code=401, detail="Invalid credentials")
    return user

# 路由里直接用：
@app.get("/me")
async def get_me(user: User = Depends(get_current_user)):
    return user
⚠️ 常见误区 最常踩的坑 ▶
❌ 误区✅ 正确做法
数据库 session 依赖里用 return db，请求结束后 session 没关闭，连接泄漏 数据库连接类资源 必须用 yield，让 await db.close() 钩子能跑
把 Depends(get_db) 写在路径函数里：
async def get_item(db=Depends(get_db)): ...		 这种写法是合法的，但建议 显式声明类型 db: AsyncSession = Depends(get_db)，编辑器能补全
每个路由都新建一个 AsyncSessionLocal()，绕开 get_db 统一用 Depends(get_db)，后续要换连接池、加事务管理只改一处
yield 之后忘记写 try/finally，异常时资源不释放 用 async with AsyncSessionLocal() as session，上下文管理器保证 yield 退出时一定执行 close
连接池耗尽：忘记 yield 关闭 db = 每次请求泄漏一个连接。生产环境跑 1 小时后 QueuePool limit of size 10 overflow 10 reached，接口全面 500。
