第五节 · SQLAlchemy 异步集成

把 SQLAlchemy 2.0 的 ORM + 异步引擎 + AsyncSession 拼到 FastAPI 上，让数据真正落库且不阻塞事件循环。

给 ORM 换上异步引擎，FastAPI 才能真正并发访问数据库。
⏱️ 预估阅读 8 分钟
🎯 学习目标 本节你要带走什么 ▶
  • 理解 create_async_engine + async_sessionmaker 的角色
  • 会用 SQLAlchemy 2.0 新语法 Mapped + mapped_column 定义模型
  • 掌握异步 CRUD 四件套：execute / get / add + commit + refresh
  • 知道 expire_on_commit=False 在异步环境必须设置
  • 会用 lifespan 在启动时建表
与后端开发的关联：SQLAlchemy 是 Python 后端事实标准的 ORM。FastAPI 异步 + SQLAlchemy 2.0 异步 = 现代 Python 后端的「标准装配」。生产环境的 MySQL/PostgreSQL 接入只需换连接串，代码 0 改动。
📖 核心概念 三层架构 + 异步 CRUD ▶
核心定义 AsyncEngine（连接池） → async_sessionmaker（session 工厂） → AsyncSession（单次请求的会话）。路由通过 Depends(get_db) 拿一个 session 做事。

1. 三层架构

Engine          进程级单例：连接池，复用 TCP 连接
   ↓
async_sessionmaker(engine)   工厂：每次调用造一个新 session
   ↓
AsyncSession      请求级：从工厂拿，用完关
   ↓
db.execute(...)  真正执行 SQL（await 让出控制权）

2. 异步 CRUD 模板

操作异步写法
查全部result = await db.execute(select(Item)); items = result.scalars().all()
按主键查item = await db.get(Item, item_id)
按条件查单个result = await db.execute(select(Item).where(Item.name == n)); item = result.scalar_one_or_none()
创建db.add(item); await db.commit(); await db.refresh(item)
更新item.x = y; await db.commit()
删除await db.delete(item); await db.commit()
分页await db.execute(select(Item).offset(skip).limit(limit))

3. SQLAlchemy 2.0 新模型声明

老写法（1.x）：                新写法（2.0）：

class Item(Base):              class Item(Base):
    __tablename__ = "items"       __tablename__ = "items"
    id = Column(                  id: Mapped[int] = mapped_column(
        Integer, primary_key=True)      primary_key=True)
    name = Column(               name: Mapped[str] = mapped_column(
        String, nullable=False)         nullable=False)
    price = Column(Float)         price: Mapped[float]

↑ 类型和字段定义分离              ↑ 类型注解 = 字段类型，一行搞定

4. expire_on_commit=False 的作用

默认情况下，commit() 后 ORM 对象的所有属性会"过期"，下次访问就会触发新查询（重新从 db 加载）。在异步环境里，对象可能已经 detach，触发查询会报错 DetachedInstanceError。设置 expire_on_commit=False 关闭这个机制。

💻 代码示例 从 database.py / models/item.py / routers/items.py 摘录 ▶

① 异步引擎 + Session 工厂（来自 database.py）

from sqlalchemy.ext.asyncio import create_async_engine, AsyncSession, async_sessionmaker
from sqlalchemy.orm import DeclarativeBase

# 开发用 SQLite（无需安装 MySQL）
# 切换 MySQL：mysql+aiomysql://root:password@localhost:3306/fastapi_demo
DATABASE_URL = "sqlite+aiosqlite:///./fastapi_demo.db"

engine = create_async_engine(
    DATABASE_URL,
    echo=True,   # 打印 SQL，学习阶段保留，生产改 False
)

# expire_on_commit=False：避免 commit 后访问属性触发新查询（异步环境必须）
AsyncSessionLocal = async_sessionmaker(engine, expire_on_commit=False)


class Base(DeclarativeBase):
    pass


async def init_db():
    """建表（main.py startup 事件调用）"""
    async with engine.begin() as conn:
        await conn.run_sync(Base.metadata.create_all)

② ORM 模型（来自 models/item.py，SQLAlchemy 2.0 风格）

from sqlalchemy.orm import Mapped, mapped_column
from app.database import Base


class Item(Base):
    __tablename__ = "items"

    id: Mapped[int] = mapped_column(primary_key=True, index=True)
    name: Mapped[str] = mapped_column(nullable=False)
    price: Mapped[float] = mapped_column(nullable=False)
    description: Mapped[str | None] = mapped_column(nullable=True)

③ 路由里的异步 CRUD（来自 routers/items.py）

from fastapi import APIRouter, Depends, HTTPException
from sqlalchemy.ext.asyncio import AsyncSession
from sqlalchemy import select

from app.deps import get_db
from app.models.item import Item
from app.schemas import ItemCreate, ItemUpdate, ItemResponse

router = APIRouter(prefix="/items", tags=["商品"])


# 查列表
@router.get("/", response_model=list[ItemResponse])
async def list_items(
    skip: int = 0,
    limit: int = 10,
    db: AsyncSession = Depends(get_db),
):
    result = await db.execute(select(Item).offset(skip).limit(limit))
    return result.scalars().all()


# 查单个
@router.get("/{item_id}", response_model=ItemResponse)
async def get_item(item_id: int, db: AsyncSession = Depends(get_db)):
    item = await db.get(Item, item_id)
    if not item:
        raise HTTPException(status_code=404, detail=f"Item {item_id} not found")
    return item


# 创建
@router.post("/", response_model=ItemResponse, status_code=201)
async def create_item(
    data: ItemCreate,
    db: AsyncSession = Depends(get_db),
):
    db_item = Item(**data.model_dump())
    db.add(db_item)
    await db.commit()
    await db.refresh(db_item)   # 刷新拿到数据库生成的 id
    return db_item


# 部分更新
@router.patch("/{item_id}", response_model=ItemResponse)
async def update_item(item_id: int, data: ItemUpdate, db: AsyncSession = Depends(get_db)):
    item = await db.get(Item, item_id)
    if not item:
        raise HTTPException(status_code=404, detail=f"Item {item_id} not found")

    # exclude_unset=True：只更新请求里实际传了值的字段
    update_data = data.model_dump(exclude_unset=True)
    for field, value in update_data.items():
        setattr(item, field, value)

    await db.commit()
    await db.refresh(item)
    return item


# 删除
@router.delete("/{item_id}", status_code=204)
async def delete_item(item_id: int, db: AsyncSession = Depends(get_db)):
    item = await db.get(Item, item_id)
    if not item:
        raise HTTPException(status_code=404, detail=f"Item {item_id} not found")
    await db.delete(item)
    await db.commit()

④ 启动时建表（main.py 的 lifespan）

from contextlib import asynccontextmanager
from fastapi import FastAPI
from app.database import init_db
from app.routers.items import router as items_router


# 生命周期管理（替代已废弃的 on_event）
@asynccontextmanager
async def lifespan(app: FastAPI):
    # 启动时建表
    await init_db()
    yield
    # 关闭时可做清理


app = FastAPI(title="FastAPI 学习 Demo", lifespan=lifespan)
app.include_router(items_router)
⚠️ 常见误区 最常踩的坑 ▶
❌ 误区✅ 正确做法
忘了 expire_on_commit=False，commit 后访问 ORM 属性报 DetachedInstanceError 异步 session 工厂必须传 expire_on_commit=False
用老的 query() 写法：db.query(Item).filter(...).all()（1.x 风格） 用 2.0 风格 await db.execute(select(Item).where(...))，可配合 async with 和显式 await
用 sessionmaker 而不是 async_sessionmaker，造出的是同步 session 异步项目必须 from sqlalchemy.ext.asyncio import async_sessionmaker
用废弃的 @app.on_event("startup")，FastAPI 0.110+ 已弃用 用 @asynccontextmanager + lifespan 参数
create 之后忘了 await db.refresh(item)，返回的 ORM 对象没有数据库生成的 id commit 后必须 refresh，才能拿到自增 id、默认值等数据库侧生成字段
驱动不匹配：连接串里协议名要带异步驱动前缀：sqlite+aiosqlite / postgresql+asyncpg / mysql+aiomysql。写成 sqlite:///./app.db 用的是同步驱动，create_async_engine 会报错。
