# 路由与请求参数

> 学会用一行装饰器，把 URL 映射到处理函数，顺手拿到参数校验。

**一句话定义**：路由 = URL + HTTP方法的映射规则，请求参数 = 客户端传给后端的数据，分三种：路径参数、查询参数、请求体。

## 最小代码片段

```python
from fastapi import FastAPI
from pydantic import BaseModel

app = FastAPI()

# 路径参数：从 URL 里取
@app.get("/items/{item_id}")
async def get_item(item_id: int):
    return {"id": item_id}

# 查询参数：URL ?key=value
@app.get("/items/")
async def list_items(skip: int = 0, limit: int = 10):
    return {"skip": skip, "limit": limit}

# 请求体：JSON body，用 Pydantic 接收
class ItemCreate(BaseModel):
    name: str
    price: float

@app.post("/items/")
async def create_item(item: ItemCreate):
    return item
```

## 三种参数对比

| 类型 | 位置 | 示例 | 适用场景 |
|------|------|------|---------|
| 路径参数 | `/items/{id}` | GET /items/42 | 唯一资源定位 |
| 查询参数 | `?k=v` | GET /items/?limit=10 | 过滤/分页/可选参数 |
| 请求体 | JSON body | POST /items/ | 创建/更新资源 |

## 1 个常见坑

**路径参数顺序冲突**：`/items/me` 和 `/items/{id}` 同时存在时，`/items/me` 必须写在前面，否则 `me` 会被当成 `{id}` 匹配。

```python
# ✅ 正确顺序
@app.get("/items/me")
async def get_me(): ...

@app.get("/items/{item_id}")
async def get_item(item_id: int): ...
```
