# Pydantic 数据验证

> 用类型注解写一份数据契约，前端发错格式自动返回 422。

**一句话定义**：Pydantic = 用 Python 类型注解自动做数据验证和序列化的库，FastAPI 用它接收请求、返回响应，验证失败自动报 422。

## 最小代码片段

```python
from pydantic import BaseModel, Field
from typing import Optional

# 请求 Schema（接收客户端数据）
class ItemCreate(BaseModel):
    name: str = Field(..., min_length=1, max_length=100)
    price: float = Field(..., gt=0)          # gt=0：必须大于0
    description: Optional[str] = None        # 可选字段

# 响应 Schema（返回给客户端的数据，可以不同于存库的字段）
class ItemResponse(BaseModel):
    id: int
    name: str
    price: float

    model_config = {"from_attributes": True}  # 允许从 ORM 对象转换

# 路由中使用
@app.post("/items/", response_model=ItemResponse)
async def create_item(item: ItemCreate):
    # item 已经过验证，name 一定有值，price 一定 > 0
    ...
```

## 请求 vs 响应 Schema 分开写的原因

| Schema 类型 | 作用 | 字段特点 |
|-------------|------|---------|
| `XxxCreate` | 接收创建请求 | 无 id，有必填字段 |
| `XxxUpdate` | 接收更新请求 | 字段全部 Optional |
| `XxxResponse` | 返回给客户端 | 有 id，可能排除密码等敏感字段 |

## 1 个常见坑

**`response_model` 会过滤字段**：路由函数实际返回的对象字段比 `response_model` 多也没问题，Pydantic 只序列化 Schema 里声明的字段。密码字段只要不放进 `XxxResponse`，就不会泄露给客户端。
