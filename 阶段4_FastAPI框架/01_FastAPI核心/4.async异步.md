# async / await 异步

> 等数据库时不阻塞别的请求，用 async/await 撑住高并发。

**一句话定义**：`async/await` = 让程序在等待 IO（数据库查询、网络请求）时不阻塞，去处理其他请求，提升并发能力。FastAPI 天生异步，路由必须配套异步驱动。

## 核心概念

```
同步：等数据库返回 → 什么都不做 → 继续处理
异步：等数据库返回 → 去处理其他请求 → 数据库好了再回来
```

## 最小代码片段

```python
import asyncio

# async def 声明异步函数
async def fetch_user(user_id: int):
    await asyncio.sleep(0.1)   # await = 等待这个 IO 操作，期间可处理其他事
    return {"id": user_id}

# FastAPI 路由：async def 路由 + await 调用异步函数
@app.get("/users/{user_id}")
async def get_user(user_id: int):
    user = await fetch_user(user_id)   # await 异步调用
    return user
```

## 什么场景必须用异步

| 场景 | 要求 |
|------|------|
| 数据库查询（SQLAlchemy async） | 必须 `await`，驱动必须是异步的（`aiosqlite`/`asyncpg`） |
| Redis 操作（aioredis） | 必须 `await` |
| HTTP 请求（httpx AsyncClient） | 必须 `await` |
| 纯 CPU 计算（循环/排序） | 不需要 async，直接用普通函数 |

## 1 个常见坑

**禁止在异步路由里用同步数据库驱动**：`pymysql`/`psycopg2` 是同步的，在 `async def` 里调用会**阻塞整个事件循环**，让并发完全失效。必须换成 `aiosqlite`/`asyncpg`。

```python
# ❌ 危险：同步驱动 + 异步路由
@app.get("/users/")
async def list_users():
    conn = pymysql.connect(...)   # 阻塞！

# ✅ 正确：异步驱动
@app.get("/users/")
async def list_users(db: AsyncSession = Depends(get_db)):
    result = await db.execute(select(User))  # 非阻塞
```
