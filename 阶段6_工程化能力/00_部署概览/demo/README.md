## Demo 详细说明

### Demo 1: 部署概念（1_deploy_concept.py）

通过实际操作理解"部署"的含义：

1. **启动本地 HTTP 服务器**：在本机启动一个简单的 Web 服务器
2. **查看本机 IP**：展示内网 IP（如 `192.168.x.x`）和公网 IP 的区别
3. **端口演示**：理解端口在网络通信中的角色
4. **本地可访问 vs 外部不可访问**：直观体验"本地开发"的局限性

运行后会在终端显示本机网络信息，并启动一个本地服务器供浏览器访问。

### Demo 2: 请求链路（2_request_chain.py）

追踪用户请求经过的每个环节：

1. **DNS 解析**：使用 `socket` 查询域名对应的 IP 地址
2. **HTTP 请求**：向真实服务器发送请求并测量耗时
3. **各环节耗时分析**：分别测量 DNS 查询、TCP 连接、数据传输的时间
4. **CDN vs 直连**：比较通过 CDN 和直接访问源站的速度差异

### Demo 3: 三个部署层次（3_deploy_levels.py）

动手体验三种部署方式：

1. **静态部署**：生成 HTML 文件并用 Python HTTP 服务器托管
2. **应用部署**：启动带 API 路由的后端服务
3. **容器化部署**：展示 Dockerfile 模板（仅打印，不执行 Docker 命令）

### Demo 4: 核心概念（4_core_concepts.py）

实践部署中的核心概念：

1. **环境隔离**：读写环境变量，体验不同环境的配置差异
2. **构建概念**：展示 Python 项目打包的基本思路
3. **回滚概念**：用 Git 模拟版本切换和回滚
4. **零停机部署**：模拟新旧进程的平滑切换

## 核心要点

1. **部署 = 让代码从本地走向互联网**，核心是服务器、网络和域名
2. **用户请求经过多个环节**（DNS → CDN → Nginx → 应用 → 数据库），每个环节都可能成为瓶颈
3. **根据项目规模选择部署方式**：静态部署 → 应用部署 → 容器化部署
4. **CI/CD、环境隔离、回滚机制**是安全部署的基础保障

## 常见问题

### Q: Demo 1 启动服务器后，为什么手机访问不了？

因为你的电脑和手机可能不在同一个 Wi-Fi 网络下，或者路由器防火墙阻止了访问。在同一局域网内，用手机浏览器输入电脑的局域网 IP 和端口即可访问。

### Q: Demo 2 中 DNS 查询的 IP 为什么和浏览器看到的不一样？

可能原因是 CDN。浏览器访问时 DNS 返回的是 CDN 节点 IP，而 `socket.gethostbyname()` 可能返回不同的 CDN 节点或源站 IP。

### Q: Demo 3 的静态部署和真正的 Vercel 部署有什么区别？

Demo 只是用 Python 模拟了静态文件托管的概念。真正的 Vercel 会提供全球 CDN、自动 HTTPS、自动构建等功能。

### Q: Demo 4 中 Git 回滚演示需要我先初始化 Git 仓库吗？

不需要，Demo 会自动在临时目录中初始化 Git 仓库来演示。
