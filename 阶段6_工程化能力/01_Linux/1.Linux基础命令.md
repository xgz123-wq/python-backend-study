# Linux 基础命令

> 服务器命令行是后端的工作台，先把目录、文件、管道、重定向这些日常操作练熟。

## 一、Linux 发行版简介

Linux 不是一个单独的操作系统，而是一个**内核**。围绕这个内核，不同组织打包出了各种**发行版**（Distribution）。

### 主流发行版对比

| 发行版 | 包管理器 | 特点 | 典型使用场景 |
|--------|---------|------|-------------|
| **Ubuntu** | `apt` | 社区活跃、文档丰富、上手简单 | 开发机、云服务器、AI/ML 环境 |
| **Debian** | `apt` | 稳定至上、更新保守 | 追求稳定的服务器 |
| **CentOS / Rocky Linux** | `yum` / `dnf` | 企业级、与 RHEL 兼容 | 企业服务器（CentOS 8 已停更，Rocky Linux 是其替代） |
| **Alpine** | `apk` | 极小体积（~5MB） | Docker 容器基础镜像 |

> **后端开发者建议**：新手首选 Ubuntu 22.04 LTS（长期支持版），云服务器上遇到最多的也是 Ubuntu。

## 二、文件系统结构

Windows 用盘符（C:\、D:\）组织文件，Linux 用**一棵以 `/`（根目录）为起点的树**。

```
/                    # 根目录，一切的起点
├── bin              # 基础命令（ls, cp, mv 等）
├── etc              # 配置文件（系统级"设置"）
│   ├── nginx/       #   Nginx 配置
│   ├── ssh/         #   SSH 配置
│   └── passwd       #   用户信息
├── home             # 普通用户的家目录
│   └── alice/       #   用户 alice 的家目录（~alice）
├── root             # root 用户的家目录
├── var              # 可变数据（日志、缓存、数据库文件）
│   ├── log/         #   系统日志（/var/log/syslog, /var/log/nginx/）
│   └── lib/         #   数据库文件（如 MySQL 的 /var/lib/mysql/）
├── usr              # 用户安装的程序
│   ├── bin/         #   用户安装的命令
│   └── local/       #   手动编译安装的软件（/usr/local/bin）
├── tmp              # 临时文件（重启后可能清空）
├── opt              # 第三方大型软件（如 JDK、Anaconda）
├── dev              # 设备文件（/dev/null, /dev/sda）
├── proc             # 虚拟文件系统，映射内核和进程信息
│   └── cpuinfo      #   CPU 信息
└── sys              # 虚拟文件系统，映射设备和驱动信息
```

### 后端开发者最常接触的目录

| 目录 | 场景 |
|------|------|
| `/etc/nginx/` | 修改 Nginx 反向代理配置 |
| `/var/log/` | 查看系统和应用日志 |
| `/home/你的用户名/` | 日常工作目录 |
| `/opt/` | 安装 JDK、Anaconda 等 |
| `/tmp/` | 临时调试文件 |

## 三、基础命令

### 3.1 目录与文件导航

```bash
# pwd - 显示当前工作目录（Print Working Directory）
pwd
# 输出: /home/alice/projects

# cd - 切换目录（Change Directory）
cd /var/log              # 进入绝对路径
cd nginx                 # 进入相对路径（当前目录下的 nginx）
cd ..                    # 回到上级目录
cd ~                     # 回到家目录（等价于 cd）
cd -                     # 回到上一次所在目录（很实用）

# ls - 列出目录内容
ls                       # 列出当前目录的文件
ls -l                    # 长格式（显示权限、大小、日期）
ls -la                   # 包含隐藏文件（以 . 开头的文件）
ls -lh                   # 人类可读的大小（K, M, G）
ls -lt                   # 按时间排序（最新在前）
```

### 3.2 目录和文件操作

```bash
# mkdir - 创建目录
mkdir project                # 创建单级目录
mkdir -p a/b/c               # 递归创建多级目录（-p = parents）

# touch - 创建空文件 / 更新时间戳
touch readme.txt             # 如果文件不存在就创建，存在则更新修改时间

# cp - 复制
cp file.txt backup.txt           # 复制文件
cp -r src_dir/ dest_dir/         # 递归复制目录（-r = recursive）
cp -i file.txt backup.txt        # 覆盖前询问（-i = interactive）

# mv - 移动 / 重命名
mv old_name.txt new_name.txt     # 重命名（本质是移动）
mv file.txt /tmp/                # 移动到 /tmp 目录

# rm - 删除
rm file.txt                  # 删除文件
rm -i file.txt               # 删除前确认（推荐养成习惯）
rm -r directory/             # 递归删除目录

# ⚠️ 高危命令，务必小心
# rm -rf /                   # ← 永远不要执行！删除整个系统
# rm -rf *                   # ← 删除当前目录所有文件，不可恢复
```

### 3.3 查看文件内容

```bash
# cat - 一次性输出全部内容（适合短文件）
cat /etc/hostname
cat file1.txt file2.txt      # 拼接多个文件

# head / tail - 查看开头 / 结尾
head -n 20 access.log        # 前 20 行
tail -n 20 access.log        # 后 20 行
tail -f access.log           # 实时追踪日志（-f = follow），调试必备

# less - 分页查看（适合大文件，按 q 退出）
less /var/log/syslog

# wc - 统计行数/字数/字节数
wc -l access.log             # 行数（-l = lines）
```

### 3.4 搜索与定位

```bash
# which - 查找命令的完整路径
which python3                # /usr/bin/python3
which nginx                  # /usr/sbin/nginx

# find - 在文件系统中搜索文件
find /var/log -name "*.log"              # 按名称搜索
find /home -size +100M                   # 按大小搜索（> 100MB）
find . -name "*.pyc" -mtime +7           # 7 天前修改的 .pyc 文件
find . -type f -name "*.conf"            # 只搜索普通文件（-type f）
find . -type d -name "node_modules"      # 只搜索目录（-type d）

# locate - 快速搜索（基于数据库，需先 updatedb）
locate nginx.conf            # 比 find 快，但数据库可能不是最新
```

### 3.5 其他实用命令

```bash
# echo - 输出文本
echo "Hello World"           # 输出字符串
echo $PATH                   # 输出环境变量

# df / du - 磁盘使用
df -h                        # 各分区磁盘使用情况（-h = human-readable）
du -sh /var/log/             # 指定目录的总大小

# history - 命令历史
history                      # 查看最近执行的命令
!42                          # 重新执行 history 中第 42 条命令
!!                           # 重新执行上一条命令
```

## 四、管道和重定向

这是 Linux 最强大的设计哲学：**每个程序做好一件事，通过管道组合它们**。

### 4.1 重定向

```bash
# > 输出重定向（覆盖写入）
echo "hello" > file.txt              # 写入文件（覆盖原内容）
ls -l > listing.txt                  # 把 ls 的输出保存到文件

# >> 输出重定向（追加写入）
echo "world" >> file.txt             # 追加到文件末尾
echo "error occurred" >> errors.log  # 日志追加

# < 输入重定向
sort < unsorted.txt                  # 从文件读取输入
mysql -u root -p < init.sql          # 把 SQL 文件喂给 MySQL

# 2> 错误输出重定向
python app.py 2> errors.log          # 只重定向 stderr
python app.py > output.log 2>&1      # stdout 和 stderr 都写入同一文件
python app.py > /dev/null 2>&1       # 丢弃所有输出（静默运行）
```

### 4.2 管道

```bash
# | 把前一个命令的 stdout 传给后一个命令作为 stdin
cat access.log | grep "500"                  # 查找状态码 500 的请求
ps aux | grep python                         # 查找 Python 进程
history | grep "pip install"                 # 在命令历史中搜索

# 多级管道组合
cat access.log | grep "200" | wc -l          # 统计 200 响应次数
ls -l | sort -k5 -n | tail -5               # 找出最大的 5 个文件
cat error.log | grep "Exception" | sort | uniq -c | sort -rn
                                              # 统计异常出现次数并排序

# xargs - 把 stdin 转成命令参数
find . -name "*.log" | xargs rm             # 删除所有 .log 文件
find . -name "*.py" | xargs wc -l           # 统计所有 .py 文件行数
```

### 4.3 标准流总结

| 名称 | 文件描述符 | 缩写 | 默认去向 |
|------|-----------|------|---------|
| 标准输入 | 0 | stdin | 键盘 |
| 标准输出 | 1 | stdout | 终端屏幕 |
| 标准错误 | 2 | stderr | 终端屏幕 |

## 五、包管理

### 5.1 apt（Ubuntu / Debian）

```bash
# 更新软件源索引（必做第一步）
sudo apt update

# 升级所有已安装的软件
sudo apt upgrade -y

# 安装软件
sudo apt install nginx -y
sudo apt install python3-pip -y

# 卸载软件
sudo apt remove nginx              # 卸载但保留配置
sudo apt purge nginx               # 卸载并删除配置

# 搜索软件
apt search redis

# 查看已安装的包
apt list --installed
```

### 5.2 yum / dnf（CentOS / Rocky Linux）

```bash
sudo yum update                    # 更新所有软件
sudo yum install nginx -y          # 安装
sudo yum remove nginx              # 卸载
yum search redis                   # 搜索

# CentOS 8+ 使用 dnf（yum 的升级版）
sudo dnf install python3 -y
```

### 5.3 通用技巧

```bash
# 从源码编译安装（当包管理器版本太旧时）
wget https://example.com/software-1.0.tar.gz
tar -xzf software-1.0.tar.gz
cd software-1.0
./configure --prefix=/usr/local
make
sudo make install

# 使用 snap（Ubuntu 通用包格式）
sudo snap install code --classic    # 安装 VS Code
```

## 六、常见坑

### 坑 1：`rm` 删除不可恢复

Linux 没有回收站！`rm` 删除的文件**无法通过常规方式恢复**。

```bash
# 错误示范
rm -rf /path/to/dir    # 手滑多打一个空格就完了
rm -rf / path/to/dir   # ← 注意空格！这会先删 / 再删 path

# 防护措施
# 1. 用 rm -i 确认后再删
# 2. 用 trash-cli 替代 rm（移入回收站）
# 3. 重要操作前先 ls 确认路径
ls /path/to/dir && rm -rf /path/to/dir
```

### 坑 2：`>` 覆盖 vs `>>` 追加

```bash
echo "important" > config.txt     # 覆盖了原文件！
echo "important" >> config.txt    # 追加到末尾（安全）

# 开启 shell 保护：防止意外覆盖
set -o noclobber                  # 之后 > 不能覆盖已有文件
```

### 坑 3：find 命令性能问题

```bash
# 错误示范：在根目录搜索，会遍历整个磁盘，非常慢
find / -name "config.py"

# 正确做法：限定搜索范围
find /home/alice/projects -name "config.py"
```

### 坑 4：管道中的 grep 缓冲问题

```bash
# 问题：管道中 grep 会缓冲输出，导致 tail -f | grep 看不到实时结果
tail -f app.log | grep "error"     # 可能看不到实时输出

# 解决：使用 --line-buffered 强制逐行输出
tail -f app.log | grep --line-buffered "error"
```

### 坑 5：忘记 `sudo` 导致权限不足

```bash
# 常见报错
apt install nginx
# E: Could not open lock file ... Permission denied

# 解决：加上 sudo
sudo apt install nginx
```

### 坑 6：`cd` 到带空格的路径

```bash
# 错误
cd /path/to/my folder        # bash 认为是两个参数

# 正确
cd "/path/to/my folder"      # 加引号
cd /path/to/my\ folder       # 用反斜杠转义空格
```

## 七、实用快捷键

| 快捷键 | 功能 |
|--------|------|
| `Tab` | 自动补全命令或路径（最常用） |
| `Tab Tab` | 显示所有补全候选 |
| `Ctrl + R` | 反向搜索命令历史 |
| `Ctrl + C` | 终止当前命令 |
| `Ctrl + Z` | 暂停当前命令（放入后台） |
| `Ctrl + D` | 退出当前终端 |
| `Ctrl + L` | 清屏（等价于 `clear`） |
| `Ctrl + A` / `Ctrl + E` | 跳到行首 / 行尾 |
| `Ctrl + W` | 删除光标前一个单词 |
| `↑` / `↓` | 浏览上一条 / 下一条命令 |
