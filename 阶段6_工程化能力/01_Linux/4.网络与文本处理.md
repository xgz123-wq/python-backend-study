# 网络与文本处理

> curl、netstat、grep、awk——线上排错就是靠这套组合从日志里找答案。

## 一、网络基础命令

### 1.1 curl - HTTP 请求工具

```bash
# 基本 GET 请求
curl https://httpbin.org/get

# 显示响应头
curl -I https://httpbin.org/get        # -I = HEAD 请求，只看响应头

# 带参数的请求
curl "https://httpbin.org/get?name=alice&age=20"

# POST 请求
curl -X POST https://httpbin.org/post \
     -H "Content-Type: application/json" \
     -d '{"name": "alice", "age": 20}'

# 下载文件
curl -o file.zip https://example.com/file.zip        # -o 指定输出文件名
curl -O https://example.com/file.zip                  # -O 使用 URL 中的文件名
curl -L https://example.com/redirect                  # -L 跟随重定向

# 实用场景：测试 API
curl -s https://api.github.com/users/octocat | python3 -m json.tool
# -s 静默模式，json.tool 格式化输出
```

### 1.2 wget - 文件下载

```bash
# 基本下载
wget https://example.com/file.zip

# 后台下载 + 断点续传
wget -c -b https://example.com/large-file.iso
# -c = continue 断点续传
# -b = background 后台下载

# 递归下载整个网站
wget -r -np -k https://docs.example.com/
# -r = recursive 递归
# -np = no-parent 不追溯上级目录
# -k = 转换链接为本地链接

# curl vs wget
# curl: 更灵活，支持各种 HTTP 方法，适合 API 调试
# wget: 下载文件更方便，支持递归下载
```

### 1.3 网络诊断

```bash
# ping - 测试连通性
ping google.com                  # 持续 ping（Ctrl+C 停止）
ping -c 5 google.com            # 只 ping 5 次
ping -c 1 -W 2 192.168.1.1     # 超时 2 秒（-W = timeout）

# ss - 查看网络连接和端口（现代工具，替代 netstat）
ss -tlnp                        # 查看监听中的 TCP 端口
# -t = TCP, -l = listening, -n = 数字显示, -p = 显示进程
ss -tnp                         # 查看所有 TCP 连接（包括已建立的）
ss -unp                         # 查看 UDP 端口

# 输出示例：
# State    Recv-Q  Send-Q  Local Address:Port  Peer Address:Port  Process
# LISTEN   0       128     0.0.0.0:80          0.0.0.0:*          users:(("nginx",pid=1234,fd=6))
# LISTEN   0       128     0.0.0.0:22          0.0.0.0:*          users:(("sshd",pid=567,fd=3))

# netstat - 旧版网络查看工具（部分系统需安装 net-tools）
sudo netstat -tlnp              # 功能同 ss -tlnp
sudo netstat -anp | grep 8080   # 查看 8080 端口

# ip - 查看和配置网络接口（替代 ifconfig）
ip addr                         # 查看所有网络接口的 IP 地址
ip route                        # 查看路由表
ip link                         # 查看网络接口状态

# ifconfig - 旧版网络配置（需安装 net-tools）
ifconfig                        # 类似 ip addr
ifconfig eth0                   # 查看指定接口
```

## 二、端口和防火墙

### 2.1 ufw - 简单防火墙（Ubuntu）

```bash
# 基本操作
sudo ufw status                 # 查看防火墙状态和规则
sudo ufw enable                 # 启用防火墙
sudo ufw disable                # 禁用防火墙

# 开放/关闭端口
sudo ufw allow 22/tcp           # 允许 SSH（TCP 22）
sudo ufw allow 80/tcp           # 允许 HTTP
sudo ufw allow 443/tcp          # 允许 HTTPS
sudo ufw allow 8080             # 允许 8080 端口（TCP+UDP）
sudo ufw deny 3306              # 拒绝 3306 端口（MySQL 不对外暴露）
sudo ufw delete allow 8080      # 删除规则

# 常用预设
sudo ufw allow ssh              # 等价于 allow 22/tcp
sudo ufw allow http             # 等价于 allow 80/tcp
sudo ufw allow https            # 等价于 allow 443/tcp

# 限制连接频率（防暴力破解）
sudo ufw limit ssh              # 限制 SSH 连接频率
```

### 2.2 iptables 基础

```bash
# iptables 是更底层的防火墙工具，ufw 实际上是 iptables 的前端

# 查看当前规则
sudo iptables -L -n -v          # -L 列出, -n 数字显示, -v 详细信息

# 基本规则
sudo iptables -A INPUT -p tcp --dport 80 -j ACCEPT       # 允许 80 端口入站
sudo iptables -A INPUT -p tcp --dport 443 -j ACCEPT      # 允许 443 端口入站
sudo iptables -A INPUT -s 192.168.1.0/24 -j ACCEPT       # 允许内网访问
sudo iptables -D INPUT -p tcp --dport 80 -j ACCEPT       # 删除规则

# 保存规则（重启后生效）
sudo iptables-save > /etc/iptables/rules.v4

# ⚠️ 生产环境建议用 ufw，iptables 规则复杂容易出错
```

## 三、文本处理三剑客

### 3.1 grep - 搜索文本

```bash
# 基本搜索
grep "error" app.log                    # 在文件中搜索 "error"
grep -i "error" app.log                 # -i 忽略大小写
grep -n "error" app.log                 # -n 显示行号
grep -c "error" app.log                 # -c 统计匹配行数
grep -v "error" app.log                 # -v 反向匹配（排除）
grep -r "TODO" ./src/                   # -r 递归搜索目录

# 正则表达式
grep -E "^[0-9]{4}-[0-9]{2}" app.log   # -E 扩展正则，搜索日期格式
grep -E "(ERROR|WARN)" app.log          # 搜索 ERROR 或 WARN
grep -E "status=(4|5)[0-9]{2}" access.log  # 搜索 4xx 和 5xx 状态码

# 上下文
grep -A 3 "error" app.log              # 显示匹配行及后面 3 行（After）
grep -B 2 "error" app.log              # 显示匹配行及前面 2 行（Before）
grep -C 3 "error" app.log              # 显示匹配行及前后各 3 行（Context）

# 实用组合
ps aux | grep python | grep -v grep     # 查找 Python 进程
cat /etc/passwd | grep "/bin/bash"      # 查找使用 bash 的用户
```

### 3.2 sed - 流编辑器

```bash
# 替换文本（最常用功能）
sed 's/old/new/' file.txt               # 替换每行第一个匹配
sed 's/old/new/g' file.txt              # 替换每行所有匹配（g = global）
sed 's/old/new/gi' file.txt             # 忽略大小写替换

# 修改文件（-i 原地修改）
sed -i 's/DEBUG/INFO/g' config.txt      # 直接修改文件
sed -i.bak 's/old/new/g' file.txt       # 修改前备份（推荐）

# 删除行
sed '3d' file.txt                       # 删除第 3 行
sed '/^$/d' file.txt                    # 删除空行
sed '/^#/d' config.txt                  # 删除注释行（# 开头）

# 提取行
sed -n '5,10p' file.txt                 # 只输出第 5-10 行（-n 抑制默认输出）
sed -n '/ERROR/p' app.log              # 只输出包含 ERROR 的行

# 在指定位置插入/追加
sed '3i\新的一行内容' file.txt           # 在第 3 行前插入
sed '3a\新的一行内容' file.txt           # 在第 3 行后追加

# 批量替换配置文件中的 IP
sed -i 's/192.168.1.100/10.0.0.1/g' nginx.conf
```

### 3.3 awk - 强大的文本分析工具

```bash
# 基本用法：按列提取
# 假设 access.log 格式：IP - - [日期] "请求" 状态码 大小
awk '{print $1}' access.log              # 输出第一列（IP 地址）
awk '{print $1, $7}' access.log          # 输出第 1 和第 7 列

# 指定分隔符
awk -F: '{print $1}' /etc/passwd        # 以 : 分隔，输出用户名
awk -F',' '{print $2}' data.csv         # 以逗号分隔，输出第 2 列

# 条件过滤
awk '$3 > 100 {print $1, $3}' data.txt  # 第 3 列 > 100 的行
awk '/ERROR/ {print NR, $0}' app.log    # 包含 ERROR 的行（NR = 行号）

# 统计和计算
awk '{sum += $5} END {print sum}' data.txt           # 求第 5 列的和
awk '{sum += $5} END {print sum/NR}' data.txt        # 求平均值

# 实用案例：统计 Nginx 访问日志中各 IP 的访问量
awk '{print $1}' access.log | sort | uniq -c | sort -rn | head -10

# 实用案例：统计 HTTP 状态码分布
awk '{print $9}' access.log | sort | uniq -c | sort -rn

# 格式化输出
awk '{printf "%-20s %s\n", $1, $7}' access.log       # 对齐输出
```

## 四、日志查看

### 4.1 tail - 实时追踪日志

```bash
# 查看最后 N 行
tail -n 50 /var/log/nginx/access.log

# 实时追踪（调试必备）
tail -f /var/log/nginx/error.log         # -f = follow
tail -F /var/log/app/app.log             # -F = 追踪 + 文件被轮转后重新打开

# 组合使用
tail -f app.log | grep --line-buffered "ERROR"    # 实时只看错误
tail -f app.log | grep --line-buffered -E "ERROR|WARN"  # 看错误和警告
```

### 4.2 less - 分页查看

```bash
less /var/log/syslog
# 快捷键：
# g     - 跳到文件开头
# G     - 跳到文件末尾
# /搜索词 - 向下搜索
# ?搜索词 - 向上搜索
# n     - 下一个搜索结果
# N     - 上一个搜索结果
# q     - 退出
# 空格  - 下一页
# b     - 上一页
```

### 4.3 journalctl - systemd 日志

```bash
# 查看某个服务的日志
sudo journalctl -u nginx                  # Nginx 的全部日志
sudo journalctl -u myapp -f               # 实时追踪（类似 tail -f）
sudo journalctl -u myapp --since today    # 只看今天的
sudo journalctl -u myapp --since "1 hour ago"  # 最近 1 小时
sudo journalctl -u myapp --no-pager       # 不用分页器（适合 grep）

# 查看系统日志
sudo journalctl -b                        # 本次启动以来的日志
sudo journalctl -p err                    # 只看错误级别
sudo journalctl --disk-usage              # 日志占用空间
sudo journalctl --vacuum-size=100M        # 清理日志到 100MB
```

## 五、Shell 脚本基础

### 5.1 基本结构

```bash
#!/bin/bash
# 上面这行叫 shebang，指定用哪个解释器运行脚本

# 这是一个注释
echo "Hello, World!"
```

```bash
# 运行脚本（两种方式）
chmod +x script.sh && ./script.sh    # 添加执行权限后直接运行
bash script.sh                        # 用 bash 解释器运行
```

### 5.2 变量

```bash
#!/bin/bash

# 定义变量（等号两边不能有空格！）
name="Alice"
age=25
server_port=8080

# 使用变量
echo "Name: $name"
echo "Age: ${age}"                   # 用 {} 明确变量边界
echo "Port: $server_port"

# 命令替换（把命令输出赋给变量）
current_date=$(date +%Y-%m-%d)
file_count=$(ls -1 | wc -l)
echo "Today: $current_date, Files: $file_count"

# 环境变量
echo "HOME: $HOME"
echo "PATH: $PATH"
echo "User: $USER"

# 位置参数（命令行传入的参数）
echo "脚本名: $0"
echo "第1个参数: $1"
echo "第2个参数: $2"
echo "所有参数: $@"
echo "参数个数: $#"
```

### 5.3 条件判断

```bash
#!/bin/bash

# if 语句
name="Alice"
if [ "$name" = "Alice" ]; then
    echo "Hello Alice!"
elif [ "$name" = "Bob" ]; then
    echo "Hello Bob!"
else
    echo "Unknown user"
fi

# 数字比较
age=25
if [ "$age" -ge 18 ]; then           # -ge = greater or equal
    echo "成年人"
fi

# 数字比较运算符：-eq(等于) -ne(不等于) -gt(大于) -lt(小于) -ge(>=) -le(<=)

# 文件测试
if [ -f "/etc/nginx/nginx.conf" ]; then    # -f 文件是否存在
    echo "Nginx 配置存在"
fi

if [ -d "/var/log/nginx" ]; then           # -d 目录是否存在
    echo "Nginx 日志目录存在"
fi

# 逻辑运算
if [ -f "config.txt" ] && [ -r "config.txt" ]; then   # && = AND
    echo "配置文件存在且可读"
fi

if [ "$DEBUG" = "true" ] || [ "$VERBOSE" = "true" ]; then  # || = OR
    echo "调试模式"
fi
```

### 5.4 循环

```bash
#!/bin/bash

# for 循环
for name in Alice Bob Charlie; do
    echo "Hello, $name!"
done

# 遍历文件
for file in /var/log/*.log; do
    echo "处理: $file"
    wc -l "$file"
done

# C 风格 for 循环
for ((i=1; i<=5; i++)); do
    echo "第 $i 次"
done

# while 循环
count=0
while [ "$count" -lt 5 ]; do
    echo "count = $count"
    count=$((count + 1))
done

# 读取文件每一行
while IFS= read -r line; do
    echo "行内容: $line"
done < input.txt
```

### 5.5 函数

```bash
#!/bin/bash

# 定义函数
greet() {
    local name=$1                # local 声明局部变量
    local age=$2
    echo "Hello, $name! You are $age years old."
}

# 调用函数
greet "Alice" 25
greet "Bob" 30

# 实用函数：带颜色的日志输出
log_info() {
    echo -e "\033[32m[INFO]\033[0m $(date '+%H:%M:%S') $1"
}

log_error() {
    echo -e "\033[31m[ERROR]\033[0m $(date '+%H:%M:%S') $1"
}

log_info "服务启动成功"
log_error "数据库连接失败"
```

### 5.6 实用脚本示例

```bash
#!/bin/bash
# deploy.sh - 简单的部署脚本

set -e                           # 任何命令失败立即退出
set -u                           # 使用未定义变量时报错

APP_DIR="/home/alice/myapp"
LOG_FILE="/var/log/deploy.log"

log() {
    echo "[$(date '+%Y-%m-%d %H:%M:%S')] $1" | tee -a "$LOG_FILE"
}

log "开始部署..."

cd "$APP_DIR"

log "拉取最新代码..."
git pull origin main

log "安装依赖..."
pip install -r requirements.txt

log "重启服务..."
sudo systemctl restart myapp

log "检查服务状态..."
sleep 3
if systemctl is-active --quiet myapp; then
    log "部署成功！"
else
    log "部署失败，正在回滚..."
    git reset --hard HEAD~1
    sudo systemctl restart myapp
    log "已回滚到上一版本"
    exit 1
fi
```

## 六、常见坑

### 坑 1：Shell 脚本等号两边有空格

```bash
# 错误
name = "Alice"        # ← bash 认为 name 是命令，= 和 "Alice" 是参数
if [ $name = "Bob" ]  # ← $name 为空时会语法错误

# 正确
name="Alice"          # 等号两边不能有空格
if [ "$name" = "Bob" ]  # 变量加引号防止空值报错
```

### 坑 2：curl 默认不跟随重定向

```bash
# 问题：请求返回 301/302 时 curl 不自动跳转
curl https://example.com          # 可能只看到重定向响应

# 解决：加 -L 参数
curl -L https://example.com       # 跟随重定向
```

### 坑 3：grep 管道缓冲导致看不到实时输出

```bash
# 问题：tail -f | grep 看不到实时结果
tail -f app.log | grep "error"    # grep 默认缓冲输出

# 解决：使用 --line-buffered
tail -f app.log | grep --line-buffered "error"
```

### 坑 4：sed -i 在不同系统上的行为差异

```bash
# macOS (BSD sed)
sed -i '' 's/old/new/g' file.txt    # -i 后必须跟备份后缀

# Linux (GNU sed)
sed -i 's/old/new/g' file.txt       # -i 后不需要空字符串

# 跨平台兼容写法
sed -i.bak 's/old/new/g' file.txt   # 两种系统都支持
```

### 坑 5：Shell 脚本中变量未加引号

```bash
# 问题：文件名包含空格时出错
file="my document.txt"
cat $file                # ← bash 拆成 cat "my" "document.txt"，报错

# 正确：变量永远加引号
cat "$file"              # ← 正确，"my document.txt" 作为整体

# if 判断中也一样
if [ -f "$file" ]; then  # ← 加引号
    echo "文件存在"
fi
```

### 坑 6：忘记 `set -e` 导致错误被忽略

```bash
#!/bin/bash
# 没有 set -e 的脚本
cd /nonexistent_dir       # 失败但继续执行
rm -rf *.tmp              # 在错误目录下删除文件！

# 加上 set -e
set -e
cd /nonexistent_dir       # 失败立即退出，不会执行危险的 rm
```
