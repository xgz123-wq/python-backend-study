# Docker 常用命令

> 镜像、容器、数据卷、网络——日常 20 条命令就够覆盖 90% 的容器操作。

## 一、概述

Docker 的命令体系庞大，但日常开发中高频使用的命令不超过 20 个。本章按照**镜像操作 → 容器操作 → 数据卷 → 网络**的顺序，系统梳理常用命令，并在末尾提供速查表。

## 二、镜像操作

### 2.1 拉取镜像 — `docker pull`

```bash
# 拉取官方镜像（默认 latest 标签）
docker pull python

# 指定版本标签
docker pull python:3.12-slim

# 拉取第三方镜像
docker pull bitnami/redis:7.2
```

镜像标签（Tag）的命名规则：`[registry/]repository[:tag]`

- `python:3.12-slim` → 官方仓库，3.12 精简版
- `bitnami/redis:7.2` → bitnami 组织维护的 Redis 镜像
- `myregistry.com/myapp:v1.0` → 私有仓库的镜像

### 2.2 查看本地镜像 — `docker images`

```bash
# 列出所有镜像
docker images

# 输出示例：
# REPOSITORY   TAG          IMAGE ID       CREATED        SIZE
# python       3.12-slim    a1b2c3d4e5f6   2 weeks ago    125MB
# redis        7.2          f6e5d4c3b2a1   3 weeks ago    117MB

# 过滤特定仓库的镜像
docker images python

# 只显示镜像 ID（便于批量操作）
docker images -q
```

### 2.3 构建镜像 — `docker build`

```bash
# 在当前目录构建镜像，指定名称和标签
docker build -t myapp:v1.0 .

# 指定 Dockerfile 路径
docker build -t myapp:v1.0 -f docker/Dockerfile .

# 构建时不使用缓存
docker build --no-cache -t myapp:v1.0 .

# 传递构建参数
docker build --build-arg PYTHON_VERSION=3.12 -t myapp:v1.0 .
```

### 2.4 给镜像打标签 — `docker tag`

```bash
# 给已有镜像打新标签
docker tag myapp:v1.0 myrepo/myapp:latest
docker tag myapp:v1.0 registry.example.com/myapp:v1.0
```

`docker tag` 不会复制镜像，只是创建了一个新的引用（类似硬链接）。

### 2.5 推送镜像 — `docker push`

```bash
# 登录 Docker Hub
docker login

# 推送镜像
docker push myrepo/myapp:latest

# 推送到私有仓库
docker push registry.example.com/myapp:v1.0
```

### 2.6 删除镜像 — `docker rmi`

```bash
# 按名称删除
docker rmi myapp:v1.0

# 按 Image ID 删除
docker rmi a1b2c3d4e5f6

# 强制删除（即使有容器引用）
docker rmi -f myapp:v1.0

# 清理所有未被使用的镜像
docker image prune -a
```

### 2.7 查看镜像历史

```bash
# 查看镜像的构建层
docker history python:3.12-slim
```

## 三、容器操作

### 3.1 运行容器 — `docker run`

这是最重要的命令，选项非常多：

```bash
# 最简运行
docker run python:3.12-slim python -c "print('hello')"

# 常用选项组合
docker run -d \                    # 后台运行（detached）
  --name my-container \            # 指定容器名称
  -p 8000:8000 \                   # 端口映射 宿主机:容器
  -v /host/path:/container/path \  # 挂载数据卷
  -e ENV_VAR=value \               # 设置环境变量
  --restart unless-stopped \       # 重启策略
  python:3.12-slim uvicorn app:app --host 0.0.0.0 --port 8000
```

关键参数详解：

| 参数              | 说明                                 |
|-----------------|--------------------------------------|
| `-d`            | 后台运行，不占用终端                     |
| `-it`           | 交互式运行（进入容器 shell 时使用）        |
| `--name`        | 给容器起名字，方便后续引用                 |
| `-p host:cont`  | 端口映射                              |
| `-v src:dst`    | 挂载卷或绑定目录                        |
| `-e KEY=VAL`    | 注入环境变量                           |
| `--rm`          | 容器停止后自动删除                       |
| `--restart`     | 重启策略：no / on-failure / always / unless-stopped |
| `--network`     | 指定网络                              |

### 3.2 查看容器 — `docker ps`

```bash
# 查看正在运行的容器
docker ps

# 查看所有容器（包括已停止的）
docker ps -a

# 输出示例：
# CONTAINER ID   IMAGE            STATUS         PORTS                    NAMES
# a1b2c3d4e5f6   python:3.12     Up 2 hours     0.0.0.0:8000->8000/tcp   my-app
# f6e5d4c3b2a1   redis:7.2       Exited (0)                              my-redis

# 只显示容器 ID
docker ps -q
```

### 3.3 容器生命周期管理

```bash
# 停止容器（发送 SIGTERM，优雅关闭）
docker stop my-container

# 强制停止（发送 SIGKILL）
docker kill my-container

# 启动已停止的容器
docker start my-container

# 重启容器
docker restart my-container

# 删除容器（必须先停止）
docker rm my-container

# 强制删除运行中的容器
docker rm -f my-container

# 清理所有已停止的容器
docker container prune
```

### 3.4 进入容器 — `docker exec`

```bash
# 在运行中的容器内执行命令
docker exec my-container python -c "print('inside container')"

# 交互式进入容器 shell（最常用）
docker exec -it my-container /bin/bash

# 如果容器没有 bash，用 sh
docker exec -it my-container /bin/sh
```

`docker exec` 与 `docker run` 的区别：`exec` 是在**已运行的容器内**执行命令，`run` 是**创建新容器**。

### 3.5 查看日志 — `docker logs`

```bash
# 查看全部日志
docker logs my-container

# 实时跟踪日志（类似 tail -f）
docker logs -f my-container

# 查看最近 100 行
docker logs --tail 100 my-container

# 显示时间戳
docker logs -t my-container

# 查看指定时间之后的日志
docker logs --since "2024-01-01T00:00:00" my-container
```

### 3.6 查看容器详细信息

```bash
# 查看容器配置信息（JSON 格式）
docker inspect my-container

# 查看容器的 IP 地址
docker inspect -f '{{range .NetworkSettings.Networks}}{{.IPAddress}}{{end}}' my-container

# 查看资源占用
docker stats my-container

# 查看容器内的进程
docker top my-container
```

### 3.7 文件复制

```bash
# 从宿主机复制到容器
docker cp /host/file.txt my-container:/container/file.txt

# 从容器复制到宿主机
docker cp my-container:/container/file.txt /host/file.txt
```

## 四、数据卷（Volume）

### 4.1 为什么需要数据卷

容器的可写层是临时的，容器删除后数据就丢失了。数据卷解决了以下问题：

- **数据持久化**：数据库容器的数据不能丢
- **数据共享**：多个容器之间共享文件
- **备份迁移**：卷数据可以独立备份

### 4.2 卷的操作

```bash
# 创建命名卷
docker volume create my-data

# 查看所有卷
docker volume ls

# 查看卷详情
docker volume inspect my-data

# 删除卷
docker volume rm my-data

# 清理未使用的卷
docker volume prune
```

### 4.3 三种挂载方式

```bash
# 1. 命名卷（Named Volume）— 推荐，由 Docker 管理
docker run -d -v my-data:/var/lib/mysql mysql:8.0

# 2. 绑定挂载（Bind Mount）— 开发常用，直接映射宿主机目录
docker run -d -v /home/user/project:/app myapp

# 3. tmpfs 挂载 — 仅存于内存，容器停止即消失
docker run -d --tmpfs /tmp myapp
```

对比：

| 方式       | 持久化  | 共享    | 典型场景           |
|----------|--------|---------|------------------|
| 命名卷     | 是      | 多容器共享 | 数据库存储          |
| 绑定挂载   | 是      | 多容器共享 | 开发时代码热更新      |
| tmpfs    | 否      | 不可共享  | 临时敏感文件         |

## 五、网络（Network）

### 5.1 Docker 网络类型

Docker 提供四种网络驱动：

**bridge（桥接）— 默认模式**
- 容器通过虚拟网桥 `docker0` 通信
- 容器之间可以用容器名互相访问（同一网络下）
- 最常用的模式

**host（主机）**
- 容器直接使用宿主机的网络栈
- 没有网络隔离，端口直接暴露
- 性能最好，仅 Linux 支持

**none（无网络）**
- 容器没有网络连接
- 用于纯安全隔离场景

**自定义网络**
- 用户创建的 bridge 网络
- 支持自动 DNS 解析（容器名 → IP）
- 推荐在生产环境中使用

### 5.2 网络操作

```bash
# 查看所有网络
docker network ls

# 创建自定义网络
docker network create my-network

# 在指定网络中运行容器
docker run -d --network my-network --name web myapp
docker run -d --network my-network --name db mysql:8.0

# 此时 web 容器可以直接用 "db" 作为主机名连接数据库
# 例如：mysql://db:3306/mydb

# 将运行中的容器连接到网络
docker network connect my-network my-container

# 断开网络
docker network disconnect my-network my-container

# 删除网络
docker network rm my-network

# 清理未使用的网络
docker network prune
```

### 5.3 端口映射

```bash
# 映射单个端口
docker run -p 8000:80 myapp

# 映射多个端口
docker run -p 8000:80 -p 8443:443 myapp

# 绑定到特定 IP
docker run -p 127.0.0.1:8000:80 myapp

# 随机分配宿主机端口
docker run -p 80 myapp
docker port <container_id>  # 查看分配的端口
```

## 六、系统清理

```bash
# 一键清理所有未使用的资源（镜像、容器、网络、卷）
docker system prune

# 加上 -a 清理所有未使用的镜像（不仅是 dangling 镜像）
docker system prune -a

# 加上 --volumes 同时清理未使用的卷
docker system prune -a --volumes

# 查看 Docker 磁盘占用
docker system df
```

> **警告**：`docker system prune -a --volumes` 会删除所有未被运行中容器引用的资源，包括你可能还想保留的镜像和卷数据。请在执行前仔细确认。

## 八、常见坑

### 坑 1：`docker stop` 和 `docker kill` 的区别

`docker stop` 发送 SIGTERM 信号，给应用 10 秒的优雅关闭时间（处理完当前请求、释放资源）。`docker kill` 直接发送 SIGKILL，强制终止。生产环境应该使用 `docker stop`。

### 坑 2：`docker rm` 不等于 `docker stop` + `docker rm`

`docker rm` 只能删除**已停止**的容器。如果想删除运行中的容器，需要 `docker rm -f`（等同于先 kill 再 rm）。但 `-f` 是强制操作，可能导致数据损坏。

### 坑 3：端口冲突

`docker run -p 8000:80` 失败？可能是宿主机的 8000 端口已被占用。用 `netstat -ano | findstr :8000`（Windows）或 `lsof -i :8000`（Linux/macOS）检查。

### 坑 4：容器名冲突

`docker run --name my-app ...` 报错 "name is already in use"？说明之前有同名容器（即使已停止）。先用 `docker rm my-app` 删除旧容器，或用 `docker ps -a` 查看。

### 坑 5：忘记 `-d` 导致终端被占用

运行容器时不加 `-d`，容器会以前台模式运行，终端被占用。按 `Ctrl+C` 会停止容器。正确做法是加 `-d` 后台运行，再用 `docker logs -f` 查看日志。
