# 进程管理与环境配置

> systemd 守住进程不挂、环境变量把配置和代码分离，这是上线前的最后一公里。

## 1. 概念解释

### 进程管理

在开发环境中，我们用 `python app.py` 或 `uvicorn app:app` 启动服务。但如果终端关闭、SSH 断开、服务器重启，进程就会终止。

**进程管理器**（如 systemd、Supervisor）的职责：

- **守护进程**：进程崩溃后自动重启
- **开机启动**：服务器重启后自动拉起服务
- **日志管理**：收集 stdout/stderr 并持久化
- **资源控制**：限制内存、CPU 使用量
- **优雅停止**：发送信号让进程完成当前请求后再退出

### 环境配置

不同环境（开发、测试、生产）需要不同的配置：

- 数据库连接字符串
- API 密钥
- 日志级别
- 第三方服务地址

**硬编码配置到代码中**是严重的安全隐患和维护噩梦。

---

## 2. 为什么需要

| 问题 | 没有进程管理 | 有进程管理 |
|------|-------------|-----------|
| 进程崩溃 | 服务停止，需要人工介入 | 自动重启，几秒恢复 |
| 服务器重启 | 服务不会自动启动 | 开机自动拉起 |
| 日志管理 | 输出到终端，丢失 | 持久化到文件/systemd journal |
| 多实例部署 | 手动管理多个进程 | systemd 模板统一管理 |
| 配置泄露 | 代码中硬编码密钥 | .env 文件与代码分离 |

```
├── 调试模式：False
└── 日志级别：WARNING
```

## 4. 核心用法

### 4.1 systemd 服务配置

systemd 是 Linux 默认的进程管理器，几乎所有现代 Linux 发行版都使用它。

```ini
# /etc/systemd/system/myapp.service

[Unit]
Description=My FastAPI Application
# 在网络就绪后启动
After=network.target
# 依赖 PostgreSQL（如果使用本地数据库）
Wants=postgresql.service

[Service]
# 运行用户（不要用 root！）
User=www-data
Group=www-data

# 工作目录
WorkingDirectory=/opt/myapp

# 环境变量文件
EnvironmentFile=/opt/myapp/.env

# 启动命令
ExecStart=/opt/myapp/venv/bin/gunicorn \
    -c /opt/myapp/gunicorn_config.py \
    app.main:app

# 重启策略：崩溃后总是重启
Restart=always
# 重启间隔（秒）
RestartSec=5

# 优雅停止的超时时间
TimeoutStopSec=30

# 安全加固
PrivateTmp=true
ProtectSystem=strict
ProtectHome=true
NoNewPrivileges=true

# 标准输出/错误 → systemd journal
StandardOutput=journal
StandardError=journal

[Install]
# 在多用户模式下启动（正常开机）
WantedBy=multi-user.target
```

**常用命令**：

```bash
# 重新加载配置（修改 .service 文件后必须执行）
sudo systemctl daemon-reload

# 启动/停止/重启服务
sudo systemctl start myapp
sudo systemctl stop myapp
sudo systemctl restart myapp

# 优雅重启（完成当前请求后重启）
sudo systemctl reload myapp

# 查看服务状态
sudo systemctl status myapp

# 设置开机启动
sudo systemctl enable myapp

# 禁用开机启动
sudo systemctl disable myapp

# 查看日志
sudo journalctl -u myapp -f          # 实时跟踪
sudo journalctl -u myapp --since today  # 今天的日志
sudo journalctl -u myapp --no-pager  # 全部日志
```

### 4.2 环境变量管理

#### 方式一：.env 文件（推荐开发环境使用）

```bash
# .env 文件
DATABASE_URL=postgresql://user:pass@localhost/mydb
SECRET_KEY=your-secret-key-here
API_KEY=sk-xxxxx
DEBUG=true
LOG_LEVEL=DEBUG
```

**Python 中读取 .env**：

```python
# 使用 python-dotenv
from dotenv import load_dotenv
import os

load_dotenv()  # 加载 .env 文件

database_url = os.getenv("DATABASE_URL")
secret_key = os.getenv("SECRET_KEY")
debug = os.getenv("DEBUG", "false").lower() == "true"
```

```python
# 使用 pydantic-settings（推荐 FastAPI 项目）
from pydantic_settings import BaseSettings

class Settings(BaseSettings):
    database_url: str
    secret_key: str
    api_key: str
    debug: bool = False
    log_level: str = "INFO"

    class Config:
        env_file = ".env"
        env_file_encoding = "utf-8"

settings = Settings()
```

#### 方式二：系统环境变量（推荐生产环境使用）

```bash
# 在 /etc/environment 中设置（全局）
DATABASE_URL=postgresql://user:pass@prod-db/mydb
SECRET_KEY=production-secret-key

# 在 systemd service 中设置
[Service]
Environment="DATABASE_URL=postgresql://user:pass@prod-db/mydb"
Environment="SECRET_KEY=production-secret-key"

# 或者使用 EnvironmentFile
[Service]
EnvironmentFile=/etc/myapp/env
```

#### 方式三：密钥管理服务（高级生产环境）

```
HashiCorp Vault / AWS Secrets Manager / Azure Key Vault
├── 动态生成数据库密码
├── 自动轮转密钥
└── 审计日志
```

### 4.3 配置分离：开发/测试/生产

```python
# config/settings.py

import os
from pydantic_settings import BaseSettings

class BaseConfig(BaseSettings):
    """基础配置（所有环境共享）"""
    app_name: str = "MyApp"
    api_prefix: str = "/api/v1"
    cors_origins: list[str] = []

class DevConfig(BaseConfig):
    """开发环境配置"""
    debug: bool = True
    log_level: str = "DEBUG"
    database_url: str = "sqlite:///./dev.db"
    cors_origins: list[str] = ["*"]

class ProdConfig(BaseConfig):
    """生产环境配置"""
    debug: bool = False
    log_level: str = "WARNING"
    database_url: str  # 必须从环境变量读取
    cors_origins: list[str] = ["https://myapp.com"]

    class Config:
        env_file = ".env"

# 根据环境变量选择配置
def get_settings():
    env = os.getenv("APP_ENV", "dev")
    config_map = {
        "dev": DevConfig,
        "test": DevConfig,
        "prod": ProdConfig,
    }
    return config_map[env]()
```

### 4.4 日志管理

#### 结构化日志

```python
import logging
import json
from datetime import datetime

class JSONFormatter(logging.Formatter):
    """将日志格式化为 JSON（便于日志收集系统解析）"""

    def format(self, record):
        log_data = {
            "timestamp": datetime.utcnow().isoformat(),
            "level": record.levelname,
            "message": record.getMessage(),
            "module": record.module,
            "function": record.funcName,
            "line": record.lineno,
        }
        # 添加额外字段
        if hasattr(record, "request_id"):
            log_data["request_id"] = record.request_id
        if hasattr(record, "user_id"):
            log_data["user_id"] = record.user_id

        return json.dumps(log_data, ensure_ascii=False)

# 配置日志
handler = logging.StreamHandler()
handler.setFormatter(JSONFormatter())

logger = logging.getLogger("myapp")
logger.setLevel(logging.INFO)
logger.addHandler(handler)

# 使用
logger.info("用户登录成功", extra={"user_id": 123, "request_id": "abc-123"})
# 输出: {"timestamp": "2024-01-01T12:00:00", "level": "INFO", "message": "用户登录成功", "user_id": 123, ...}
```

#### 日志轮转

```python
from logging.handlers import RotatingFileHandler

# 按文件大小轮转
handler = RotatingFileHandler(
    filename="/var/log/myapp/app.log",
    maxBytes=10 * 1024 * 1024,  # 10 MB
    backupCount=5,               # 保留 5 个历史文件
    encoding="utf-8"
)

# 按时间轮转（每天一个文件）
from logging.handlers import TimedRotatingFileHandler

handler = TimedRotatingFileHandler(
    filename="/var/log/myapp/app.log",
    when="midnight",       # 每天午夜轮转
    interval=1,            # 每 1 天
    backupCount=30,        # 保留 30 天
    encoding="utf-8"
)
```

**生产环境推荐**：使用 systemd journal，由系统统一管理日志轮转和持久化。

```bash
# 查看日志
sudo journalctl -u myapp -f

# 限制日志大小
sudo journalctl --vacuum-size=500M

# 限制日志保留时间
sudo journalctl --vacuum-time=7d
```

### 4.5 Supervisor 作为替代方案

如果系统不使用 systemd（如某些 Docker 容器、旧版系统），可以使用 Supervisor。

```bash
# 安装
sudo apt install supervisor
```

```ini
# /etc/supervisor/conf.d/myapp.conf

[program:myapp]
command=/opt/myapp/venv/bin/gunicorn -c /opt/myapp/gunicorn_config.py app.main:app
directory=/opt/myapp
user=www-data
autostart=true
autorestart=true
redirect_stderr=true
stdout_logfile=/var/log/myapp/supervisor.log
environment=APP_ENV="prod"
```

```bash
# 重新加载配置
sudo supervisorctl reread
sudo supervisorctl update

# 管理服务
sudo supervisorctl start myapp
sudo supervisorctl stop myapp
sudo supervisorctl restart myapp
sudo supervisorctl status myapp
```

**systemd vs Supervisor 对比**：

| 特性 | systemd | Supervisor |
|------|---------|-----------|
| 系统集成 | 原生集成 | 第三方 |
| 开机启动 | 自动支持 | 需要额外配置 |
| 日志管理 | journal 统一管理 | 自己管理文件 |
| 资源限制 | cgroups 精细控制 | 有限支持 |
| 配置复杂度 | 中等 | 简单 |
| 适用场景 | 生产服务器 | 容器、旧系统 |

## 5. 常见坑

### 5.1 忘记 daemon-reload

```bash
# 修改了 .service 文件但没有重新加载
# 错误：systemctl restart 后还是旧配置

# 解决：每次修改后都要执行
sudo systemctl daemon-reload
sudo systemctl restart myapp
```

### 5.2 使用 root 用户运行服务

```ini
# ❌ 错误：使用 root
User=root

# ✅ 正确：使用专用用户
User=www-data
```

**风险**：如果应用有漏洞，攻击者直接获得 root 权限。

### 5.3 .env 文件权限过大

```bash
# ❌ 错误：所有人可读
-rw-r--r-- 1 root root .env

# ✅ 正确：只有服务用户可读
sudo chown www-data:www-data .env
sudo chmod 600 .env
-rw------- 1 www-data www-data .env
```

### 5.4 虚拟环境路径错误

```ini
# ❌ 错误：使用系统 Python
ExecStart=/usr/bin/python -m gunicorn app.main:app

# ✅ 正确：使用虚拟环境中的 Python
ExecStart=/opt/myapp/venv/bin/gunicorn app.main:app
```

### 5.5 工作目录未设置

```ini
# ❌ 错误：没有 WorkingDirectory
# 导致相对路径导入失败

# ✅ 正确：明确设置工作目录
WorkingDirectory=/opt/myapp
```

### 5.6 环境变量未生效

```bash
# 问题：在 .bashrc 中设置的环境变量，systemd 服务读不到

# 解决：在 service 文件中明确指定
[Service]
EnvironmentFile=/opt/myapp/.env
# 或者
Environment="KEY=value"
```

### 5.7 日志文件权限问题

```bash
# 错误：Permission denied: '/var/log/myapp/app.log'

# 解决：创建目录并设置权限
sudo mkdir -p /var/log/myapp
sudo chown www-data:www-data /var/log/myapp
sudo chmod 755 /var/log/myapp
```

## 6. 总结

| 要点 | 说明 |
|------|------|
| systemd | Linux 标准进程管理器，生产首选 |
| .env 文件 | 开发环境使用，生产环境用系统环境变量 |
| 配置分离 | 根据 APP_ENV 自动选择配置 |
| 结构化日志 | JSON 格式，便于日志收集和分析 |
| 日志轮转 | 防止日志占满磁盘 |
| 安全 | 不用 root 运行，.env 权限 600 |
| Supervisor | systemd 不可用时的替代方案 |
